public class Student {
    private String studentId;    // 学生ID
    private String name;         // 名前
    private int age;             // 年齢
    private String department;   // 学部
    // getter/setter等
}

// メインクラスでの配列データ管理
Student[] students = {
    new Student("S001", "田中太郎", 20, "情報工学部"),

    new Student("S003", "鈴木次郎", 19, "文学部")
};

    データベーステーブル： 

    student テーブル
+----+----------+-----+----------+
| id | name     | age | department |

    🗄️ データベース・テーブル・フィールドの概念理解

    1. データベース（Database）とは

    データベースは、複数のテーブルをまとめて管理する「データの倉庫」です。

    sampledb（データベース）
├── student テーブル    ← 学生情報
├── course テーブル     ← 科目情報  
├── grade テーブル      ← 成績情報
└── teacher テーブル    ← 教師情報

    身近な例で理解： - データベース = 学校全体 - テーブル = 各教室（学生の教室、先生の教室など） - データ = 教室にいる人々

    2. テーブル（Table）とは

    テーブルは、同じ種類のデータをまとめて管理する「データの表」です。

    student テーブル（学生情報）
+----+----------+-----+----------+
| id | name     | age | department |
+----+----------+-----+----------+
| 1  | 田中太郎  | 20  | 情報工学部 |
| 2  | 佐藤花子  | 21  | 経済学部   |
+----+----------+-----+----------+

    Excelとの比較： - テーブル = Excelのワークシート - 行（Row） = Excelの1行（1件のデータ） - 列（Column） = Excelの1列（項目）

    3. フィールド（Field/Column）とは

    フィールドは、テーブルの「列」のことで、データの項目を表します。

    student テーブルのフィールド：
- id フィールド        ← 学生を識別するID
- name フィールド      ← 学生の名前
- age フィールド       ← 学生の年齢  
- department フィールド ← 学生の学部

    Java クラスとの対応： 

    public class Student {
    private int id;           // ← id フィールド
    private String name;      // ← name フィールド
    private int age;          // ← age フィールド
    private String department; // ← department フィールド
}

    重要なポイント: Javaクラスのフィールドとデータベースのフィールドはほぼ同じ概念です！

    📝 SQL基本文法：CREATE・INSERT・SELECTの理解

    SQLとは

    SQL（Structured Query Language）は、データベースを操作するための専用言語です。

    SQLの特徴： - 宣言的: 「何をしたいか」を記述（手順ではなく結果を指定） - 標準化: どのデータベースでもほぼ同じ文法 - シンプル: 英語に近い自然な記述

    1. CREATE文：テーブルの作成

    CREATE文は、新しいテーブルを作成するコマンドです。

    基本文法

    CREATE TABLE テーブル名 (
    フィールド名1 データ型 制約,
    フィールド名2 データ型 制約,
    フィールド名3 データ型 制約
);

    実践例：studentテーブルの作成

    CREATE TABLE student (
    id INT AUTO_INCREMENT PRIMARY KEY,
    name VARCHAR(100) NOT NULL,
    age INT NOT NULL,
    department VARCHAR(100) NOT NULL
);

    各部分の説明： - id INT AUTO_INCREMENT PRIMARY KEY: ID（整数、自動採番、主キー） - name VARCHAR(100) NOT NULL: 名前（最大100文字、必須） - age INT NOT NULL: 年齢（整数、必須） - department VARCHAR(100) NOT NULL: 学部（最大100文字、必須）

    主要なデータ型

    データ型 説明 Java対応
    INT 整数 int
    VARCHAR(n) 可変長文字列（最大n文字） String
    TEXT 長文テキスト String
    DATE 日付 LocalDate
    DATETIME 日時 LocalDateTime

    主要な制約

    制約 説明
    PRIMARY KEY 主キー（一意識別）
    NOT NULL 必須項目
    AUTO_INCREMENT 自動採番
    UNIQUE 重複禁止

    2. INSERT文：データの挿入

    INSERT文は、テーブルにデータを追加するコマンドです。

    基本文法

    INSERT INTO テーブル名 (フィールド名1, フィールド名2, ...) 
VALUES (値1, 値2, ...);

    実践例：学生データの追加

    -- 1件ずつ追加
INSERT INTO student (name, age, department) 
VALUES ('田中太郎', 20, '情報工学部');

INSERT INTO student (name, age, department) 
VALUES ('佐藤花子', 21, '経済学部');

INSERT INTO student (name, age, department) 
VALUES ('鈴木次郎', 19, '文学部');

    ポイント： - idはAUTO_INCREMENTなので指定不要（自動で1, 2, 3...が設定される） - 文字列はシングルクォートで囲む - 数値はクォート不要

    複数データの一括挿入

    INSERT INTO student (name, age, department) VALUES 
('田中太郎', 20, '情報工学部'),
('佐藤花子', 21, '経済学部'),
('鈴木次郎', 19, '文学部');

    3. SELECT文：データの取得

    SELECT文は、テーブルからデータを取得するコマンドです。

    基本文法

    SELECT フィールド名1, フィールド名2, ... 
FROM テーブル名
WHERE 条件;

    実践例：データの取得

    全データの取得： 

    SELECT * FROM student;

    実行結果： 

    +----+----------+-----+----------+
| id | name     | age | department |
+----+----------+-----+----------+
| 1  | 田中太郎  | 20  | 情報工学部 |
| 2  | 佐藤花子  | 21  | 経済学部   |
| 3  | 鈴木次郎  | 19  | 文学部    |
+----+----------+-----+----------+

    特定フィールドのみ取得： 

    SELECT name, age FROM student;

    実行結果： 

    +----------+-----+
| name     | age |
+----------+-----+
| 田中太郎  | 20  |
| 佐藤花子  | 21  |
| 鈴木次郎  | 19  |
+----------+-----+

    条件付き取得： 

    SELECT * FROM student WHERE age >= 20;

    実行結果： 

    +----+----------+-----+----------+
| id | name     | age | department |
+----+----------+-----+----------+
| 1  | 田中太郎  | 20  | 情報工学部 |
| 2  | 佐藤花子  | 21  | 経済学部   |
+----+----------+-----+----------+

    SQLの記述ポイント： - 大文字小文字: SELECTはselectでも可（慣習的に大文字） - セミコロン: 文末には;を付ける - スペース: 適切な空白で読みやすく記述

    🔗 Week 5システムのデータベース化設計

    配列データからテーブル設計への変換

    Week 5で使用していた配列データを、データベースのテーブル設計に変換してみましょう。

    Week 5のJavaクラス

    public class Student {
    private String studentId;    // 学生ID
    private String name;         // 名前
    private int age;             // 年齢
    private String department;   // 学部

    // 成績データも含んでいた場合
    private int mathScore;       // 数学の点数
    private int englishScore;    // 英語の点数
    private int scienceScore;    // 理科の点数
}

    データベース設計への変換

    Option 1: 単一テーブル設計 

    CREATE TABLE student (
    id INT AUTO_INCREMENT PRIMARY KEY,
    student_id VARCHAR(10) NOT NULL UNIQUE,
    name VARCHAR(100) NOT NULL,
    age INT NOT NULL,
    department VARCHAR(100) NOT NULL,
    math_score INT,
    english_score INT,
    science_score INT
);

    Option 2: 正規化設計（推奨） 

    -- 学生基本情報テーブル
CREATE TABLE student (
    id INT AUTO_INCREMENT PRIMARY KEY,
    student_id VARCHAR(10) NOT NULL UNIQUE,
    name VARCHAR(100) NOT NULL,
    age INT NOT NULL,
    department VARCHAR(100) NOT NULL
);

-- 成績テーブル（学生と科目の関係を管理）
CREATE TABLE grade (
    id INT AUTO_INCREMENT PRIMARY KEY,
    student_id INT NOT NULL,
    subject VARCHAR(50) NOT NULL,
    score INT NOT NULL,
    FOREIGN KEY (student_id) REFERENCES student(id)
);

    正規化設計の利点： - データの重複削除: 学生情報は1箇所のみ - 拡張性: 科目を後から追加しやすい - 整合性: データの矛盾を防げる

('S002', '佐藤花子', 21, '経済学部'),

    gradeテーブル：

    Q5: 複雑なクエリが書けません

    A: 段階的に学習しましょう：
