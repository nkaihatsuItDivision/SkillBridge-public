    📊 集約関数とは

    集約関数の基本概念

    集約関数は、複数行のデータを1つの値にまとめる関数です。データの統計分析や要約に必須の技術です。

    -- 基本的な集約関数
SELECT COUNT(*) FROM student;        -- レコード数をカウント
SELECT SUM(score) FROM score;       -- 合計値を計算
SELECT AVG(score) FROM score;       -- 平均値を計算
SELECT MAX(score) FROM score;       -- 最大値を取得
SELECT MIN(score) FROM score;       -- 最小値を取得

    集約関数の種類と用途

    関数 用途 例
    COUNT() 行数のカウント 学生数、履修者数
    SUM() 合計値の計算 総得点、総単位数
    AVG() 平均値の計算 平均点、平均年齢
    MAX() 最大値の取得 最高点、最新日付
    MIN() 最小値の取得 最低点、最古日付

    🔍 基本的な集約関数の実践

    ステップ1: 全体統計の取得

    まず、全データに対する基本的な統計を取得してみましょう。

    -- 学生管理システムの基本統計
SELECT 
    COUNT(*) as total_students,
    MIN(age) as youngest_age,
    MAX(age) as oldest_age,
    AVG(age) as average_age
FROM student;

    📘 集計処理の概念図

    実行結果例： 

    total_students | youngest_age | oldest_age | average_age
            15 |           18 |         24 |       20.8

    ステップ2: 成績データの統計分析

    成績テーブルから、より詳細な統計情報を取得します。

    -- 成績データの基本統計
SELECT 
    COUNT(*) as total_scores,
    COUNT(DISTINCT student_id) as students_with_scores,
    COUNT(DISTINCT subject_code) as subjects_with_scores,
    SUM(score) as total_points,
    AVG(score) as average_score,
    MAX(score) as highest_score,
    MIN(score) as lowest_score
FROM score;

    🎯 ポイント：COUNT()の使い分け - COUNT(*): 全行数（NULL値も含む） - COUNT(列名): NULL値を除いた行数 - COUNT(DISTINCT 列名): 重複を除いた行数

    ステップ3: NULL値を考慮した集約

    実際のデータでは、NULL値の扱いが重要です。

    -- NULL値を考慮した統計（emailフィールドの例）
SELECT 
    COUNT(*) as total_students,
    COUNT(email) as students_with_email,
    COUNT(*) - COUNT(email) as students_without_email,
    ROUND(COUNT(email) * 100.0 / COUNT(*), 1) as email_rate_percent
FROM student;

    📊 GROUP BY句によるグループ化

    GROUP BY句の基本概念

    GROUP BY句は、指定した列の値が同じレコードをグループ化し、グループごとに集約関数を適用します。

    📘 GROUP BYの概念図

    ステップ1: 学部別の学生数

    -- 学部別の学生数を集計
SELECT 
    department,
    COUNT(*) as student_count
FROM student 
GROUP BY department
ORDER BY student_count DESC;

    実行結果例： 

    department          | student_count
情報工学部          |             6
経済学部           |             4
文学部             |             3
理学部             |             2

    🎯 HAVING句による条件フィルタリング

    WHERE句とHAVING句の違い

    • WHERE句: グループ化前の行をフィルタリング
    • HAVING句: グループ化後の集計結果をフィルタリング

    📘 WHEREとHAVINGの概念図

    ステップ1: HAVING句の基本使用

    -- 平均点が80点以上の科目のみ表示
SELECT 
    sub.subject_name,
    COUNT(sc.score) as student_count,
    ROUND(AVG(sc.score), 1) as avg_score
FROM subject sub
LEFT JOIN score sc ON sub.subject_code = sc.subject_code
GROUP BY sub.subject_code, sub.subject_name
HAVING AVG(sc.score) >= 80
ORDER BY avg_score DESC;

    ステップ2: 複数条件でのフィルタリング

    -- 履修者が3名以上で、かつ平均点が75点以上の科目
SELECT 
    sub.subject_name,
    COUNT(sc.score) as student_count,
    ROUND(AVG(sc.score), 1) as avg_score,
    MAX(sc.score) as max_score
FROM subject sub
INNER JOIN score sc ON sub.subject_code = sc.subject_code
GROUP BY sub.subject_code, sub.subject_name
HAVING COUNT(sc.score) >= 3 AND AVG(sc.score) >= 75
ORDER BY avg_score DESC;

    ステップ3: WHERE句とHAVING句の組み合わせ

    -- 2024年度の成績で、学部ごとの平均点を算出（平均点70点以上のみ）
SELECT 
    s.department,
    COUNT(sc.score) as total_scores,
    ROUND(AVG(sc.score), 1) as avg_score,
    COUNT(DISTINCT s.student_id) as unique_students
FROM student s
INNER JOIN score sc ON s.student_id = sc.student_id
WHERE sc.semester LIKE '2024%'  -- WHERE: グループ化前のフィルタ
GROUP BY s.department
HAVING AVG(sc.score) >= 70      -- HAVING: グループ化後のフィルタ
ORDER BY avg_score DESC;

    📈 実用的な統計分析レポート

    レポート1: 総合成績ランキング

    -- 学生別の総合成績ランキング（複数科目受講者のみ）
SELECT 
    s.student_id,
    s.name,
    s.department,
    COUNT(sc.score) as subjects_taken,
    ROUND(AVG(sc.score), 1) as avg_score,
    SUM(sc.score) as total_score,
    MAX(sc.score) as best_score,

        WHEN AVG(sc.score) >= 90 THEN '優秀'
        WHEN AVG(sc.score) >= 80 THEN '良好'
        WHEN AVG(sc.score) >= 70 THEN '普通'
        ELSE '要改善'
    END as performance_level
FROM student s

SELECT 
    sub.subject_name,
    sub.credits,
    COUNT(sc.score) as student_count,
    ROUND(AVG(sc.score), 1) as avg_score,

SELECT 
    s.department,
    COUNT(sc.score) as score_count,
    ROUND(AVG(sc.score), 1) as dept_avg,
    (SELECT ROUND(AVG(score), 1) FROM score) as overall_avg,

SELECT 
    sub.subject_name,

    COUNT(sc.score) as enrollment,

    💡 よくある質問

    Q1: GROUP BY使用時に「エラー1055」が発生します

    A: SELECT句でGROUP BY句に含まれていない列を指定している可能性があります：

    -- ❌ エラー：nameがGROUP BYに含まれていない
SELECT name, department, COUNT(*)
FROM student
GROUP BY department;

-- ✅ 正しい：集約関数を使用するか、GROUP BYに含める
SELECT department, COUNT(*) as student_count
FROM student
GROUP BY department;

    Q2: HAVINGとWHEREの違いがよくわかりません

    A: 実行順序で覚えましょう： 1. WHERE → データを絞り込み 2. GROUP BY → グループ化 3. 集約関数 → 計算実行 4. HAVING → 計算結果を絞り込み

    SELECT department, AVG(age)
FROM student
WHERE age >= 18        -- 1. 18歳以上のデータのみ
GROUP BY department   -- 2. 学部でグループ化
HAVING AVG(age) > 20  -- 4. 平均年齢20歳超の学部のみ

    Q3: 複雑な集計でパフォーマンスが悪くなります

    A: 以下の最適化を検討してください： - インデックス追加：GROUP BY対象列に適切なインデックス - 事前フィルタ：WHERE句での不要データ除外 - LIMIT句活用：必要最小限のデータ取得
