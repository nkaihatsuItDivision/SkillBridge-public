    4. トランザクション：データ整合性を保つ安全な操作

    🔒 トランザクションとは？

    📘 トランザクションの概念

    トランザクションとは、複数のSQL操作をひとまとまりとして扱う仕組みです。

    トランザクションの特徴（ACID特性）

    • A（Atomicity/原子性）: 全て成功するか、全て失敗するか
    • C（Consistency/一貫性）: データの整合性が保たれる
    • I（Isolation/分離性）: 他の操作と独立して実行される
    • D（Durability/永続性）: 確定した変更は永続化される

    📝 トランザクションの基本操作

    -- トランザクション開始
BEGIN;

-- 複数のSQL操作を実行
INSERT INTO student (student_id, name, age, department) 
VALUES ('S015', '田村花恵', 20, '経済学部');

UPDATE student 
SET age = 22 
WHERE student_id = 'S005';

DELETE FROM student 
WHERE student_id = 'S009';

-- 全て成功した場合：変更を確定
COMMIT;

-- 問題があった場合：変更を取り消し
-- ROLLBACK;

    🔧 実践的なトランザクション活用例

    例：学生の学部変更に伴う一連の操作

    -- トランザクション開始
BEGIN;

-- 学生の学部を変更
UPDATE student 
SET department = '情報工学部' 
WHERE student_id = 'S003';

-- 関連する情報も同時に更新（実際のシステムでは関連テーブルも更新）
-- この例では年齢も同時に調整
UPDATE student 
SET age = 20 
WHERE student_id = 'S003';

-- 変更内容を確認
SELECT student_id, name, age, department 
FROM student 
WHERE student_id = 'S003';

-- 問題なければ確定
COMMIT;

    実行結果： 

    +------------+--------------+-----+----------------+
| student_id | name         | age | department     |
+------------+--------------+-----+----------------+
| S003       | 鈴木次郎     |  20 | 情報工学部     |
+------------+--------------+-----+----------------+
1 row in set (0.00 sec)

Query OK, 0 rows affected (0.01 sec)

    エラー時のロールバック例

    -- トランザクション開始
BEGIN;

-- 意図的にエラーを発生させる例
INSERT INTO student (student_id, name, age, department) 
VALUES ('S001', '重複テスト', 25, 'テスト学部');  -- S001は既存のため重複エラー

-- エラーが発生した場合の処理
-- ERROR 1062 (23000): Duplicate entry 'S001' for key 'student_id'

-- エラーが発生したので変更を取り消し
ROLLBACK;

    💡 トランザクションのメリット

    1. データ整合性の確保: 途中でエラーが発生しても一部だけ更新される心配がない
    2. 安全性の向上: 問題があれば全体を取り消せる
    3. 複雑な操作の管理: 複数テーブルにまたがる操作も安全に実行

    📘 CRUD処理の統合イメージ

    CRUDは、データベース操作の基本となる4つの操作の頭文字です：

    -- 学生管理システムの統計レポート作成
-- レッスン7.3の技術と7.4の技術を組み合わせ

-- 基本統計
SELECT 
    COUNT(*) as '総学生数',
    COUNT(DISTINCT department) as '学部数',
    ROUND(AVG(age), 1) as '平均年齢',
    MIN(age) as '最年少',
    MAX(age) as '最年長'
FROM student;

-- 学部別詳細分析
SELECT 
    department as '学部',
    COUNT(*) as '在籍数',
    ROUND(AVG(age), 1) as '平均年齢',
    MIN(age) as '最年少',
    MAX(age) as '最年長'
FROM student 
GROUP BY department 
ORDER BY COUNT(*) DESC, department;

-- 年齢分布分析
SELECT 
    age as '年齢',
    COUNT(*) as '人数',
    ROUND(COUNT(*) * 100.0 / (SELECT COUNT(*) FROM student), 1) as '割合(%)'
FROM student 
GROUP BY age 
ORDER BY age;

    8. セキュリティとベストプラクティス

    🔐 SQL操作のセキュリティ原則

    1. WHERE句の必須化

    -- ❌ 危険な操作
UPDATE student SET age = 20;          -- 全レコード更新
DELETE FROM student;                  -- 全レコード削除

-- ✅ 安全な操作
UPDATE student SET age = 20 WHERE student_id = 'S001';
DELETE FROM student WHERE student_id = 'S001';

    2. トランザクションの活用

    -- 重要な操作は必ずトランザクション内で実行
BEGIN;
-- 複数のSQL操作
COMMIT;  -- または ROLLBACK;

    3. バックアップの確認

    -- 重要な削除・更新前は必ずデータを確認
