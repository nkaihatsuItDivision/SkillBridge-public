    Week 7.5で構築したテーブル構造

    Week 7.5で、あなたはWeek 5の学生管理システムを完全にデータベース化しました：

    -- 学生基本情報テーブル
CREATE TABLE student (
    id INT AUTO_INCREMENT PRIMARY KEY,
    student_id VARCHAR(10) NOT NULL UNIQUE,
    name VARCHAR(100) NOT NULL,
    age INT NOT NULL,
    department VARCHAR(100) NOT NULL,
    email VARCHAR(255),
    enrollment_date DATE,
    status ENUM('active', 'inactive', 'graduated') DEFAULT 'active'
);

-- 科目マスタテーブル
CREATE TABLE subject (
    id INT AUTO_INCREMENT PRIMARY KEY,
    subject_code VARCHAR(10) NOT NULL UNIQUE,
    subject_name VARCHAR(100) NOT NULL,
    credits INT NOT NULL,
    department VARCHAR(100)
);

-- 成績管理テーブル
CREATE TABLE score (
    id INT AUTO_INCREMENT PRIMARY KEY,
    student_id VARCHAR(10) NOT NULL,
    subject_code VARCHAR(10) NOT NULL,
    score INT NOT NULL,
    semester VARCHAR(20) NOT NULL,
    grade_letter ENUM('A', 'B', 'C', 'D', 'F'),
    FOREIGN KEY (student_id) REFERENCES student(student_id),
    FOREIGN KEY (subject_code) REFERENCES subject(subject_code)
);

    🔧 LEFT JOINの活用

    LEFT JOINとは？

    LEFT JOINは、左側（FROM句で指定）のテーブルのデータをすべて取得し、右側のテーブルから対応するデータがあれば結合する機能です。

    INNER JOINとLEFT JOINの違い

    -- INNER JOIN：成績があるデータのみ
SELECT s.name, sc.score
FROM student s
INNER JOIN score sc ON s.student_id = sc.student_id;

-- LEFT JOIN：全学生を表示（成績がなくてもNULLで表示）
SELECT s.name, sc.score
FROM student s
LEFT JOIN score sc ON s.student_id = sc.student_id;

    🔍 サブクエリの基本

    サブクエリとは？

    サブクエリは、SQL文の中に埋め込まれた別のSQL文です。メインクエリの条件として、別の検索結果を利用できます。

    サブクエリの基本構文

    SELECT カラム名
FROM テーブル名
WHERE カラム名 = (SELECT カラム名 FROM テーブル名 WHERE 条件);

    -- 1. 優秀学生推薦リスト
SELECT 
    s.student_id,
    s.name,
    s.department,
    COUNT(sc.id) as subjects_taken,
    ROUND(AVG(sc.score), 1) as avg_score,
    MIN(sc.score) as lowest_score

INNER JOIN score sc ON s.student_id = sc.student_id
WHERE s.status = 'active'
GROUP BY s.student_id, s.name, s.department
HAVING AVG(sc.score) >= 85 
   AND COUNT(sc.id) >= 2 
   AND MIN(sc.score) >= 60
ORDER BY avg_score DESC;

-- 2. 学部間比較レポート
SELECT 
    s.department,
    COUNT(DISTINCT s.student_id) as student_count,
    ROUND(AVG(sc.score), 1) as dept_avg_score,
    ROUND(AVG(subject_count.count), 1) as avg_subjects_per_student,
    SUM(CASE WHEN sc.grade_letter = 'A' THEN 1 ELSE 0 END) as total_a_grades,
    SUM(CASE WHEN sc.grade_letter = 'B' THEN 1 ELSE 0 END) as total_b_grades
FROM student s
LEFT JOIN score sc ON s.student_id = sc.student_id
LEFT JOIN (
    SELECT student_id, COUNT(*) as count
    FROM score
    GROUP BY student_id
) subject_count ON s.student_id = subject_count.student_id
WHERE s.status = 'active'
GROUP BY s.department
ORDER BY dept_avg_score DESC;

    💡 よくある質問

    Q1: INNER JOINとLEFT JOINの使い分けは？

    A: データの取得方針で決めましょう：

    • INNER JOIN: 「両方にデータがある確実な情報のみ欲しい」場合
    • LEFT JOIN: 「すべてのデータを表示し、関連情報があれば表示したい」場合

    Q2: サブクエリとJOINはどちらが効率的ですか？

    A: 一般的にJOINの方が効率的ですが、用途によります：

    -- サブクエリ版（理解しやすい）
SELECT * FROM student 
WHERE student_id IN (SELECT student_id FROM score WHERE score >= 90);

-- JOIN版（高速）
SELECT DISTINCT s.* FROM student s
INNER JOIN score sc ON s.student_id = sc.student_id
WHERE sc.score >= 90;

    Q3: 複雑なクエリのデバッグ方法は？

    A: 段階的に構築しましょう：

    1. 基本的なSELECTから開始
    2. 1つずつJOINを追加
    3. WHEREやGROUP BYを追加
    4. 最後にサブクエリや複雑な条件を追加
