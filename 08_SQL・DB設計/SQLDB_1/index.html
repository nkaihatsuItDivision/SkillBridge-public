    public class FileBasedStudentSystem {
    private static final String STUDENT_FILE = "students.txt";
    private static Map<String, Student> students = new HashMap<>();

    // ファイルからデータを読み込み
    public static void loadFromFile() {
        try (BufferedReader reader = new BufferedReader(new FileReader(STUDENT_FILE))) {
            String line;
            while ((line = reader.readLine()) != null) {
                String[] parts = line.split(",");
                if (parts.length >= 4) {
                    String id = parts[0];
                    String name = parts[1];
                    int age = Integer.parseInt(parts[2]);
                    String major = parts[3];

                    Student student = new UndergraduateStudent(name, age, major);
                    students.put(id, student);
                }
            }
            System.out.println("ファイルからデータを読み込みました");
        } catch (IOException | NumberFormatException e) {
            System.err.println("ファイル読み込みエラー: " + e.getMessage());
        }
    }

    // ファイルにデータを保存
    public static void saveToFile() {
        try (PrintWriter writer = new PrintWriter(new FileWriter(STUDENT_FILE))) {
            for (Map.Entry<String, Student> entry : students.entrySet()) {
                writer.printf("%s,%s,%d,%s%n", 
                    entry.getKey(),
                    entry.getValue().getName(),
                    entry.getValue().getAge(),
                    entry.getValue().getMajor());
            }
            System.out.println("ファイルにデータを保存しました");
        } catch (IOException e) {
            System.err.println("ファイル保存エラー: " + e.getMessage());
        }
    }
}

    3. データベースとは何か？

    💡 データベースの基本概念

    これまで見てきた問題を解決するために開発されたのがデータベース（Database）です。

    データベースの定義

    データベースとは： - 整理された方法でデータを保存・管理するシステム - 複数のユーザーが同時に安全にアクセスできる - データの整合性・信頼性・効率性を確保する

    身近な例で理解しよう

    図書館の管理システムで考えてみましょう：

    図書館（ファイル管理方式）の問題
    • 本の台帳：手書きのノートで管理
    • 貸出記録：別のノートで管理
    • 会員情報：また別のノートで管理

    問題： - 同じ時間に複数の司書が記録すると混乱 - 本の検索に時間がかかる - 記録ミスによる在庫の不一致 - 台帳の紛失リスク

    図書館（データベース方式）の解決
    • 統合管理システム：すべての情報を一元管理
    • 同時アクセス制御：複数の司書が安全に操作
    • 高速検索：著者・タイトル・カテゴリで瞬時に検索
    • 自動整合性チェック：在庫数の自動計算・矛盾検出
    • バックアップ・復旧：データの安全な保護

    このレッスンで理解したこと

    現在のデータ管理の限界

    • ✅ データの一時性: プログラム終了でデータ消失
    • ✅ 共有の困難: 複数ユーザー・PC間でのデータ共有問題
    • ✅ 検索の非効率: 大量データでの処理速度問題
    • ✅ 整合性の課題: 手動管理によるデータ不整合リスク

    データベースの価値

    • ✅ 永続性: データの確実な保存・管理
    • ✅ 同時アクセス: 複数ユーザーの安全な並行処理
    • ✅ 高速処理: 効率的な検索・集計機能
    • ✅ 整合性保証: 自動的なデータ整合性チェック

    企業・ビジネスでの重要性

    • ✅ 意思決定支援: データに基づく迅速で正確な判断
    • ✅ 競争優位: データ活用による差別化・効率化
    • ✅ コンプライアンス: 法的要件・規制への確実な対応
    • ✅ キャリア価値: 就職・昇進での技術的差別化
