    Step 2: SSH設定による安全な通信

    🔒 なぜSSH設定が必要なのか？

    HTTPS vs SSH

    HTTPS接続：
- 毎回ユーザー名・パスワード入力が必要
- パスワード管理が大変
- セキュリティリスク

SSH接続：
- 一度設定すれば認証自動化
- 公開鍵暗号による高セキュリティ
- プロフェッショナル開発者の標準

    🛠️ SSH設定手順

    1. SSHキーの生成

    Eclipseを開き、Ctrl + Alt + T（またはメニュー > Window > Show View > Other > General > Terminal）でターミナルを開きます。

    ssh-keygen -t rsa -b 4096 -C &quot;your_email@example.com&quot;

    コマンド解説： - ssh-keygen: SSHキー生成コマンド - -t rsa: RSA暗号化方式を使用 - -b 4096: 4096ビットの鍵長（高セキュリティ） - -C: コメント（メールアドレスを識別用に付与）

    実行結果： 

    Generating public/private rsa key pair.
Enter file in which to save the key (/home/username/.ssh/id_rsa):

    Enterキーを押してデフォルトの保存場所を使用します。

    Enter passphrase (empty for no passphrase):

    強いパスフレーズを入力（推奨）またはEnterキーで空のまま進みます。

    2. 公開鍵の確認

    cat ~/.ssh/id_rsa.pub

    実行結果： 

    ssh-rsa AAAAB3NzaC1yc2EAAAADAQABAAACAQCxxxxx... your_email@example.com

    この全体をコピーしてください（ssh-rsaから最後のメールアドレスまで）。

    3. GitHubへの公開鍵登録

    GitHub上での設定： 1. GitHub > Settings > SSH and GPG keys 2. "New SSH key"をクリック 3. Title: 識別しやすい名前（例：「My Development PC」） 4. Key: 先ほどコピーした公開鍵を貼り付け 5. "Add SSH key"をクリック

    📘 GitHub SSH鍵のセットアップ

    4. SSH接続テスト

    ssh -T git@github.com

    実行結果： 

    Hi username! You&apos;ve successfully authenticated, but GitHub does not provide shell access.

    このメッセージが表示されればSSH設定成功です！

    ⚠️ 初回接続時の確認： 

    The authenticity of host &apos;github.com&apos; can&apos;t be established.
...
Are you sure you want to continue connecting (yes/no)?
    "yes"と入力してEnterキーを押してください。

    Step 4: ローカル・リモート連携

    🔗 リモートリポジトリとの接続

    1. 現在のリポジトリ確認

    Week 5システムのディレクトリに移動し、リモートリポジトリの設定状況を確認します：

    cd /path/to/StudentManagementSystem
git remote -v

    実行結果： 

    （何も表示されない = リモートリポジトリ未設定）

    2. リモートリポジトリの追加

    git remote add origin git@github.com:username/student-management-system.git

    コマンド解説： - git remote add: リモートリポジトリを追加 - origin: リモートリポジトリの名前（慣例的にmainリポジトリをoriginと命名） - git@github.com:...: 先ほど作成したリポジトリのSSH URL

    3. リモートリポジトリ設定確認

    git remote -v

    実行結果： 

    origin  git@github.com:username/student-management-system.git (fetch)
origin  git@github.com:username/student-management-system.git (push)

    fetch: リモートから取得するURL
    push: リモートに送信するURL

    4. 接続テスト

    git fetch origin

    実行結果： 

    warning: no common commits
remote: Enumerating objects: 0, done.
remote: Counting objects: 100% (0/0), done.
remote: Total 0 (delta 0), reused 0 (delta 0), pack-reused 0

    エラーが出なければ接続成功です！

    🎯 リモートリポジトリの概念理解

    ローカルとリモートの関係

    📘 ローカルとリモートの関係

    ローカルリポジトリ（あなたのPC）
├── .git/（Gitの管理データ）
├── src/（Javaソースコード）
├── StudentManagementSystem.java
└── ... 

        ↕ git push / git pull

リモートリポジトリ（GitHub）
├── .git/（Gitの管理データ）
├── src/（同期されたソースコード）
├── StudentManagementSystem.java
└── ...

    originの役割

    origin = 主要なリモートリポジトリの別名

実際のURL:
git@github.com:username/student-management-system.git

別名:
origin

    これにより、毎回長いURLを入力する必要がなくなります： 

    # URLを直接指定（面倒）
git push git@github.com:username/student-management-system.git main

# originという別名を使用（簡単）
git push origin main

    💡 トラブルシューティング

    よくある問題と解決方法

    SSH接続エラー

    Permission denied (publickey)
    解決方法： 1. 公開鍵が正しくGitHubに登録されているか確認 2. SSHキーのパスが正しいか確認: ls ~/.ssh/ 3. SSH設定ファイルの確認: cat ~/.ssh/config

    リモートリポジトリ接続エラー

    fatal: remote origin already exists
    解決方法： 
    git remote remove origin
git remote add origin git@github.com:username/repository.git

    認証情報エラー

    fatal: Authentication failed
    解決方法： 1. SSH接続テスト再実行: ssh -T git@github.com 2. HTTPSではなくSSH URLを使用しているか確認 3. GitHubのSSHキー設定を再確認
