    🕐 なぜ履歴管理が重要なのか？

    📈 開発現場のリアルな課題

    📘 git logによる履歴の概要

    よくある開発トラブル

    1. 「昨日まで動いていたのに突然動かない」 → 何を変更したか分からない
    2. 「新機能を追加したらバグが発生」 → 安全な状態に戻したい
    3. 「過去のバージョンの方が良かった」 → 特定時点の状態を確認したい
    4. 「間違って重要なファイルを削除」 → 以前の状態を復旧したい

    履歴管理による解決

    • ✅ 変更履歴の詳細確認: いつ・誰が・何を変更したかが明確
    • ✅ 安全な状態への復旧: 問題発生前の安全な状態に戻れる
    • ✅ 過去バージョンの参照: 以前の実装方法を確認できる
    • ✅ 実験的開発: 失敗しても元に戻せるので積極的に試せる

    💡 履歴管理の威力

    履歴管理をマスターすると、開発で「タイムマシン」を使えるようになります！

    • 過去に戻る: 問題のない状態に瞬時に復旧
    • 未来を試す: 実験的な変更を安全に試行
    • 並行開発: 複数の機能を同時に開発（Week 6.6で学習）
    • 安心感: 「失敗しても戻せる」という心理的安全性

    Detached HEAD状態の理解

    Step 3: git reset による変更の取り消し

    🔄 resetの3つのモード

    git resetは、コミット履歴を調整する強力なコマンドです。用途に応じて3つのモードがあります：

    📘 git reset --hardによる復元

    resetの3つのモード

    1. --soft: コミットのみ取り消し、変更は保持
    2. --mixed (デフォルト): コミットとステージを取り消し、ファイルは保持
    3. --hard: すべてを取り消し、ファイルも元に戻す

    ⚠️ 安全な実験環境の準備

    実際のresetを試す前に、安全な実験用コミットを作成しましょう：

    実験用の変更作成

    echo "# 実験用のテストファイル" > test_file.txt
git add test_file.txt
git commit -m "実験用コミット: テストファイル追加"

    コマンド実行結果: 

    [main c3d4e5f] 実験用コミット: テストファイル追加
 1 file changed, 1 insertion(+)
 create mode 100644 test_file.txt

    さらに実験用変更

    echo "追加内容" >> test_file.txt
git add test_file.txt
git commit -m "実験用コミット2: テストファイル更新"

    コマンド実行結果: 

    [main d4e5f6a] 実験用コミット2: テストファイル更新
 1 file changed, 1 insertion(+)

    現在の履歴確認

    git log --oneline

    コマンド実行結果: 

    d4e5f6a 実験用コミット2: テストファイル更新
c3d4e5f 実験用コミット: テストファイル追加

    3. --hard reset（最も強力・注意が必要）

    ⚠️ 重要な警告: --hardは変更を完全に削除します。事前にバックアップを確認してください。

    📘 git reset モードと安全な復元のワークフロー図

    Step 4: 安全な履歴操作のベストプラクティス

    🛡️ 安全性を重視した操作手順

    1. 作業前の確認習慣

    # 現在の状態確認
git status

# 履歴確認
git log --oneline

# 変更内容確認
git diff

    2. 重要な変更前のバックアップ

    # 重要な作業前にタグを作成
git tag backup-before-experiment

# または新しいブランチを作成
git branch backup-main

    3. 段階的なreset実践

    # まずは安全な--softから試す
git reset --soft HEAD~1

# 問題なければ--mixedを検討
git reset HEAD~1

# 最後の手段として--hardを使用
git reset --hard <commit-id>

    4. 復旧不可能な操作の回避

    安全な操作: - ✅ git checkout: 過去の確認のみ - ✅ git reset --soft: コミットメッセージの修正 - ✅ git reset --mixed: ステージングの調整

    注意が必要な操作: - ⚠️ git reset --hard: データの完全削除 - ⚠️ 強制push: リモートリポジトリの書き換え

    🔄 トラブル時の復旧方法

    reflogによる履歴復旧

    Gitは削除されたコミットも一定期間保持しています：

    git reflog
