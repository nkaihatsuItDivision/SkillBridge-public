    1. Fork機能：独立した開発環境の構築

    🔧 Forkとは何か

    Fork: GitHubで他人のリポジトリを自分のアカウントに独立してコピーする機能

    Forkの仕組み

    元のリポジトリ（upstream）
     ↓ Fork
自分のリポジトリ（origin）  ← ここで自由に開発
     ↓ Pull Request
元のリポジトリ（upstream）  ← 変更提案・統合

    なぜForkが必要？

    1. 権限管理: 元のリポジトリへの直接変更権限がない場合
    2. 安全性: 元のコードに影響を与えずに実験・開発
    3. 独立性: 自分のペースでの開発・テスト
    4. 協業: 複数開発者での効率的な協業

    🚀 Fork操作の実践

    Step 1: 自分のリポジトリをFork対象として準備

    public class Calculator {
    public static void main(String[] args) {
        System.out.println(&quot;Calculator v2.0 - 四則演算対応&quot;);
        System.out.println(&quot;=========================&quot;);

        int a = 20, b = 5;
        System.out.println(&quot;計算対象: &quot; + a + &quot;, &quot; + b);
        System.out.println();

        System.out.println(&quot;加算結果: &quot; + add(a, b));
        System.out.println(&quot;減算結果: &quot; + subtract(a, b));
        System.out.println(&quot;乗算結果: &quot; + multiply(a, b));
        System.out.println(&quot;除算結果: &quot; + divide(a, b));
    }

    /**
     * 加算処理
     * @param a 第一引数
     * @param b 第二引数
     * @return a + b
     */
    public static int add(int a, int b) {
        return a + b;
    }

    /**
     * 減算処理
     * @param a 第一引数
     * @param b 第二引数
     * @return a - b
     */
    public static int subtract(int a, int b) {
        return a - b;
    }

    /**
     * 乗算処理
     * @param a 第一引数
     * @param b 第二引数
     * @return a * b
     */
    public static int multiply(int a, int b) {
        return a * b;
    }

    /**
     * 除算処理（ゼロ除算チェック付き）
     * @param a 第一引数
     * @param b 第二引数
     * @return a / b
     */
    public static int divide(int a, int b) {
        if (b == 0) {
            System.out.println(&quot;エラー: ゼロで除算することはできません&quot;);
            return 0;
        }
        return a / b;
    }
}
