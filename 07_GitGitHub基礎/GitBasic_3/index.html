    レッスン6.2で理解したGitの基本概念

    前回のレッスンで、あなたは以下のGit概念を理解しました：

    Gitの基本的な仕組み

    • 分散型バージョン管理：各開発者が完全なリポジトリを持つ仕組み
    • リポジトリ：プロジェクトの変更履歴を保存する「データベース」
    • コミット：変更の「セーブポイント」として履歴に記録する操作
    • 履歴管理：時系列でのプロジェクト成長過程の追跡機能

    Git操作の3つのエリア

    • 作業ディレクトリ：実際にファイル編集を行う場所
    • ステージングエリア：コミット準備のためのファイル選択場所
    • リポジトリ：コミットされた変更履歴の保存場所

    🔧 Git環境の準備

    Git インストール状況の確認

    まず、お使いのコンピューターにGitがインストールされているか確認しましょう。

    Windows環境でのGit確認

    コマンドプロンプトまたはPowerShellを開いて、以下のコマンドを実行してください：

    git --version

    実行結果： 

    git version 2.41.0.windows.1

    上記のようにバージョン情報が表示されれば、Gitは正常にインストールされています。

    Gitがインストールされていない場合

    📘 Gitのインストールガイド

    以下の手順でGitをインストールしてください：

    1. Git公式サイトにアクセス：https://git-scm.com/
    2. Download for Windowsをクリック
    3. ダウンロードしたインストーラーを実行
    4. 基本的にデフォルト設定で「Next」を連続クリック
    5. インストール完了後、コマンドプロンプトでgit --versionを実行して確認

    Git初期設定（重要）

    Gitを使用する前に、ユーザー情報の設定が必要です。これは、コミットの作成者を記録するために使用されます。

    ユーザー名とメールアドレスの設定

    git config --global user.name &quot;あなたの名前&quot;
git config --global user.email &quot;あなたのメールアドレス&quot;

    設定例： 

    git config --global user.name &quot;Taro Student&quot;
git config --global user.email &quot;taro.student@example.com&quot;

    実行結果： 

    （特に表示されませんが、設定は正常に完了しています）

    設定内容の確認

    設定が正しく行われたか確認しましょう：

    git config --global --list

    実行結果： 

    user.name=Taro Student
user.email=taro.student@example.com
（その他の設定項目）

    日本語ファイル名の対応設定

    日本語のファイル名を正しく扱うための設定も行っておきましょう：

    git config --global core.quotepath false

    実行結果： 

    （特に表示されませんが、設定は正常に完了しています）

    💡 よくある質問

    Q1: コミットメッセージを間違えた場合はどうすれば良いですか？

    A: 直前のコミットメッセージは以下で修正できます：

    git commit --amend -m &quot;修正後のコミットメッセージ&quot;

    ただし、既に他の人と共有したコミットは修正を避けるのが安全です。

    Q2: git add で特定のファイルだけを追加したい場合は？

    A: ファイル名を個別に指定できます：

    git add src/main/Student.java
git add src/main/Person.java src/main/StudentManagementSystem.java

    Q3: 間違ってファイルをステージングした場合の取り消し方法は？

    A: 以下のコマンドでステージングを取り消せます：

    git reset HEAD ファイル名

    全てのステージングを取り消す場合： 

    git reset HEAD

    Q4: コミットした後でファイルを追加し忘れに気づいた場合は？

    A: 追加し忘れたファイルをgit addしてから：

    git add 追加し忘れたファイル
git commit --amend --no-edit

    --no-editにより、コミットメッセージはそのままで、ファイルだけ追加されます。

    Q5: .gitignore を後から作成した場合、既に追跡されているファイルは？

    A: 既に追跡されているファイルは、以下で追跡を停止できます：

    git rm --cached bin/
git commit -m &quot;削除: bin/ディレクトリの追跡を停止&quot;

    その後、.gitignoreの設定が有効になります。
