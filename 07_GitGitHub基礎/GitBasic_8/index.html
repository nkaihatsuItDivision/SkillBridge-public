    📋 1. Issue管理：プロジェクトの組織的管理

    Issueとは何か？

    Issue（イシュー） = プロジェクトでの課題・問題・要望を管理する仕組み

    Issueで管理する内容

    • バグ報告: 動作しない機能やエラーの報告
    • 機能要望: 新しく追加したい機能のリクエスト
    • 質問: 使い方やコードに関する質問
    • ドキュメント: マニュアルや説明の改善要求
    • リファクタリング: コード品質改善の提案

    🛠️ ステップ1: 初めてのIssue作成

    あなたのリポジトリでIssue機能を有効化

    1. GitHubリポジトリにアクセス

    2. ブラウザで https://github.com/ユーザー名/student-management-system を開く

    3. Issuesタブをクリック

    4. リポジトリページ上部の「Issues」タブを選択

    5. New issueボタンをクリック

    6. 緑色の「New issue」ボタンを押す

    バグ報告Issueの作成

    以下の内容で最初のIssueを作成してみましょう：

    Title（タイトル）: 

    [BUG] 学生削除時の確認メッセージが表示されない

    Comment（本文）: 

    ## 🐛 バグの概要
学生データを削除する際に、確認メッセージが表示されずに即座に削除されてしまいます。

## 📋 再現手順
1. 学生管理システムを起動
2. メニューで「4. 学生を削除」を選択
3. 学生IDを入力
4. 確認なしで即座に削除される

## 🎯 期待される動作
削除前に「本当に削除しますか？(y/n)」のような確認メッセージを表示

## 🔧 提案する修正
StudentManager.javaの削除メソッドに確認処理を追加

## 📝 備考
- 重要なデータを誤って削除する危険性があります
- ユーザビリティの向上が期待できます

    Labels（ラベル）: - bug（バグ報告用） - priority: high（高優先度）

    Assignees（担当者）: - 自分自身をアサイン

    1. Create issueボタンをクリック

    📘 GitHubのIssue作成フォーム

    実行結果： 

    Issue #1 が作成されました！
- タイトル: [BUG] 学生削除時の確認メッセージが表示されない
- 番号: #1（自動採番）
- ステータス: Open
- 作成者: あなた
- 担当者: あなた

    🛠️ ステップ2: 機能要望Issueの作成

    続けて、機能要望のIssueも作成してみましょう：

    Title: 

    [FEATURE] 学生データのCSV出力機能を追加

    Comment: 

    ## 💡 機能の概要
学生データをCSVファイルに出力する機能を追加したいです。

## 🎯 実現したいこと
- 全学生データをCSV形式でエクスポート
- ファイル名は自動で日付を含む（例：students_2025-08-06.csv）
- ID、名前、年齢、メールアドレスを出力

## 🔧 実装予定箇所
- StudentManager.javaに新しいメソッド `exportToCSV()` を追加
- メニューに「6. CSVエクスポート」を追加

## 📝 備考
- データのバックアップとして活用
- 他システムとの連携時に有用

    Labels: - enhancement（機能拡張） - priority: medium（中優先度）

    🛠️ ステップ3: 質問Issueの作成

    Title: 

    [QUESTION] Week 5システムのテスト方法について

    Comment: 

    ## ❓ 質問内容
Week 5で作成した学生管理システムに、自動テストを追加したいのですが、どのような方法が推奨されますか？

## 🔍 知りたいこと
- JUnitテストの追加手順
- テストケースの設計方法
- カバレッジの計測方法

## 📚 参考にしたいこと
- ベストプラクティスのサンプルコード
- 推奨テストフレームワーク

## 🎯 最終目標
信頼性の高いシステムにするためのテスト基盤整備

    Labels: - question（質問） - help wanted（ヘルプ求む）

    📊 Issue管理のベストプラクティス

    効果的なタイトルの付け方

    ✅ 良い例:
[BUG] ログイン時にNullPointerExceptionが発生
[FEATURE] ダークモード対応
[DOCS] README.mdにセットアップ手順を追加

❌ 悪い例:
バグがある
新機能追加
ドキュメント

    ラベルの活用方法

    • 種類別: bug, enhancement, question, documentation
    • 優先度別: priority: high, priority: medium, priority: low
    • 難易度別: good first issue, help wanted
    • 状態別: in progress, needs review

    📋 2. GitHub Projects：かんばんボードでの進捗管理

    GitHub Projectsとは？

    GitHub Projects = Issue・Pull Requestを視覚的に管理するかんばんボード機能

    かんばんボードの基本構造

    📋 To Do → 🚧 In Progress → 👀 Review → ✅ Done
（予定）   （作業中）      （レビュー中） （完了）

    🛠️ ステップ4: 初めてのProjectボード作成

    新しいProjectの作成

    1. リポジトリのProjectsタブをクリック

    2. リポジトリページの「Projects」タブを選択

    3. New projectボタンをクリック

    4. 「New project」（新しいプロジェクト）を選択

    5. プロジェクト設定

    6. Project name: 学生管理システム開発進捗
    7. Description: Week 5システムの改善・機能追加の進捗管理

    8. Template: Basic kanban（基本かんばん）を選択

    9. Create projectボタンをクリック

    カラム（列）の設定

    デフォルトで以下のカラムが作成されます： - To do: 予定・未着手のタスク - In progress: 作業中のタスク - Done: 完了したタスク

    🛠️ ステップ5: IssueをProjectボードに追加

    作成したIssueをボードに配置

    1. To doカラムの「+ Add cards」をクリック

    2. 先ほど作成した3つのIssueを選択

    3. [BUG] 学生削除時の確認メッセージが表示されない
    4. [FEATURE] 学生データのCSV出力機能を追加

    5. [QUESTION] Week 5システムのテスト方法について

    6. 「Add selected」をクリック

    優先度に応じた並び替え

    ドラッグ&ドロップで以下の順序に並び替え：

    To do カラム（上から順）: 1. [BUG] 学生削除時の確認メッセージが表示されない（最優先） 2. [FEATURE] 学生データのCSV出力機能を追加 3. [QUESTION] Week 5システムのテスト方法について

    🛠️ ステップ6: 作業進捗の管理体験

    作業開始の体験

    1. バグ修正カードを「In progress」に移動

    2. [BUG] カードをドラッグして「In progress」カラムに移動

    3. Issue詳細でコメント追加

    4. Issueページで進捗コメントを追加： 

      ## 🔧 作業開始

削除処理の確認メッセージ追加に着手しました。

## 🎯 概要

このプロジェクトは、Java学習プログラムWeek 5の集大成として開発された、
オブジェクト指向プログラミングの高度な概念を統合した学生管理システムです。

### ✨ 特徴
- **継承階層**: Person → Student の明確な継承関係
- **ポリモーフィズム**: 統一インターフェースによる異なる学生タイプの処理

    🏗️ OOP設計

    クラス設計図

    Person (抽象基底クラス)
    ├── Student (具象クラス)
    │   ├── RegularStudent (正規学生)
    │   ├── ExchangeStudent (交換留学生)
    │   └── AuditStudent (聴講生)
    │
    └── interfaces/
        ├── Reportable (レポート生成)
        ├── Validatable (バリデーション)
        └── Comparable&lt;Student&gt; (比較)

    🔐 4. ライセンス選択：オープンソースの法的基盤

    なぜライセンスが重要なのか？

    法的保護の重要性: - 著作権保護: あなたのコードの権利を明確化 - 利用条件明示: 他の人がどう使って良いかを規定 - 責任範囲: 何に対して責任を負うかを明確化 - 企業での利用: 商用利用の可否を明示

    ライセンスなしの危険性

    ❌ ライセンスなし = 他人は一切利用できない
  → 学習目的でも見ることすらできない可能性
  → 採用担当者もコードを確認できない
  → オープンソースコミュニティに貢献できない

    主要ライセンスの比較

    ライセンス 商用利用 修正・配布 ソース公開義務 おすすめ用途
    MIT ✅ ✅ ❌ 学習・ポートフォリオ
    Apache 2.0 ✅ ✅ ❌ 企業プロジェクト
    GPL v3 ✅ ✅ ✅ オープンソース促進
    BSD ✅ ✅ ❌ シンプル・自由
