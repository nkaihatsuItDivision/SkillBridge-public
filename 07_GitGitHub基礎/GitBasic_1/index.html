    😰 Week 5システムで発生する管理課題

    1. コードの変更履歴が不明

    あなたのWeek 5システム（600-700行）を想像してください：

    // StudentServiceImpl.java の一部
@Override
public void registerStudent(Student student) throws ValidationException {
    validateStudent(student);
    List&lt;Student&gt; students = studentRepository.findAll();

    // この部分を昨日修正したのですが...
    if (students.stream().anyMatch(s -&gt; s.getStudentId().equals(student.getStudentId()))) {
        throw new ValidationException(&quot;学生ID &quot; + student.getStudentId() + &quot; は既に登録されています&quot;);
    }

    studentRepository.save(student);
    log.info(&quot;学生が正常に登録されました: {}&quot;, student.getName());
}

    発生する問題: - 昨日の修正内容が思い出せない - なぜその修正をしたのか理由が不明 - 修正前のコードに戻したくても方法がない - バグが発生した原因を特定できない

    2. チーム開発での混乱

    あなたがWeek 5システムを完成させた後、同僚の田中さんと一緒に機能追加することになったとします：

    📘 チーム開発での混乱状況

    発生する典型的な問題:

    // あなたの修正（月曜日の夜）
public class StudentServiceImpl {
    public void updateGrade(String studentId, Subject subject, int score) {
        // 新しい成績更新ロジックを追加
        Student student = findStudentById(studentId);
        student.updateGrade(subject, score);
        // バリデーション強化
        if (score &lt; 0 || score &gt; 100) {
            throw new ValidationException(&quot;点数は0-100の範囲で入力してください&quot;);
        }
        studentRepository.save(student);
    }
}

    // 田中さんの修正（火曜日の朝、あなたの修正を知らない）
public class StudentServiceImpl {
    public void updateGrade(String studentId, Subject subject, int score) {
        // 同じメソッドを別の方法で修正
        if (score &gt; 100) {
            score = 100; // 自動補正機能を追加
        }
        Student student = findStudentById(studentId);
        student.setGrade(subject, score);
        studentRepository.save(student);
    }
}

    結果: どちらの修正を採用すべきか分からず、機能が重複・競合してしまいます。

    3. バックアップの限界

    多くの人が最初に考える解決策：

    プロジェクトフォルダ/
├── StudentManagementSystem_最新/
├── StudentManagementSystem_修正前/
├── StudentManagementSystem_20250805/
├── StudentManagementSystem_田中さん版/
├── StudentManagementSystem_最終版/
├── StudentManagementSystem_最終版_2/
└── StudentManagementSystem_最終版_本当に最終/

    問題点: - どれが本当に最新なのか分からない - フォルダサイズが膨大になる（600-700行×複数バージョン） - 変更内容の比較が手作業で非効率 - 複数人での同期が困難

    🔥 実際の開発現場で起こる深刻な問題

    ケース1: 緊急バグ修正

    状況: Week 5システムが本番稼働中、緊急でバグ修正が必要

    // 本番稼働中のコード（正常動作）
public ValidationResult validateStudent(Student student) {
    if (student.getName() == null || student.getName().trim().isEmpty()) {
        return ValidationResult.error(&quot;氏名は必須です&quot;);
    }
    // 他の検証処理...
    return ValidationResult.success();
}

// あなたが新機能開発中のコード（開発途中）
public ValidationResult validateStudent(Student student) {
    // 国際化対応の開発中（まだ未完成）
    if (student.getName() == null || isInvalidName(student.getName())) {
        return ValidationResult.error(getMessage(&quot;validation.name.required&quot;));
    }
    // 新しい検証ルールを追加中（テスト未完了）
    if (!isValidStudentFormat(student)) {
        return ValidationResult.error(&quot;学生情報の形式が正しくありません&quot;);
    }
    return ValidationResult.success();
}

    問題: - 本番の安定版コードがどれか分からない - 緊急修正を適用する基準が曖昧 - 新機能開発を巻き戻す方法がない

    ケース2: チーム拡大時の混乱

    状況: プロジェクトが成功し、開発チームが5人に拡大

    📘 5人チーム開発での情報共有の複雑さ

    発生する問題: - 全員の修正を統合する方法がない - 誰がどの部分を担当しているか不明 - 修正内容の重複や競合が頻発 - 品質管理が困難（誰がレビューするのか？）

    💡 バージョン管理システムによる解決

    バージョン管理システムとは

    バージョン管理システムは、これらの課題を根本的に解決するツールです：

    1. 完全な変更履歴管理

    # 例：Gitでの履歴表示
commit a1b2c3d (2025-08-05 14:30)
Author: あなた &lt;you@company.com&gt;
Message: InternationalStudent クラスの国籍判定ロジックを修正

commit e4f5g6h (2025-08-05 10:15)
Author: 田中さん &lt;tanaka@company.com&gt;  
Message: 成績計算における小数点処理のバグを修正

commit i7j8k9l (2025-08-04 16:45)
Author: あなた &lt;you@company.com&gt;
Message: Lombok @Builder パターンを Student クラスに適用

    2. 安全な協業機能

    • ブランチ: 各開発者が独立して作業
    • マージ: 安全に変更を統合
    • プルリクエスト: コードレビューとディスカッション

    3. 確実な復元機能

    • 任意の時点に完全復元可能
    • 特定の変更だけを取り消し可能
    • 実験的な修正も安心して実施

    🎯 Week 6で習得する技術

    Git: 分散バージョン管理システム

    • 世界標準: 99%以上の開発現場で使用
    • 分散型: 各開発者が完全な履歴を保持
    • 高性能: 大規模プロジェクトでも高速動作

    GitHub: Git のクラウドサービス

    • コード共有: 世界中の開発者との協業
    • プロジェクト管理: Issue、プロジェクトボード
    • 自動化: CI/CD、自動テスト

    Week 6での実践内容

    Week 5で完成させたOOP応用版学生管理システムを使って：

    1. Git管理下に配置して履歴管理を開始
    2. 新機能追加（InternationalStudent等）をブランチで開発
    3. GitHubでリモート管理とチーム開発体験
    4. プルリクエストでコードレビュー実践

    💡 よくある質問

    Q1: 「個人開発なら必要ないのでは？」

    A: 個人開発でも非常に有効です：

    // 例：実験的な機能追加
// 「もし失敗しても元に戻せる」安心感で大胆な挑戦が可能

// 実験前
public class StudentService {
    public List&lt;Student&gt; searchStudents(String keyword) {
        // 基本的な検索実装
        return students.stream()
            .filter(s -&gt; s.getName().contains(keyword))
            .collect(Collectors.toList());
    }
}

// 実験的改良（Gitがあるので安心してトライ）
public class StudentService {
    public List&lt;Student&gt; searchStudents(String keyword) {
        // 高度な検索機能（曖昧検索・重み付け等）を実験
        return fuzzySearchWithScoring(keyword);
    }
}

    Q3: 「Week 5システムが完璧なので変更したくない」

    A: Gitは変更を安全に管理するツールです：

    • 現在の完璧なコードは保護されます
    • すべての変更が履歴として記録されます
    • いつでも完璧な状態に復元可能です
    • むしろ実験や改良が安全になります
