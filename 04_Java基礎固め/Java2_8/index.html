    1. なぜメソッドオーバーロードが必要なのか？

    🛠️ ステップ1: レッスン3.6の復習から始めよう

    レッスン3.6で、こんな文字列検索メソッドを作りましたね：

    public static boolean searchText(String text, String keyword) {
    if (text == null || keyword == null) {
        return false;
    }
    return text.contains(keyword);
}

    これは基本的な機能として素晴らしいのですが、実際に使っていると不便を感じることがありませんか？

    日常での使用パターンの多様性

    現実的な検索ニーズ: 

    // ケース1: 基本検索
searchText("Hello World", "World");  // true

// ケース2: 大文字小文字を無視したい
searchText("Hello World", "world");  // false（残念...）

// ケース3: 複数キーワードのAND検索をしたい
searchText("Java Spring Boot", "Java", "Spring");  // できない...

// ケース4: 検索開始位置を指定したい
searchText("HelloHelloHello", "Hello", 5);  // できない...

    現在の解決方法（不便）: 

    // 名前を変えて別々のメソッドを作る必要がある
public static boolean searchText(String text, String keyword) { ... }
public static boolean searchTextIgnoreCase(String text, String keyword) { ... }
public static boolean searchTextMultiple(String text, String... keywords) { ... }
public static boolean searchTextFromIndex(String text, String keyword, int startIndex) { ... }

    この方法の問題点: - メソッド名を覚えるのが大変 - どのメソッドを使えばいいか迷う - 一貫性がなくて美しくない

    📘 メソッドのオーバーロード比較

    2. メソッドオーバーロードの基本概念

    🛠️ ステップ2: オーバーロードの魔法

    メソッドオーバーロードとは、同じ名前で異なる引数を持つメソッドを複数定義することです。

    // 同じ名前「searchText」で、異なる引数パターンを定義
public static boolean searchText(String text, String keyword) {
    // 基本検索
}

public static boolean searchText(String text, String keyword, boolean ignoreCase) {
    // 大文字小文字を考慮するかの選択
}

public static boolean searchText(String text, String keyword, int startIndex) {
    // 検索開始位置の指定
}

public static boolean searchText(String text, String... keywords) {
    // 複数キーワードのAND検索
}

    オーバーロードの利点: 1. 統一されたインターフェース: 全てsearchTextで呼び出せる 2. 使いやすさの向上: 利用者が覚えるのは1つの名前だけ 3. 段階的な機能提供: 簡単→高度な使い方を選択可能 4. コードの美しさ: 一貫性のあるAPIデザイン

    オーバーロード解決の仕組み

    📘 オーバーロード解決の仕組み

    Javaコンパイラは、引数の型・数・順序を見て、適切なメソッドを自動選択します：

    searchText("Hello", "World");                    // 引数2個 → 基本版を呼び出し
searchText("Hello", "World", true);              // 引数3個(String,String,boolean) → 大文字小文字無視版
searchText("Hello", "World", 5);                 // 引数3個(String,String,int) → 開始位置指定版
searchText("Hello", "Java", "Spring", "Boot");   // 可変長引数 → 複数キーワード版

    この自動選択により、利用者は何も考えずに直感的に使えます！

    public class StringUtils {

    // 1. 基本的な文字列検索
    public static boolean search(String text, String keyword) {
        if (text == null || keyword == null) {
            return false;
        }
        return text.contains(keyword);
    }

    // 2. 大文字小文字を無視する検索
    public static boolean search(String text, String keyword, boolean ignoreCase) {
        if (text == null || keyword == null) {
            return false;
        }

        if (ignoreCase) {
            return text.toLowerCase().contains(keyword.toLowerCase());
        } else {
            return text.contains(keyword);
        }
    }

    // 3. 検索開始位置を指定する検索
    public static boolean search(String text, String keyword, int startIndex) {
        if (text == null || keyword == null || startIndex < 0) {
            return false;
        }

        if (startIndex >= text.length()) {
            return false;
        }

        return text.substring(startIndex).contains(keyword);
    }

    // 4. 複数キーワードのAND検索
    public static boolean search(String text, String... keywords) {
        if (text == null || keywords == null || keywords.length == 0) {
            return false;
        }

        for (String keyword : keywords) {
            if (!search(text, keyword)) {  // 基本版を再利用
                return false;
            }
        }
        return true;
    }

    // テスト用mainメソッド
    public static void main(String[] args) {
        String text = "Java Spring Boot Framework Development";

        System.out.println("=== StringUtils検索テスト ===");

        // 基本検索
        System.out.println("基本検索:");
        System.out.println("  'Java' → " + search(text, "Java"));
        System.out.println("  'java' → " + search(text, "java"));

        // 大文字小文字無視検索
        System.out.println("\n大文字小文字無視検索:");
        System.out.println("  'java'(無視) → " + search(text, "java", true));
        System.out.println("  'java'(厳密) → " + search(text, "java", false));

        // 開始位置指定検索
        System.out.println("\n開始位置指定検索:");
        System.out.println("  'a'(位置0から) → " + search(text, "a", 0));
        System.out.println("  'a'(位置10から) → " + search(text, "a", 10));

        // 複数キーワード検索
        System.out.println("\n複数キーワード検索:");
        System.out.println("  'Java','Spring' → " + search(text, "Java", "Spring"));
        System.out.println("  'Java','Python' → " + search(text, "Java", "Python"));
    }
}

public static String format(String text, int width, char paddingChar) {
    if (text == null) {
        text = "";
    }

    text = text.trim();

    if (text.length() >= width) {
        return text;
    }

    StringBuilder result = new StringBuilder(text);
    while (result.length() < width) {
        result.append(paddingChar);
    }
    return result.toString();
}

// 4. 複数の整形オプション
public static String format(String text, boolean upperCase, boolean removeSpaces, int maxLength) {
    if (text == null) {
        return "";
    }

    String result = text.trim();

    if (upperCase) {
        result = result.toUpperCase();
    }

    if (removeSpaces) {
        result = result.replace(" ", "");
    }

    if (maxLength > 0 && result.length() > maxLength) {
        result = result.substring(0, maxLength);
    }

    return result;
}

    // 1. 基本的なCSV分割
public static String[] parseCSV(String csvLine) {
    if (csvLine == null || csvLine.trim().isEmpty()) {
        return new String[0];
    }
    return csvLine.split(",");
}

// 2. カスタム区切り文字でのCSV分割
public static String[] parseCSV(String csvLine, String delimiter) {
    if (csvLine == null || csvLine.trim().isEmpty()) {
        return new String[0];
    }

    if (delimiter == null || delimiter.isEmpty()) {
        delimiter = ",";
    }

    return csvLine.split(delimiter);
}

// 3. 引用符を考慮した安全なCSV分割（レッスン3.7の技術を活用）
public static String[] parseCSV(String csvLine, String delimiter, boolean handleQuotes) {
    if (!handleQuotes) {
        return parseCSV(csvLine, delimiter);  // シンプル版を呼び出し
    }

    if (csvLine == null || csvLine.trim().isEmpty()) {
        return new String[0];
    }

    if (delimiter == null || delimiter.isEmpty()) {
        delimiter = ",";
    }

    // 引用符を考慮した分割（レッスン3.7の技術を使用）
    ArrayList<String> result = new ArrayList<>();
    StringBuilder current = new StringBuilder();
    boolean inQuotes = false;

    for (int i = 0; i < csvLine.length(); i++) {
        char c = csvLine.charAt(i);

        if (c == '"') {
            inQuotes = !inQuotes;
        } else if (c == delimiter.charAt(0) && !inQuotes) {
            result.add(current.toString().trim());
            current = new StringBuilder();
        } else {
            current.append(c);
        }
    }

    result.add(current.toString().trim());
    return result.toArray(new String[0]);
}

// 4. 完全機能版：空値の処理も含む
public static String[] parseCSV(String csvLine, String delimiter, boolean handleQuotes, String emptyValue) {
    String[] parts = parseCSV(csvLine, delimiter, handleQuotes);

    for (int i = 0; i < parts.length; i++) {
        if (parts[i].isEmpty()) {
            parts[i] = emptyValue;
        }
    }

    return parts;
}

    Q1: オーバーロードと似たような名前のメソッドを複数作る違いは？

    A: コンパイルエラーまたは予期しない動作になります：

    Q3: オーバーロードできない場合はありますか？

    A: 引数の型と数が完全に同じ場合はオーバーロードできません：

    // ❌ これは不可能（引数の型と数が同じ）

// ✅ これは可能（引数の型が違う）

public static String process(String text, boolean option) { ... }  // OK
