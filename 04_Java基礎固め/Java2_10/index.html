    🐛 デバッグとは何か？

    デバッグの基本概念

    デバッグ（Debug） = 「虫（Bug）を取り除く」という意味

    プログラムの「バグ」とは： - 期待した動作をしない部分 - エラーが発生する箇所 - 予期しない結果を出力する処理 - 処理が途中で止まってしまう原因

    なぜデバッグが重要なのか？

    実務での現実: - 新規開発: 書いたコードの約20-30%にはバグが含まれる - 保守・改修: 既存システムの理解にデバッグ技術が必須 - チーム開発: 他人が書いたコードの調査・修正 - 効率性: デバッグスキルがあると開発速度が2-3倍向上

    プログラマーの時間配分（実態）: - 新機能開発: 30% - バグ修正・デバッグ: 40% - 既存コード理解・調査: 20% - テスト・確認: 10%

    つまり、デバッグスキルは実務で最も重要な技術の一つです！

    🔧 Eclipse デバッグ機能の全体像

    デバッグで使用する主要機能

    1. ブレークポイント: 指定した行でプログラムを一時停止
    2. ステップ実行: 1行ずつプログラムを実行して動作を追跡
    3. 変数監視: 変数の値をリアルタイムで確認
    4. 呼び出し履歴: メソッドの呼び出し順序を確認
    5. 式の評価: デバッグ中に式の値を計算・確認

    デバッグの基本的な流れ

    1. バグの症状を確認 → 何がおかしいのか把握
2. 問題箇所を推測 → どこにバグがありそうか仮説立て
3. ブレークポイント設定 → 調査したい箇所で実行を停止
4. デバッグ実行 → 実際にプログラムをデバッグモードで動かす
5. ステップ実行 → 1行ずつ動作を確認
6. 変数の値を確認 → 期待値と実際の値を比較
7. 原因特定・修正 → バグの原因を見つけて修正
8. 動作確認 → 修正後のテスト実行

    📘 デバッグ画面のレイアウト

    🎯 実践1: ブレークポイントの基本操作

    バグを含むサンプルプログラム

    まずは、意図的にバグを含むプログラムを作成してデバッグを体験しましょう。

    ファイル名: StudentGradeProcessor.java

    import java.util.*;
import java.io.*;

public class StudentGradeProcessor {

    public static void main(String[] args) {
        System.out.println("=== 学生成績処理プログラム ===");

        // 学生データの作成
        ArrayList<String> students = createStudentData();

        // 成績の計算と表示
        processGrades(students);

        // ファイル出力
        saveToFile(students);

        System.out.println("処理が完了しました。");
    }

    // 学生データを作成するメソッド（バグが含まれています）
    public static ArrayList<String> createStudentData() {
        ArrayList<String> students = new ArrayList<>();

        students.add("田中太郎,85,92,78");
        students.add("佐藤花子,90,88,95");
        students.add("山田次郎,75,80,82");
        students.add(""); // 空のデータ（バグの原因1）
        students.add("鈴木三郎,95,87,91");

        return students;
    }

    public static String cleanText(String text, boolean removeBlanks, boolean trimSpaces) {
        if (text == null) {
            return "";
        }

        String result = text;

        if (trimSpaces) {
            result = result.trim();
        }

        if (removeBlanks) {
            result = result.replaceAll("\\s+", " "); // 連続スペースを1つに
        }

        // バグ5: 空文字チェックが不適切
        if (result.length() == 0) {
            return null; // nullを返すべきではない
        }

        return result;
    }
