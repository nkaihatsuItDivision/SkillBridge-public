    🔍 配列の仕組みを詳しく理解しよう

    配列ってそもそも何？

    配列の定義 配列とは、同じデータ型の値を連続したメモリ領域に格納するデータ構造です。

    📘 配列のメモリ図

    [メモリ上での配列のイメージ図]
int[] numbers = {10, 20, 30, 40, 50};

メモリアドレス: 1000  1004  1008  1012  1016
値:           [10]  [20]  [30]  [40]  [50]
インデックス:    0     1     2     3     4

    配列の特徴

    1. 連続メモリ配置: 要素が隣り合ったメモリ位置に配置
    2. 固定サイズ: 作成時にサイズが決まり、後から変更不可
    3. インデックスアクセス: 番号（インデックス）で直接要素にアクセス
    4. 高速アクセス: インデックスがあれば瞬時に要素にアクセス可能

    なぜ配列は高速なのか？

    int[] numbers = {10, 20, 30, 40, 50};
int value = numbers[3];  // なぜこれが高速？

    計算式: 配列の開始アドレス + (インデックス × データサイズ) - numbers[3] = 1000番地 + (3 × 4バイト) = 1012番地 - 計算一回で直接アクセス！

    実習1: 配列の基本操作確認

    Eclipseで新しいクラス ArrayMasterClass を作成してください。

    public class ArrayMasterClass {
    public static void main(String[] args) {
        System.out.println("=== 配列の基本操作マスター ===\n");

        // 1. 配列の宣言と初期化の3パターン
        System.out.println("【1. 配列の宣言と初期化】");

        // パターン1: 宣言と同時に値を設定
        int[] pattern1 = {10, 20, 30, 40, 50};
        System.out.println("パターン1（値指定）: " + java.util.Arrays.toString(pattern1));

        // パターン2: サイズ指定で宣言、後で値を設定
        int[] pattern2 = new int[5];
        pattern2[0] = 100;
        pattern2[1] = 200;
        pattern2[2] = 300;
        pattern2[3] = 400;
        pattern2[4] = 500;
        System.out.println("パターン2（サイズ指定）: " + java.util.Arrays.toString(pattern2));

        // パターン3: 宣言と初期化を分ける
        int[] pattern3;
        pattern3 = new int[]{1, 2, 3, 4, 5};
        System.out.println("パターン3（分割宣言）: " + java.util.Arrays.toString(pattern3));

        System.out.println();
    }
}

    実行して確認 各パターンの違いと使い分けを確認してください。

    実習2: 配列のアクセスと走査

    上記のコードに以下を追加してください：

            // 2. 個別要素へのアクセス
        System.out.println("【2. 個別要素へのアクセス】");
        int[] numbers = {10, 20, 30, 40, 50};

        System.out.println("最初の要素: numbers[0] = " + numbers[0]);
        System.out.println("3番目の要素: numbers[2] = " + numbers[2]);
        System.out.println("最後の要素: numbers[" + (numbers.length-1) + "] = " + numbers[numbers.length-1]);
        System.out.println("配列のサイズ: numbers.length = " + numbers.length);
        System.out.println();

        // 3. for文での配列走査
        System.out.println("【3. for文での配列走査】");
        System.out.print("通常のfor文: ");
        for (int i = 0; i < numbers.length; i++) {
            System.out.print(numbers[i] + " ");
        }
        System.out.println();

        System.out.print("拡張for文: ");
        for (int value : numbers) {
            System.out.print(value + " ");
        }
        System.out.println("\n");

    実習3: 2次元配列の操作

    さらに以下を追加してください：

            // 4. 2次元配列の操作
        System.out.println("【4. 2次元配列の操作】");

        // 学生3人、科目3つの成績表
        int[][] scores = {
            {85, 92, 78},  // 田中さんの成績
            {90, 88, 95},  // 佐藤さんの成績  
            {76, 85, 89}   // 山田さんの成績
        };
        String[] students = {"田中", "佐藤", "山田"};
        String[] subjects = {"数学", "英語", "国語"};

        // 成績表の表示
        System.out.println("=== 成績表 ===");
        System.out.print("　　　");
        for (String subject : subjects) {
            System.out.print(subject + "\t");
        }
        System.out.println();

        for (int i = 0; i < students.length; i++) {
            System.out.print(students[i] + "\t");
            for (int j = 0; j < subjects.length; j++) {
                System.out.print(scores[i][j] + "\t");
            }
            System.out.println();
        }

    演習1: 動的なデータ管理の困難さ

    新しいクラス DynamicDataChallenge を作成してください：

    import java.util.Scanner;

public class DynamicDataChallenge {
    public static void main(String[] args) {
        Scanner scanner = new Scanner(System.in);

        System.out.println("=== ToDoリスト管理（配列版）===");
        System.out.println("配列の制限を体験してみましょう\n");

        // 最大10個のToDoを管理する配列
        String[] todos = new String[10];
        int todoCount = 0;  // 現在のToDo数を管理

        while (true) {
            System.out.println("\n--- ToDoリスト ---");
            if (todoCount == 0) {
                System.out.println("（まだToDoがありません）");
            } else {
                for (int i = 0; i < todoCount; i++) {
                    System.out.println((i + 1) + ". " + todos[i]);
                }
            }

            System.out.println("\n1. ToDo追加");
            System.out.println("2. ToDo削除");
            System.out.println("3. 終了");
            System.out.print("選択してください: ");

            int choice = scanner.nextInt();
            scanner.nextLine();  // 改行を消費

            switch (choice) {
                case 1:
                    // ToDo追加
                    if (todoCount >= todos.length) {
                        System.out.println("⚠️ エラー: これ以上ToDoを追加できません！");
                        System.out.println("配列のサイズが " + todos.length + " で固定されているためです。");
                    } else {
                        System.out.print("新しいToDoを入力してください: ");
                        String newTodo = scanner.nextLine();
                        todos[todoCount] = newTodo;
                        todoCount++;
                        System.out.println("✅ ToDoを追加しました！");
                    }
                    break;

                case 2:
                    // ToDo削除
                    if (todoCount == 0) {
                        System.out.println("削除するToDoがありません。");
                    } else {
                        System.out.print("削除するToDoの番号を入力してください: ");
                        int deleteIndex = scanner.nextInt() - 1;

                        if (deleteIndex < 0 || deleteIndex >= todoCount) {
                            System.out.println("⚠️ 無効な番号です。");
                        } else {
                            // 削除処理（配列では複雑！）
                            System.out.println("削除中...");

                            // 削除する要素以降を前にずらす
                            for (int i = deleteIndex; i < todoCount - 1; i++) {
                                todos[i] = todos[i + 1];
                            }

                            todos[todoCount - 1] = null;  // 最後の要素をクリア
                            todoCount--;

                            System.out.println("✅ ToDoを削除しました！");
                            System.out.println("💭 配列での削除は要素をずらす処理が必要で複雑ですね...");
                        }
                    }
                    break;

                case 3:
                    System.out.println("プログラムを終了します。");
                    scanner.close();
                    return;

                default:
                    System.out.println("1-3の番号を選択してください。");
            }
        }
    }
}
