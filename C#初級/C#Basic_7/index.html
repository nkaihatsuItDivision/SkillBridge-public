    メソッドでコードを整理・再利用

    text メソッドの概念と作成方法を学習する。コードの重複を避け、プログラムを構造化して保守性を高める基礎を身につける。 レッスン7: メソッドでコードを整理・再利用 # レッスン7: メソッドでコードを整理・再利用

    学習目標

    • メソッドの概念と必要性を理解する
    • メソッドの定義と呼び出し方法を学ぶ
    • 引数と戻り値を使ったメソッドを作成できる
    • メソッドを使ってプログラムを構造化できる

    メソッドとは

    メソッドとは、特定の処理を一つの名前でまとめた「命令」です。 料理のレシピや作業手順書のように、「○○をする方法」を定義したものです。

    身近な例で理解する

    • 料理のレシピ：「カレーを作る」という手順をまとめたもの
    • 家電のボタン：「録画する」ボタンを押すと決まった処理が実行される
    • 計算機の機能：「√」ボタンを押すと平方根を計算する

    プログラムでのメソッド

    同じように、プログラムでも「○○を計算する」、「○○を表示する」という 処理手順に名前を付けて、いつでも呼び出せるようにしたものがメソッドです。

    なぜメソッドが必要なのか

    1. コードの重複を避ける（DRY原則）

    同じ処理を何度も書く必要がなくなります。

    2. 保守性の向上

    修正が必要な時、メソッド1箇所を変更するだけで済みます。
    メソッドなしの場合：10箇所の同じコードを全部修正
    メソッドありの場合：1箇所のメソッドを修正するだけ

    3. 可読性の向上

    複雑な処理を意味のある名前で表現できます。 

    // 何をしているか分からない
double result = a * a + b * b;
// 何をしているか明確
double result = CalculateHypotenuse(a, b);

    4. テストしやすさ

    個別の機能をメソッド単位でテストできます。

    5. チーム開発での分担

    メソッド単位で作業を分担できます。

    メソッドを使わない場合の問題

    // 同じような計算を何度も書く必要がある
int area1 = 10 * 5;
Console.WriteLine(&quot;面積1: &quot; + area1);

int area2 = 8 * 6;
Console.WriteLine(&quot;面積2: &quot; + area2);

int area3 = 12 * 4;
Console.WriteLine(&quot;面積3: &quot; + area3);

    メソッドを使った場合

    // 一度メソッドを定義すれば、何度でも使える
static int CalculateArea(int width, int height)
{
    return width * height;
}

// メソッドを呼び出して使用
int area1 = CalculateArea(10, 5);
int area2 = CalculateArea(8, 6);
int area3 = CalculateArea(12, 4);

    メソッドの基本構文

    static 戻り値の型 メソッド名(引数の型 引数名)
{
    // 処理内容
    return 戻り値; // 戻り値がある場合
}

    staticキーワードとは

    現在の学習段階では「メソッドを作る時に必要なおまじない」として理解してください。

    なぜstaticが必要なのか

    • Main メソッドも static で作られている
    • static メソッドは static メソッドからしか呼び出せない
    • 後のレッスンで学ぶ「クラス」と関連する概念

    現時点での理解

    static void メソッド名()  // ← このstaticは必須
{
    // 処理
}
    詳しい内容は別のレッスンで学習します

    引数とは

    メソッドに「材料」を渡すための仕組みです。

    身近な例

    • 自動販売機：お金（引数）を入れると、飲み物（戻り値）が出てくる
    • 計算機：数値（引数）を入力すると、計算結果（戻り値）が表示される
    • レシピ：材料（引数）を渡すと、料理（戻り値）ができる

    プログラムでの引数

    static int Add(int a, int b)  // a, bが引数（材料）
{
    return a + b;             // 戻り値（結果）
}

int result = Add(10, 20);     // 10, 20を引数として渡す

    戻り値とは

    メソッドが処理した「結果」を呼び出し元に返す仕組みです。

    戻り値がある場合（return文必須）

    static int Calculate()
{
    return 100;  // 必ず値を返す
}

    戻り値ない場合（void）

    static void Display()
{
    Console.WriteLine(&quot;表示するだけ&quot;);
    // returnは不要（何も返さない）
}

    メソッドの実行の流れ

    実行順序の理解

    static void Main(string[] args)
{
    Console.WriteLine(&quot;1. メイン開始&quot;);
    Greet(&quot;太郎&quot;);                    // ← ここでGreetメソッドを呼び出し
    Console.WriteLine(&quot;4. メイン終了&quot;);
}

static void Greet(string name)
{
    Console.WriteLine(&quot;2. Greetメソッド開始&quot;);
    Console.WriteLine(&quot;こんにちは、&quot; + name + &quot;さん&quot;);
    Console.WriteLine(&quot;3. Greetメソッド終了&quot;);
    // ここで呼び出し元（Main）に戻る
}

    実行結果

    1. メイン開始
    2. Greetメソッド開始
    こんにちは、太郎さん
    3. Greetメソッド終了
    4. メイン終了

    重要なポイント

    • メソッドが呼ばれると、そのメソッドの処理が完全に終わってから元に戻る
    • 戻り値がある場合は、return文で値を返してから元に戻る

    値がコピーされる

    static void Main(string[] args)
{
    int originalNumber = 10;
    ChangeNumber(originalNumber);
    Console.WriteLine(originalNumber);  // まだ10のまま
}

static void ChangeNumber(int number)
{
    number = 999;  // コピーされた値を変更しているだけ
    Console.WriteLine(&quot;メソッド内: &quot; + number);  // 999
}

    実行結果

    メソッド内:999
    10

    なぜこうなるか

    引数として渡される値は「コピー」です。元の変数は変更されません。
    これを「値渡し」と呼びます。

    変数のスコープ（有効範囲）

    メソッド内の変数

    メソッド内で宣言した変数は、そのメソッド内でのみ使用できます。

    static void Method1()
{
    int localVar = 100;  // Method1内でのみ使用可能
    Console.WriteLine(localVar);  // OK
}

static void Method2()
{
    Console.WriteLine(localVar);  // エラー！Method1のlocalVarは使えない
}

    引数も同様

    static void Calculate(int number)  // numberはCalculate内でのみ使用可能
{
    int result = number * 2;       // resultもCalculate内でのみ使用可能
}
// ここではnumberもresultも使えない

    例題2：配列処理をメソッドで整理

    まずは下記のコードを手作業で書き写して流れを理解しながら出力を確認しましょう

    using System;

namespace ArrayProcessing
{
    class Program
    {
        static void Main(string[] args)
        {
            Console.WriteLine(&quot;=== 配列処理プログラム ===&quot;);

            int[] numbers = {85, 92, 78, 95, 88, 76, 89, 94, 82, 87};

            DisplayArray(&quot;元の配列&quot;, numbers);

            double average = CalculateAverage(numbers);
            Console.WriteLine(&quot;平均値: &quot; + average.ToString(&quot;F1&quot;));

            int max = FindMaximum(numbers);
            Console.WriteLine(&quot;最大値: &quot; + max);

            int min = FindMinimum(numbers);
            Console.WriteLine(&quot;最小値: &quot; + min);

            int[] sortedNumbers = SortArray(numbers);
            DisplayArray(&quot;ソート後&quot;, sortedNumbers);

            int searchValue = 85;
            bool found = SearchValue(numbers, searchValue);
            Console.WriteLine(searchValue + &quot;が配列に含まれているか: &quot; + found);
        }

        static void DisplayArray(string title, int[] array)
        {
            Console.WriteLine(&quot;&quot;);
            Console.WriteLine(title + &quot;:&quot;);
            foreach (int number in array)
            {
                Console.Write(number + &quot; &quot;);
            }
            Console.WriteLine();
        }

        static double CalculateAverage(int[] array)
        {
            int total = 0;
            foreach (int number in array)
            {
                total += number;
            }
            return (double)total / array.Length;
        }

        static int FindMaximum(int[] array)
        {
            int max = array[0];
            for (int i = 1; i &lt; array.Length; i++)
            {
                if (array[i] &gt; max)
                {
                    max = array[i];
                }
            }
            return max;
        }

        static int FindMinimum(int[] array)
        {
            int min = array[0];
            for (int i = 1; i &lt; array.Length; i++)
            {
                if (array[i] &lt; min)
                {
                    min = array[i];
                }
            }
            return min;
        }

        static int[] SortArray(int[] array)
        {
            // 元の配列をコピーして、元の配列を変更しないようにする
            int[] sortedArray = new int[array.Length];
            for (int i = 0; i &lt; array.Length; i++)
            {
                sortedArray[i] = array[i];
            }

            // バブルソート
            for (int i = 0; i &lt; sortedArray.Length - 1; i++)
            {
                for (int j = 0; j &lt; sortedArray.Length - 1 - i; j++)
                {
                    if (sortedArray[j] &gt; sortedArray[j + 1])
                    {
                        int temp = sortedArray[j];
                        sortedArray[j] = sortedArray[j + 1];
                        sortedArray[j + 1] = temp;
                    }
                }
            }

            return sortedArray;
        }

        static bool SearchValue(int[] array, int searchValue)
        {
            foreach (int number in array)
            {
                if (number == searchValue)
                {
                    return true;
                }
            }
            return false;
        }
    }
}

    メソッドのオーバーロード

    メソッドオーバーロードとは

    同じ名前でも「引数の種類や個数が違えば別のメソッド」として扱われる仕組みです。

    なぜオーバーロードが便利なのか

    用途は同じだが、受け取るデータの種類が違う場合に、同じ名前を使えます。

    オーバーロードの条件

    • 引数の個数が違う
    • 引数の型が違う
    • 引数の順序が違う
    static int Add(int a, int b)
{
    return a + b;
}

static double Add(double a, double b)
{
    return a + b;
}

static int Add(int a, int b, int c)
{
    return a + b + c;
}

// 使用例
int result1 = Add(10, 20);        // int版が呼ばれる
double result2 = Add(10.5, 20.3); // double版が呼ばれる
int result3 = Add(10, 20, 30);    // 3つの引数版が呼ばれる

    メソッドの設計指針

    1. 単一責任の原則

    一つのメソッドは一つの責任だけを持つべきです。

    // 悪い例：複数の責任を持つメソッド
static void ProcessStudentDataBad(string name, int score)
{
    // 入力検証
    if (string.IsNullOrEmpty(name)) return;
    if (score &lt; 0 || score &gt; 100) return;

    // データ処理
    string grade = score &gt;= 60 ? &quot;合格&quot; : &quot;不合格&quot;;

    // 表示
    Console.WriteLine(name + &quot;: &quot; + score + &quot;点 (&quot; + grade + &quot;)&quot;);

    // ファイル保存
    // ... ファイル書き込み処理
}

// 良い例：責任を分割
static bool ValidateStudentData(string name, int score)
{
    return !string.IsNullOrEmpty(name) &amp;&amp; score &gt;= 0 &amp;&amp; score &lt;= 100;
}

static string CalculateGrade(int score)
{
    return score &gt;= 60 ? &quot;合格&quot; : &quot;不合格&quot;;
}

static void DisplayStudentResult(string name, int score, string grade)
{
    Console.WriteLine(name + &quot;: &quot; + score + &quot;点 (&quot; + grade + &quot;)&quot;);
}

    2. 適切な命名

    メソッド名は動詞で始まり、何をするかが明確に分かるようにします。

    // 良い命名例
static void CalculateTotal()      // 合計を計算する
static bool IsValidEmail()       // メールアドレスが有効かチェック
static string FormatDate()       // 日付をフォーマットする
static int FindMaxValue()        // 最大値を見つける

    重要なポイント

    メソッドの基本概念

    • メソッドとは：特定の処理をまとめて名前を付けた「命令」
    • なぜ必要か：コードの重複回避、保守性向上、可読性向上、テストしやすさ
    • DRY原則：Don't Repeat Yourself（同じコードを繰り返し書かない）

    メソッドの構成要素

    • static キーワード：現段階では「メソッド作成に必要なおまじない」として理解
    • 戻り値の型：void（何も返さない）または具体的な型（int, string等）
    • メソッド名：動詞で始まり、何をするかが明確に分かる名前
    • 引数：メソッドに渡す「材料」

    引数と戻り値の理解

    • 引数：メソッドへの入力データ（材料）
    • 戻り値：メソッドからの出力データ（結果）
    • 値渡し：引数は元の変数のコピーが渡される
    • return文：戻り値がある場合は必須

    メソッドの実行の流れ

    • 呼び出し：メソッド名を書くことで実行される
    • 実行順序：呼び出されたメソッドが完全に終わってから元に戻る
    • スコープ：メソッド内の変数はそのメソッド内でのみ使用可能

    メソッドオーバーロード

    • 同じ名前で複数定義可能：引数の型・個数・順序が異なれば別メソッド
    • 自動選択：呼び出し時の引数に応じて適切なメソッドが自動選択される
    • 利便性：用途は同じだが受け取るデータが違う場合に便利

    メソッド設計の指針

    • 単一責任の原則：1つのメソッドは1つの責任のみを持つ
    • 適切なサイズ：長すぎず、理解しやすいサイズに
    • 適切な命名：処理内容が名前から推測できるように
    • 引数の数：多すぎる場合は設計を見直す

    まとめ

    メソッドの本質

    • 処理の部品化：複雑なプログラムを小さな部品に分割
    • 再利用性：一度作った処理を何度でも使える
    • 保守性：修正や改良が簡単になる

    メソッドの3つの要素

    • 入力（引数）：処理に必要なデータを受け取る
    • 処理（メソッド本体）：具体的な処理を実行
    • 出力（戻り値）：処理結果を呼び出し元に返す

    メソッド活用の効果

    • コードの整理：複雑な処理を理解しやすい単位に分割
    • バグの軽減：同じ処理を何度も書く必要がなくなる
    • チーム開発：メソッド単位で作業分担が可能
    • テスト容易性：個別機能をメソッド単位でテスト可能

    実践的な使い方

    • 計算処理：引数を受け取り、計算結果を戻り値で返す
    • 入力処理：ユーザーからの入力を取得して戻り値で返す
    • 表示処理：データを受け取り、整形して表示（戻り値なし）
    • 検証処理：データをチェックして、真偽値で結果を返す

    メソッドのレベルアップ

    • 基本：単純な処理をメソッド化
    • 応用：配列やループと組み合わせた処理
    • 実践：複数のメソッドを組み合わせたシステム構築
