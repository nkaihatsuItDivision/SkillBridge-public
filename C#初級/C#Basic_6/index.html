    配列で複数のデータを効率管理

    text 配列の概念と基本的な使い方を学習する。複数のデータを効率的に管理し、ループと組み合わせて処理する方法を身につける。 レッスン6: 配列で複数のデータを効率管理 # レッスン6: 配列で複数のデータを効率管理

    学習目標

    • 配列の概念と必要性を理解する
    • 配列の宣言、初期化、要素へのアクセス方法を学ぶ
    • 配列とループを組み合わせた処理ができる
    • 多次元配列の基本を理解する

    配列とは

    配列とは、同じ種類のデータを連続したメモリ領域に並べて保存する仕組みです。 例えるなら「番号付きのロッカー」のようなもので、各ロッカーには同じ種類の荷物（データ）を 1つずつ入れることができ、番号（インデックス）を指定して取り出すことができます。

    なぜ配列が必要なのか

    関連するデータをまとめて管理することで： - コードが短くて済む - ループで一括処理ができる - データの追加・変更が簡単 - プログラムの保守性が向上する

    配列を使わない場合の問題

    // 10人の学生の点数を管理したい場合
int student1 = 85;
int student2 = 92;
int student3 = 78;
// ... 10個の変数が必要！

    配列を使った場合

    // 10人分の点数を1つの配列で管理
int[] scores = new int[10];

    配列の基本操作

    1. 配列の宣言と初期化

    方法1: サイズを指定して宣言

    int[] numbers = new int[5]; // 5つの要素を持つ配列

    方法2: 初期値を指定して宣言

    int[] numbers = {10, 20, 30, 40, 50};
// または
int[] numbers = new int[] {10, 20, 30, 40, 50};

    2. 配列要素へのアクセス

    int[] scores = {85, 92, 78, 95, 88};

// 読み取り（インデックスは0から始まる）
Console.WriteLine(scores[0]); // 85が表示
Console.WriteLine(scores[1]); // 92が表示

// 書き込み
scores[0] = 90; // 最初の要素を90に変更

    インデックス（添字）とは

    配列の各要素を特定するための番号です。

    なぜ0から始まるのか

    プログラムの世界では「最初の位置からの距離」で考えます： - 1番目の要素：最初の位置から0個分進んだ場所 → インデックス0 - 2番目の要素：最初の位置から1個分進んだ場所 → インデックス1 - 3番目の要素：最初の位置から2個分進んだ場所 → インデックス2 

    int[] numbers = {10, 20, 30, 40, 50};
//               ↑   ↑   ↑   ↑   ↑
//               0   1   2   3   4  ← インデックス

    3. 配列の長さとインデックスの関係

    int[] numbers = {10, 20, 30, 40, 50};

Console.WriteLine(numbers.Length);     // 5（要素の個数）
Console.WriteLine(numbers[0]);         // 10（最初の要素）
Console.WriteLine(numbers[4]);         // 50（最後の要素）
// Console.WriteLine(numbers[5]);      // エラー！存在しない
    重要：配列の長さが5の場合、有効なインデックスは0〜4まで
    最後の要素のインデックス = 配列の長さ - 1

    4.配列のエラー（境界例外）について

    配列の境界を超えた場合のエラー

    int[] numbers = {10, 20, 30};

// 正常なアクセス
Console.WriteLine(numbers[0]);  // OK: 10
Console.WriteLine(numbers[2]);  // OK: 30

// エラーになるアクセス
Console.WriteLine(numbers[3]);  // 実行時エラー！IndexOutOfRangeException
Console.WriteLine(numbers[-1]); // 実行時エラー！IndexOutOfRangeException

    エラーを防ぐ方法

    if (index &gt;= 0 &amp;&amp; index &lt; numbers.Length)
{
    Console.WriteLine(numbers[index]); // 安全なアクセス
}

    例題1：学生の点数管理システム

    まずは下記のコードを手作業で書き写して流れを理解しながら出力を確認しましょう

    using System;

namespace StudentScores
{
    class Program
    {
        static void Main(string[] args)
        {
            Console.WriteLine(&quot;=== 学生点数管理システム ===&quot;);

            // 5人の学生の点数を管理
            int[] scores = new int[5];
            string[] names = new string[5];

            // データ入力
            for (int i = 0; i &lt; 5; i++)
            {
                Console.WriteLine((i + 1) + &quot;人目の学生の名前を入力:&quot;);
                names[i] = Console.ReadLine();

                Console.WriteLine(names[i] + &quot;さんの点数を入力:&quot;);
                scores[i] = int.Parse(Console.ReadLine());
            }

            // 結果表示
            Console.WriteLine(&quot;&quot;);
            Console.WriteLine(&quot;=== 入力された成績 ===&quot;);
            for (int i = 0; i &lt; 5; i++)
            {
                Console.WriteLine(names[i] + &quot;: &quot; + scores[i] + &quot;点&quot;);
            }

            // 平均点計算
            int total = 0;
            for (int i = 0; i &lt; scores.Length; i++)
            {
                total += scores[i];
            }
            double average = (double)total / scores.Length;

            Console.WriteLine(&quot;&quot;);
            Console.WriteLine(&quot;平均点: &quot; + average.ToString(&quot;F1&quot;) + &quot;点&quot;);

            // 最高点と最低点
            int highest = scores[0];
            int lowest = scores[0];
            string topStudent = names[0];
            string bottomStudent = names[0];

            for (int i = 1; i &lt; scores.Length; i++)
            {
                if (scores[i] &gt; highest)
                {
                    highest = scores[i];
                    topStudent = names[i];
                }
                if (scores[i] &lt; lowest)
                {
                    lowest = scores[i];
                    bottomStudent = names[i];
                }
            }

            Console.WriteLine(&quot;最高点: &quot; + topStudent + &quot; (&quot; + highest + &quot;点)&quot;);
            Console.WriteLine(&quot;最低点: &quot; + bottomStudent + &quot; (&quot; + lowest + &quot;点)&quot;);
        }
    }
}

    === 学生点数管理システム ===
    1人目の学生の名前を入力:

    foreach文（配列の要素を順番に処理）

    配列やコレクションの要素を、最初から最後まで順番に1つずつ取り出して処理するための構文です。

    foreach文の基本構文

    foreach (データ型 変数名 in 配列名)
{
    // 各要素に対する処理
}

    動作の仕組み

    foreach文は自動的に以下の処理を行います：

    • 配列の最初の要素を変数に代入
    • { } 内の処理を実行
    • 次の要素を変数に代入
    • { } 内の処理を実行
    • 全ての要素を処理するまで3-4を繰り返し

    具体例

    string[] fruits = {&quot;りんご&quot;, &quot;バナナ&quot;, &quot;オレンジ&quot;};

// for文を使った場合
for (int i = 0; i &lt; fruits.Length; i++)
{
    Console.WriteLine(fruits[i]);
}

// foreach文を使った場合（より簡潔）
foreach (string fruit in fruits)
{
    Console.WriteLine(fruit);
}

    foreach文の特徴

    • 自動的に全要素を処理：インデックスを管理する必要がない
    • 境界エラーが起きない：配列の範囲を超える心配がない
    • コードが簡潔：for文より短く書ける
    • 読みやすい：「配列の各要素に対して」という意図が明確

    foreach文とfor文の比較

    for文

    • インデックスを使って要素にアクセス
    • インデックス値も一緒に取得可能
    • 配列の要素を変更可能
    • 処理順序を自由に制御可能

    foreach文

    • インデックスを意識せずに要素を順次処理
    • コードがシンプルで読みやすい
    • 配列の境界を超える心配がない
    • 要素の値は変更できない（読み取り専用）

    使い分けの指針

    • 要素の値を変更したい → for文
    • インデックスが必要 → for文
    • 単純に全要素を表示・計算したい → foreach文

    多次元配列とは

    表やマトリックスのような「縦と横」の概念でデータを管理する配列です。

    練習問題

    以下の機能を持つ「家計簿プログラム」を作成してください： 1. 1週間分の支出を記録（7日分の配列） 2. 曜日ごとの支出入力 3. 週の合計支出を計算 4. 最も支出が多かった日を表示

{
    class Program
    {
        static void Main(string[] args)
        {

            string[] dayNames = {&quot;月曜日&quot;, &quot;火曜日&quot;, &quot;水曜日&quot;, &quot;木曜日&quot;, 
                               &quot;金曜日&quot;, &quot;土曜日&quot;, &quot;日曜日&quot;};
            int[] expenses = new int[7];

            // 支出入力
            for (int i = 0; i &lt; 7; i++)
            {
                Console.WriteLine(dayNames[i] + &quot;の支出を入力してください（円）:&quot;);
                expenses[i] = int.Parse(Console.ReadLine());
            }

            // 結果表示
            Console.WriteLine(&quot;&quot;);
            Console.WriteLine(&quot;=== 週間支出レポート ===&quot;);

            int total = 0;
            for (int i = 0; i &lt; 7; i++)
            {
                Console.WriteLine(dayNames[i] + &quot;: &quot; + expenses[i] + &quot;円&quot;);
                total += expenses[i];
            }

            Console.WriteLine(&quot;&quot;);
            Console.WriteLine(&quot;週の合計支出: &quot; + total + &quot;円&quot;);
            Console.WriteLine(&quot;1日平均: &quot; + (total / 7) + &quot;円&quot;);

            // 最大支出日を探す
            int maxExpense = expenses[0];

                    maxDay = dayNames[i];
                }

        }
    }

    配列のメモリ使用量

    配列のサイズについて

    配列は宣言時にサイズを決める必要があり、後から変更できません。 

    int[] numbers = new int[1000];  // 1000個分のメモリを確保
// 実際に50個しか使わなくても、1000個分のメモリを消費

    サイズ決定の指針

    • 必要な最大数を見積もる
    • メモリ使用量とのバランスを考える
    • 動的にサイズを変えたい場合はListクラスを検討（後のレッスンで学習）

    配列を使った実践的なアルゴリズム

    バブルソート（配列の並び替え）

    using System;

namespace BubbleSort
{
    class Program
    {
        static void Main(string[] args)
        {
            int[] numbers = {64, 34, 25, 12, 22, 11, 90};

            Console.WriteLine(&quot;ソート前:&quot;);
            foreach (int num in numbers)
            {
                Console.Write(num + &quot; &quot;);
            }
            Console.WriteLine();

            // バブルソート
            for (int i = 0; i &lt; numbers.Length - 1; i++)
            {
                for (int j = 0; j &lt; numbers.Length - 1 - i; j++)
                {
                    if (numbers[j] &gt; numbers[j + 1])
                    {
                        // 要素を交換
                        int temp = numbers[j];
                        numbers[j] = numbers[j + 1];
                        numbers[j + 1] = temp;
                    }
                }
            }

            Console.WriteLine(&quot;ソート後:&quot;);
            foreach (int num in numbers)
            {
                Console.Write(num + &quot; &quot;);
            }
            Console.WriteLine();
        }
    }
}

    ソート前:
    64 34 25 12 22 11 90
    ソート後:
    11 12 22 25 34 64 90

    重要なポイント

    配列の基本概念

    • 配列とは：同じ種類のデータを番号付きで整理して保存する仕組み
    • インデックス：配列の各要素を特定する番号（0から始まる）
    • 配列の長さ：要素の個数（.Lengthで取得）

    インデックスに関する重要な注意点

    • 0から始まる：1番目の要素はインデックス0
    • 最後の要素：配列の長さ-1がインデックス
    • 境界エラー：存在しないインデックスにアクセスすると実行時エラー

    配列の宣言と初期化

    • サイズ指定：int[] array = new int[5];（後で値を設定）
    • 初期値指定：int[] array = {1, 2, 3, 4, 5};（宣言と同時に値を設定）
    • サイズは固定：宣言後にサイズ変更は不可

    for文とforeach文の使い分け

    • for文を選ぶ場面：
    • 配列の要素を変更したい時
    • インデックス番号が必要な時
    • 特定の順序で処理したい時
    • foreach文を選ぶ場面：
    • 全要素を順番に読み取るだけの時
    • 簡潔なコードを書きたい時
    • 境界エラーを避けたい時

    配列を使った基本処理パターン

    • 全要素の表示：foreachが最適
    • 合計・平均の計算：foreachまたはfor文
    • 最大値・最小値の検索：for文（比較処理が必要）
    • 特定の値の検索：for文（インデックスも必要な場合）

    エラーを避けるためのチェック

    • 境界チェック：if (index &gt;= 0 &amp;&amp; index &lt; array.Length)
    • null チェック：配列が初期化されているか確認
    • 要素数の確認：空の配列でないか確認

    まとめ

    配列の本質

    • 効率的なデータ管理：関連するデータをまとめて管理
    • 番号による管理：インデックスで個別の要素にアクセス
    • ループとの相性：繰り返し処理で真価を発揮

    配列の3つの基本操作

    • 宣言・初期化：データの入れ物を用意
    • 要素へのアクセス：インデックスを使って読み書き
    • 全体の処理：ループを使って一括処理

    実践的な活用場面

    • データの一覧管理：成績、在庫、家計簿など
    • 計算処理：合計、平均、最大値・最小値
    • 検索・並び替え：特定データの検索やソート処理
    • 表形式データ：多次元配列で行列データを管理

    今後の発展

    • Listクラス：サイズが動的に変わるコレクション
    • LINQ：配列やコレクションの高度な操作
    • アルゴリズム：配列を使った効率的な処理方法
