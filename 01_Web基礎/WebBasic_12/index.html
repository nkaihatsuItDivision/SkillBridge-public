    1. なぜVSCodeが必要なのか？

    これまでメモ帳でHTML/CSSを学習してきましたが、より複雑なWebページを作る際には専用のツールが必要になります。

    メモ帳の限界

    ❌ メモ帳での困ること： - コードの色分けがない（見づらい） - ファイルの切り替えが面倒 - 変更をブラウザで確認するのに手間 - 間違いを見つけにくい - 複数ファイルの管理が大変

    VSCodeの利点

    ✅ VSCodeでできること： - シンタックスハイライト（コードが色分けされて見やすい） - 自動補完機能（タグを途中まで打つと候補が出る） - リアルタイムプレビュー（保存と同時にブラウザが更新） - ファイル管理（複数ファイルを簡単に切り替え） - エラーチェック（間違いがあると教えてくれる）

    📘​ メモ帳 vs VSCode

    2. VSCodeのダウンロードとインストール

    ステップ1：VSCodeの公式サイトにアクセス

    1. ブラウザで https://code.visualstudio.com/ にアクセス
    2. 「Download for Windows」（またはお使いのOS）ボタンをクリック
    3. ダウンロードが開始されます

    ステップ2：インストール実行

    1. ダウンロードした VSCodeUserSetup-○○○.exe をダブルクリック
    2. 「使用許諾契約書に同意する」にチェックを入れる
    3. 「次へ」を数回クリック
    4. 以下の項目にチェックを入れることを推奨：
    5. ✅ 「デスクトップ上にアイコンを作成する」
    6. ✅ 「エクスプローラーのファイル コンテキスト メニューに [Code で開く] アクションを追加する」
    7. ✅ 「エクスプローラーのディレクトリ コンテキスト メニューに [Code で開く] アクションを追加する」
    8. 「インストール」をクリック
    9. インストール完了後、「VSCodeを実行する」にチェックを入れて「完了」

    ステップ3：初回起動

    1. VSCodeが起動します
    2. 日本語化（任意）：
    3. 左側の四角いアイコン（Extensions）をクリック
    4. 検索欄に 「Japanese」 と入力
    5. 「Japanese Language Pack for Visual Studio Code」をインストール
    6. 「Ctrl+Shift+P」(コマンドパレット)を表示して「Configure」と入力
    7. 「Configure Display Language」クリックして日本語を選択
    8. VSCodeを再起動（基本的に自動）

    3. VSCodeの基本操作

    インターフェースの説明

    VSCodeの画面構成： - エクスプローラー（左側）：ファイルとフォルダの一覧 - エディター（中央）：コードを書く場所 - ターミナル（下部）：コマンドを実行する場所（今回は使用しません）

    フォルダを開く

    重要：VSCodeはフォルダ単位で作業します

    1. 「ファイル」メニュー → 「フォルダーを開く」
    2. デスクトップに 「web-practice」 フォルダを作成
    3. 作成したフォルダを選択して 「フォルダーの選択」

    ファイルの作成

    1. エクスプローラーの 「web-practice」 を右クリック
    2. 「新しいファイル」を選択
    3. 「test.html」 と入力してEnter

    基本的なHTMLを書いてみる

    1. test.html をクリックして開く
    2. 以下のコードを入力：
    &lt;!DOCTYPE html&gt;
&lt;html lang=&quot;ja&quot;&gt;
&lt;head&gt;
    &lt;meta charset=&quot;UTF-8&quot;&gt;
    &lt;title&gt;VSCode練習&lt;/title&gt;
&lt;/head&gt;
&lt;body&gt;
    &lt;h1&gt;こんにちは、VSCode！&lt;/h1&gt;
    &lt;p&gt;これはVSCodeで作成したHTMLです。&lt;/p&gt;
&lt;/body&gt;
&lt;/html&gt;

    気づくこと： - コードが色分けされている - タグを打つと自動で閉じタグが入る - インデント（字下げ）が自動で整う

    4. Live Server拡張機能のインストール

    Live Serverは、HTMLファイルを保存すると自動でブラウザが更新される便利な機能です。

    ステップ1：拡張機能の検索

    1. 左側のメニューから 拡張機能（四角が4つのアイコン）をクリック
    2. 検索欄に 「Live Server」 と入力
    3. 「Live Server」（作者：Ritwick Dey）を見つける

    ステップ2：インストール

    1. 「インストール」ボタンをクリック
    2. インストール完了まで少し待つ
    3. 「インストール済み」と表示されれば完了

    ステップ3：Live Serverの使用

    1. test.html を開いた状態で右クリック
    2. 「Open with Live Server」を選択
    3. ブラウザが自動で開き、Webページが表示される
    4. アドレスバーに http://127.0.0.1:5500/test.html のような表示

    ※右下の「Go Live」をクリックすることでも起動可能

    💡 Live Serverの便利な点： - ファイルを保存すると自動でブラウザが更新 - 複数のファイルを同時に開ける - スマートフォンからもアクセス可能

    練習2：CSSファイルの分離（応用）

    レッスン10で学んだ外部スタイルシートを、VSCodeで実際に体験してみましょう：

    1. style.css という新しいファイルを作成
    2. HTMLから &lt;style&gt; 部分を切り取って、style.css に貼り付け
    3. HTMLの &lt;head&gt; 内に以下を追加：
    &lt;link rel=&quot;stylesheet&quot; href=&quot;style.css&quot;&gt;
    1. Live Serverで正しく表示されることを確認

    6. VSCodeの便利な機能

    Emmet（エメット）機能

    HTML構造を爆速で作成：

    1. 新しいHTMLファイルを作成
    2. 「!」（エクスクラメーション）を入力してTab
    3. 基本的なHTML構造が自動生成される

    他の便利なEmmet： - 「h1」 + Tab → &lt;h1&gt;&lt;/h1&gt; - 「p」 + Tab → &lt;p&gt;&lt;/p&gt; - 「div.box」 + Tab → &lt;div class=&quot;box&quot;&gt;&lt;/div&gt; - 「ul>li*3」 + Tab → リストが3つ自動生成

    自動整形機能

    コードを綺麗に整える： 1. Shift + Alt + F（Windows）で自動整形 2. インデントが自動で整う 3. 閉じタグの位置も修正される

    マルチカーソル機能

    同じ文字を一度に編集： 1. Ctrlを押しながら同じ単語をクリック 2. 複数箇所を同時に編集できる 3. 効率的なコード修正が可能

    7. トラブルシューティング

    よくある問題と解決方法

    問題1：Live Serverが動かない - 解決策：ファイルがHTMLであることを確認（拡張子が.html） - 解決策：VSCodeを再起動してみる

    問題2：日本語が文字化けする - 解決策：HTMLの &lt;meta charset=&quot;UTF-8&quot;&gt; を確認 - 解決策：ファイルの保存時にUTF-8を選択

    問題3：CSSが適用されない - 解決策：&lt;link&gt; タグのパスを確認 - 解決策：CSSファイル名の拡張子を確認（.css）

    問題4：拡張機能が見つからない - 解決策：インターネット接続を確認 - 解決策：VSCodeを最新版に更新

    8. フォルダ・ファイル管理のベストプラクティス

    推奨するフォルダ構成

    web-projects/
├── lesson8-portfolio/          # レッスン8用
│   ├── index.html
│   ├── style.css
│   └── images/
│       └── profile.jpg
├── practice/                   # 練習用
│   ├── test.html
│   └── style.css
└── archive/                    # 過去の作品保存用
    ├── lesson3-profile.html
    └── lesson7-layout.html

    ファイル命名のルール

    ✅ 良い命名： - index.html（メインページ） - style.css（スタイルシート） - contact.html（お問い合わせページ）

    ❌ 避けるべき命名： - テスト.html（日本語） - my homepage.html（スペース） - PAGE1.HTML（大文字）

    Git管理への準備

    将来のために覚えておくべきこと： - フォルダ構成を整理しておく - ファイル名は英語で統一 - README.mdでプロジェクトの説明を書く

    10. まとめ

    習得したスキル

    ✅ 開発環境： - VSCode：プロ仕様のコードエディタ導入 - Live Server：リアルタイムプレビュー機能 - ファイル管理：効率的なプロジェクト管理 - 拡張機能：開発効率を上げるツール活用

    メモ帳からの進歩

    Before（メモ帳）： - 手動でブラウザ更新 - ファイル切り替えが面倒 - コードが見づらい

    After（VSCode）： - 自動でブラウザ更新 - ファイル管理が簡単 - コードが見やすく、間違いも発見しやすい

    重要なポイント

    • プロと同じ環境でWeb開発ができるようになった
    • 効率的な開発が可能になった
    • より複雑なプロジェクトにも対応できる
    • 将来の学習（JavaScript等）への準備完了
