    1. HTMLタグで文章に「意味」を付ける

    前回学んだ基本構造の中の&lt;body&gt;部分に、実際にコンテンツを書いていきます。

    なぜタグが必要なのか？

    タグなしの文章（プレーンテキスト）： 

    プログラミング学習のコツ
プログラミングを効率的に学ぶためには以下のポイントが重要です。
毎日少しずつでも継続すること
実際に手を動かしてコードを書くこと
エラーを恐れずにチャレンジすること

    タグありの文章（HTML）： 

    &lt;h1&gt;プログラミング学習のコツ&lt;/h1&gt;
&lt;p&gt;プログラミングを効率的に学ぶためには以下のポイントが重要です。&lt;/p&gt;
&lt;ul&gt;
    &lt;li&gt;毎日少しずつでも&lt;strong&gt;継続&lt;/strong&gt;すること&lt;/li&gt;
    &lt;li&gt;実際に手を動かしてコードを書くこと&lt;/li&gt;
    &lt;li&gt;エラーを恐れずにチャレンジすること&lt;/li&gt;
&lt;/ul&gt;

    タグを使うメリット： - 見た目が整理される（見出しが大きく、段落が分かれる） - 意味が明確になる（これは見出し、これは段落） - 検索エンジンが理解しやすくなる - 音声読み上げソフトが正しく読める

    2. 見出しタグ（h1～h6）で文書を階層化

    見出しタグの種類と大きさ

    HTMLには6段階の見出しがあります：

    &lt;h1&gt;最も重要な見出し（一番大きい）&lt;/h1&gt;
&lt;h2&gt;2番目に重要な見出し&lt;/h2&gt;
&lt;h3&gt;3番目に重要な見出し&lt;/h3&gt;
&lt;h4&gt;4番目に重要な見出し&lt;/h4&gt;
&lt;h5&gt;5番目に重要な見出し&lt;/h5&gt;
&lt;h6&gt;最も小さい見出し（一番小さい）&lt;/h6&gt;
    出力例：

    見出しの正しい使い方

    ✅ 正しい階層構造： 

    &lt;h1&gt;料理のレシピ集&lt;/h1&gt;
    &lt;h2&gt;和食&lt;/h2&gt;
        &lt;h3&gt;肉じゃが&lt;/h3&gt;
        &lt;h3&gt;味噌汁&lt;/h3&gt;
    &lt;h2&gt;洋食&lt;/h2&gt;
        &lt;h3&gt;ハンバーグ&lt;/h3&gt;
        &lt;h3&gt;オムライス&lt;/h3&gt;

    ❌ 間違った使い方： 

    &lt;h1&gt;料理のレシピ集&lt;/h1&gt;
&lt;!-- h2を飛ばしてh4はダメ --&gt;&lt;h4&gt;和食&lt;/h4&gt;
&lt;!-- h1は1ページに1つが基本 --&gt;&lt;h1&gt;肉じゃが&lt;/h1&gt;

    見出しの重要ルール

    1. h1は1ページに1つだけ（ページの主題）
    2. 階層を飛ばさない（h1の次はh2、h2の次はh3）
    3. 見た目ではなく意味で選ぶ（大きくしたいからh1にするのはNG）

    3. 段落タグ（p）で文章を整理

    pタグの基本的な使い方

    段落とは：一つのまとまった内容を表す文章のかたまり

    &lt;p&gt;これは一つの段落です。関連する内容をひとまとめにして書きます。&lt;/p&gt;

&lt;p&gt;これは別の段落です。内容が変わるときは新しい段落にします。&lt;/p&gt;

&lt;p&gt;各段落の間には自動的にスペースが入ります。&lt;/p&gt;

    pタグを使わない場合との比較

    ❌ pタグを使わない場合： 

    これは段落１です。
これは段落２です。
これは段落３です。
    → 改行されずに一続きで表示される

    ✅ pタグを使った場合： 

    &lt;p&gt;これは段落１です。&lt;/p&gt;
&lt;p&gt;これは段落２です。&lt;/p&gt;
&lt;p&gt;これは段落３です。&lt;/p&gt;
    → 段落ごとに分かれて見やすく表示される

    段落の使い分けの目安

    • 話題が変わるときは新しい段落に
    • 長すぎる文章は複数の段落に分ける
    • 一つの段落は3～5行程度が読みやすい

    4. 改行タグ（br）の正しい使い方

    brタグの特徴

    &lt;p&gt;
    住所：東京都渋谷区〇〇&lt;br&gt;
    電話：03-1234-5678&lt;br&gt;
    メール：example@email.com
&lt;/p&gt;

    brタグの特徴： - 閉じタグがない（&lt;br&gt;だけでOK） - 強制的に改行する - 段落内での改行に使用

    brタグの適切な使用場面

    ✅ 使ってもよい場面： - 住所や連絡先の表示 - 詩や歌詞の表示 - 段落内で改行したい場合

    ❌ 使わない方がよい場面： - 段落を分けたいとき → &lt;p&gt;タグを使う - 余白を作りたいとき → CSSで調整する - レイアウト調整のため → CSSで調整する

    間違った使い方の例

    ❌ 悪い例： 

    &lt;h1&gt;タイトル&lt;/h1&gt;
&lt;!-- 余白作りにbrは使わない --&gt;&lt;br&gt;&lt;br&gt;&lt;br&gt;
&lt;p&gt;本文です&lt;/p&gt;

    ✅ 良い例： 

    &lt;h1&gt;タイトル&lt;/h1&gt;
&lt;p&gt;本文です&lt;/p&gt;

    5. 強調タグ（strong、em）で重要部分を目立たせる

    strongタグ（重要な強調）

    &lt;p&gt;このレッスンで&lt;strong&gt;最も重要&lt;/strong&gt;なポイントは正しいタグの使い分けです。&lt;/p&gt;

    strongタグの特徴： - 内容的に重要な部分に使用 - デフォルトでは太字で表示される - 検索エンジンも重要と認識する

    emタグ（強勢・語調の強調）

    &lt;p&gt;このタグは&lt;em&gt;本当に&lt;/em&gt;便利です。&lt;/p&gt;

    emタグの特徴： - 語調や強勢を表現 - デフォルトでは斜体（イタリック）で表示される - 音声読み上げ時に強調される

    strongとemの使い分け

    strong：重要性 

    &lt;p&gt;&lt;strong&gt;注意：&lt;/strong&gt;パスワードは他人に教えないでください。&lt;/p&gt;

    em：語調 

    &lt;p&gt;この映画は&lt;em&gt;本当に&lt;/em&gt;面白かった。&lt;/p&gt;

    注意：見た目だけのbタグ、iタグは避ける

    ❌ 推奨されない： 

    &lt;!-- 見た目だけの太字 --&gt;&lt;b&gt;太字&lt;/b&gt;  
&lt;!-- 見た目だけの斜体 --&gt;&lt;i&gt;斜体&lt;/i&gt;

    ✅ 推奨される： 

    &lt;!-- 意味のある強調 --&gt;&lt;strong&gt;重要&lt;/strong&gt;  
&lt;!-- 意味のある強調 --&gt;&lt;em&gt;強勢&lt;/em&gt;

    6. 実践課題

    課題：自己紹介記事を作成しよう

    以下の内容を参考に、学んだタグを使って自己紹介記事を作成してください。

    作成する内容： 1. メインタイトル：「自己紹介」 2. 基本情報セクション： - セクションタイトル：「基本情報」 - 名前、出身地、年齢など 3. 趣味セクション： - セクションタイトル：「趣味・興味」 - 好きなことや趣味について 4. 目標セクション： - セクションタイトル：「学習目標」 - プログラミング学習の目標

    手順： 1. 前回作った基本構造のテンプレートを使用 2. 以下のサンプルを参考に作成

    &lt;!DOCTYPE html&gt;
&lt;html lang=&quot;ja&quot;&gt;
&lt;head&gt;
    &lt;meta charset=&quot;UTF-8&quot;&gt;
    &lt;title&gt;田中太郎の自己紹介&lt;/title&gt;
&lt;/head&gt;
&lt;body&gt;
    &lt;h1&gt;自己紹介&lt;/h1&gt;

    &lt;h2&gt;基本情報&lt;/h2&gt;
    &lt;p&gt;はじめまして！私の名前は&lt;strong&gt;田中太郎&lt;/strong&gt;です。&lt;/p&gt;
    &lt;p&gt;
        出身：大阪府大阪市&lt;br&gt;
        年齢：25歳&lt;br&gt;
        職業：会社員
    &lt;/p&gt;

    &lt;h2&gt;趣味・興味&lt;/h2&gt;
    &lt;p&gt;読書が大好きで、特にビジネス書をよく読みます。&lt;/p&gt;
    &lt;p&gt;最近はプログラミングに&lt;em&gt;とても&lt;/em&gt;興味を持っています。&lt;/p&gt;

    &lt;h2&gt;学習目標&lt;/h2&gt;
    &lt;p&gt;半年後には&lt;strong&gt;Webサイトを自作&lt;/strong&gt;できるようになりたいです。&lt;/p&gt;
    &lt;p&gt;将来的にはエンジニアとして転職することが目標です。&lt;/p&gt;
&lt;/body&gt;
&lt;/html&gt;
    1. ファイル名を my-introduction.html で保存
    2. ブラウザで開いて確認

    解答・解説

    期待される表示： ​📘 結果一例

    使用したタグの確認： - h1：ページ全体のメインタイトル - h2：各セクションのタイトル - p：段落ごとに内容を整理 - br：住所情報内での改行 - strong：重要な情報（名前、目標）を強調 - em：語調の強調

    評価ポイント： - [ ] h1は1つだけ使用されているか - [ ] h1→h2の階層が正しいか - [ ] 段落が適切に分けられているか - [ ] strongとemが意味を持って使用されているか - [ ] brが適切な場面で使用されているか

    7. よくある間違いと改善方法

    間違い1：見出しを見た目で選ぶ

    &lt;!-- 間違い --&gt;&lt;h1&gt;メインタイトル&lt;/h1&gt;
&lt;h4&gt;小さく見せたいサブタイトル&lt;/h4&gt;
&lt;!-- h2を飛ばしている --&gt;  

&lt;!-- 正しい --&gt;&lt;h1&gt;メインタイトル&lt;/h1&gt;
&lt;h2&gt;サブタイトル&lt;/h2&gt;  
&lt;!-- 見た目はCSSで調整 --&gt;

    間違い2：段落を作らずにbrだけで改行

    &lt;!-- 間違い --&gt;段落1の内容です。&lt;br&gt;&lt;br&gt;
段落2の内容です。&lt;br&gt;&lt;br&gt;

&lt;!-- 正しい --&gt;&lt;p&gt;段落1の内容です。&lt;/p&gt;
&lt;p&gt;段落2の内容です。&lt;/p&gt;

    間違い3：意味のない強調の多用

    &lt;!-- 間違い --&gt;&lt;p&gt;&lt;strong&gt;私&lt;/strong&gt;の&lt;strong&gt;名前&lt;/strong&gt;は&lt;strong&gt;田中&lt;/strong&gt;です。&lt;/p&gt;

&lt;!-- 正しい --&gt;&lt;p&gt;私の名前は&lt;strong&gt;田中太郎&lt;/strong&gt;です。&lt;/p&gt;

    8. まとめ

    学んだこと

    ✅ 見出しタグ（h1～h6）：文書の階層構造を作る
    ✅ 段落タグ（p）：まとまった内容をグループ化する
    ✅ 改行タグ（br）：段落内での強制改行
    ✅ 強調タグ（strong、em）：重要部分や語調の強調
    ✅ 文書の構造化：意味のあるマークアップの重要性

    重要なポイント

    • 見出しは階層を守って使用：h1→h2→h3の順番を守る
    • 段落は意味のあるまとまりで区切る：話題が変わったら新しい段落に
    • brは段落分けには使わない：段落分けはpタグを使用
    • 強調は意味を持って使用：見た目だけでなく内容の重要性を考える
