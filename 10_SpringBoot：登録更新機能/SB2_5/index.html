    🔧 @PostMappingによるフォーム受信処理

    HTTPメソッドの使い分け

    Webアプリケーションでは、HTTPメソッドによって処理の種類を区別します：

    主要なHTTPメソッド： - GET: データの取得・画面の表示 - POST: データの作成・フォームの送信 - PUT/PATCH: データの更新 - DELETE: データの削除

    📘 HTTP メソッドとコントローラー

    新規登録処理の実装

    Week 9.2で作成したフォームからのPOSTリクエストを受信する処理を実装します：

    @Controller
@RequestMapping("/students")
public class StudentController {

    @Autowired
    private StudentService studentService;

    // Week 9.2で実装：フォーム表示
    @GetMapping("/create")
    public String showForm(Model model) {
        model.addAttribute("studentData", new StudentData());
        return "students/form";
    }

    // Week 9.5で実装：フォーム受信処理
    @PostMapping("/create")
    public String createStudent(
            @Valid @ModelAttribute StudentData studentData,
            BindingResult bindingResult,
            Model model) {

        // Week 9.3-9.4技術の統合活用
        if (bindingResult.hasErrors()) {
            // エラー時は入力フォームを再表示
            return "students/form";
        }

        // Week 9.1技術活用：DTOからEntityへ変換
        Student student = studentData.toEntity();

        // データベース保存
        studentService.save(student);

        // PRGパターン：成功時のリダイレクト
        return "redirect:/students";
    }
}

    📘 バリデーション統合の流れ

    エラーハンドリングの流れ

    3. データベース保存機能の実装

    JPA Repositoryによる安全な永続化

    Spring Boot Starterに含まれるJPA Repositoryを活用してデータベース保存を実装します：

    // StudentServiceクラスでの保存処理
@Service
@Transactional
public class StudentService {

    @Autowired
    private StudentRepository studentRepository;

    public Student save(Student student) {
        try {
            // JPA Repositoryによる安全な保存
            Student savedStudent = studentRepository.save(student);
            return savedStudent;
        } catch (DataAccessException e) {
            // データベースエラーの適切な処理
            throw new RuntimeException("データ保存に失敗しました", e);
        }
    }
}

    4. PRG（Post-Redirect-Get）パターン

    適切な画面遷移設計

    PRGパターンは、フォーム送信後の適切な画面遷移を実現する重要な設計パターンです：

    @PostMapping("/create")
public String createStudent(@Valid @ModelAttribute StudentData studentData,
                          BindingResult bindingResult) {

    if (bindingResult.hasErrors()) {
        // エラー時：フォーム画面を直接返す（POST → フォーム表示）
        return "students/form";
    }

    // 保存処理
    Student student = studentData.toEntity();
    studentService.save(student);

    // 成功時：リダイレクト（POST → Redirect → GET）
    return "redirect:/students";
}

    2. ユーザビリティ向上：戻るボタンでの適切な動作
    3. URL整合性：表示内容とURLの一致
    4. SEO対応：検索エンジン最適化
    // ❌ 間違った実装（直接ビューを返す）
