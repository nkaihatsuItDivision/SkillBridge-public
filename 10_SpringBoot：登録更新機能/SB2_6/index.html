    🚀 編集機能の重要性と価値

    なぜ編集機能が必要なのか？

    編集機能は、Webアプリケーションにおける必須機能です。実際の企業システムでは：

    顧客管理システム： - 顧客情報の住所変更 - 電話番号・メールアドレス更新 - 契約内容の変更

    学生管理システム： - 成績の修正・更新 - 履修科目の変更 - 個人情報の更新

    商品管理システム： - 価格の変更 - 在庫数の更新 - 商品説明の修正

    CRUD操作とは

    CRUDとは、データベース操作の基本である4つの操作の頭文字です：

    • Create（作成）：Week 9.5で習得 ✅
    • Read（読み取り）：Week 8で習得 ✅
    • Update（更新）：今回習得 🎯
    • Delete（削除）：Week 10で習得予定

    1. 既存データの取得処理

    ステップ1: IDによる特定データ取得

    編集機能では、まず編集対象のデータを取得する必要があります。

    @GetMapping("/students/{id}/edit")
public String showEditForm(@PathVariable Long id, Model model) {
    // IDによる特定データ取得
    Student student = studentRepository.findById(id)
        .orElseThrow(() -> new IllegalArgumentException("Invalid student Id:" + id));

    // Week 9.1のfromEntity()メソッド活用
    StudentData studentData = StudentData.fromEntity(student);

    model.addAttribute("studentData", studentData);
    return "students/edit";
}

    📘 Spring Boot PathVariable を使ったデータ取得

    @PathVariableの詳細理解

    @PathVariableは、URLの一部を変数として受け取るアノテーションです：

    // URL: /students/5/edit
@GetMapping("/students/{id}/edit")
public String showEditForm(@PathVariable Long id, Model model) {
    // ここで id = 5 が受け取られる
}

    実用的な活用例：

    // 複数のパスパラメータ
@GetMapping("/schools/{schoolId}/students/{studentId}/edit")
public String showEditForm(@PathVariable Long schoolId,
                          @PathVariable Long studentId, 
                          Model model) {

    楽観ロック例外処理

    @PatchMapping("/students/{id}")
public String updateStudent(@PathVariable Long id,
                          @Valid @ModelAttribute StudentData studentData,
                          BindingResult bindingResult,
                          Model model) {
    if (bindingResult.hasErrors()) {
        return "students/edit";
    }

    try {
        Student student = studentData.toEntity();
        student.setId(id);
        studentRepository.save(student);
        return "redirect:/students";

    } catch (OptimisticLockingFailureException e) {
        // 最新データを再取得
        Student latestStudent = studentRepository.findById(id)
            .orElseThrow(() -> new IllegalArgumentException("Invalid student Id:" + id));
        StudentData latestData = StudentData.fromEntity(latestStudent);

        model.addAttribute("studentData", latestData);
        model.addAttribute("errorMessage", 
            "データが他のユーザーによって更新されました。最新データで再度お試しください。");
        return "students/edit";
    }
}

    5. エラーハンドリングの強化

    更新処理特有のエラー処理

    編集機能では、新規登録にはない特有のエラーが発生する可能性があります：

    @PatchMapping("/students/{id}")
public String updateStudent(@PathVariable Long id,
                          @Valid @ModelAttribute StudentData studentData,
                          BindingResult bindingResult,
                          Model model) {

    // 1. バリデーションエラー（Week 9.3-9.4技術）
    if (bindingResult.hasErrors()) {
        return "students/edit";
    }

    // 2. データ存在チェック
    if (!studentRepository.existsById(id)) {
        model.addAttribute("errorMessage", "指定された学生が見つかりません。");
        return "students/edit";
    }

    try {
        Student student = studentData.toEntity();
        student.setId(id);
        studentRepository.save(student);
        return "redirect:/students";

    } catch (OptimisticLockingFailureException e) {
        // 3. 楽観ロック例外
        Student latestStudent = studentRepository.findById(id)
            .orElseThrow(() -> new IllegalArgumentException("Invalid student Id:" + id));
        StudentData latestData = StudentData.fromEntity(latestStudent);

        model.addAttribute("studentData", latestData);
        model.addAttribute("errorMessage", 
            "データが他のユーザーによって更新されました。最新データで再度お試しください。");
        return "students/edit";

    } catch (DataIntegrityViolationException e) {
        // 4. データ整合性例外（重複メールアドレスなど）
        model.addAttribute("errorMessage", "データの整合性エラーが発生しました。入力内容を確認してください。");
        return "students/edit";

    } catch (Exception e) {
        // 5. その他の例外
        model.addAttribute("errorMessage", "システムエラーが発生しました。しばらく時間をおいてから再度お試しください。");
        return "students/edit";
    }
}
