@PostMapping("/students")
public String registerStudent(@Valid @ModelAttribute StudentData studentData,
                              BindingResult bindingResult,
                              RedirectAttributes redirectAttributes) {
    if (bindingResult.hasErrors()) {
        return "students/register";
    }
    studentRepository.save(studentData.toEntity());
    redirectAttributes.addFlashAttribute("successMessage", "学生を登録しました");
    return "redirect:/students";
}

    なぜフィードバック機能が重要なのか？

    企業システムでは、ユーザーが操作を実行した際に「何が起こったのか」を明確に伝えることが必須です：

    • 成功時：「データが正常に保存されました」という安心感を提供
    • 失敗時：「何が問題だったのか」を分かりやすく説明
    • 操作継続性：次に何をすべきかを適切に案内

    🔧 Flash属性とは何か？

    リダイレクト時の一時的なデータ保持技術

    Flash属性は、「リダイレクト後に1回だけ表示されるメッセージ」を管理する仕組みです。

    技術的な仕組み： 1. Controller: RedirectAttributes.addFlashAttribute()でメッセージをセッションに保存 2. リダイレクト: PRGパターン（Post-Redirect-Get）で画面遷移 3. View表示: セッションからメッセージを取得して表示 4. 自動削除: 表示後、メッセージは自動的にセッションから削除

    📘 Spring Boot Flash Attribute の仕組み

    なぜFlash属性が必要なのか？

    問題: 通常のModelAttributeはリダイレクト後に消失してしまう

    // ❌ 通常のModelAttributeを使った場合（動作しない）
@PostMapping("/students")
public String registerStudent(@Valid @ModelAttribute StudentData studentData,
                              BindingResult bindingResult,
                              Model model) {
    if (bindingResult.hasErrors()) {
        return "students/register";
    }
    studentRepository.save(studentData.toEntity());

    // リダイレクト後、このメッセージは消失してしまう
    model.addAttribute("message", "学生を登録しました");
    return "redirect:/students";  // リダイレクトで model の内容は失われる
}

    解決: Flash属性を使用してリダイレクト後も一時的にメッセージを保持

    // ✅ Flash属性を使った場合（正常に動作）
@PostMapping("/students")
public String registerStudent(@Valid @ModelAttribute StudentData studentData,
                              BindingResult bindingResult,
                              RedirectAttributes redirectAttributes) {
    if (bindingResult.hasErrors()) {
        return "students/register";
    }
    studentRepository.save(studentData.toEntity());

    // リダイレクト後も一時的にメッセージが保持される

    2. 更新処理の成功メッセージ

    Week 9.6で作成した編集機能にもフィードバック機能を追加しましょう。

    StudentController.java（更新処理の改良）：

    @PatchMapping("/students/{id}")
public String updateStudent(@PathVariable Long id,
                          @Valid @ModelAttribute StudentData studentData,
                          BindingResult bindingResult,
                          RedirectAttributes redirectAttributes,
                          Model model) {

    // バリデーションエラーがある場合は編集画面に戻る
    if (bindingResult.hasErrors()) {
        return "students/edit";
    }

    try {
        // 更新処理の実行
        Student student = studentData.toEntity();
        student.setId(id);
        studentRepository.save(student);

        // 更新成功メッセージの設定
        redirectAttributes.addFlashAttribute("successMessage", 
            "学生「" + student.getName() + "」の情報を更新しました");

        return "redirect:/students";

    } catch (OptimisticLockingFailureException e) {
        // 楽観ロックエラー（他のユーザーが同時更新）
        model.addAttribute("errorMessage", 
            "データが他のユーザーによって更新されました。最新データで再度お試しください。");
        return "students/edit";

    } catch (Exception e) {
        // その他のシステムエラー
        model.addAttribute("errorMessage", 
            "更新処理中にエラーが発生しました。しばらく時間をおいてから再度お試しください。");
        return "students/edit";
    }
}

    重要なポイント

    エラー時の画面遷移の使い分け： - 成功時: redirectAttributes + redirect: を使用（PRGパターン） - エラー時: model + 直接テンプレート返却を使用（フォーム値保持）

    // 成功時：PRGパターンでリダイレクト
redirectAttributes.addFlashAttribute("successMessage", "成功しました");
return "redirect:/students";

// エラー時：フォーム値を保持して元の画面に戻る
model.addAttribute("errorMessage", "エラーが発生しました");
return "students/edit";  // リダイレクトしない

    2. Controller での定数クラス活用

    import com.example.studentmanagement.constants.Messages;

@Controller
public class StudentController {

    @PostMapping("/students")
    public String registerStudent(@Valid @ModelAttribute StudentData studentData,
                                  BindingResult bindingResult,
                                  RedirectAttributes redirectAttributes) {

        if (bindingResult.hasErrors()) {
            return "students/register";
        }

        try {
            Student student = studentData.toEntity();
            studentRepository.save(student);

            // 定数クラスを使用してメッセージを設定
            String message = String.format(Messages.STUDENT_REGISTERED, student.getName());
            redirectAttributes.addFlashAttribute("successMessage", message);

            return "redirect:/students";

        } catch (Exception e) {
            redirectAttributes.addFlashAttribute("errorMessage", Messages.SYSTEM_ERROR);
            return "redirect:/students/register";
        }
    }

    @PatchMapping("/students/{id}")
    public String updateStudent(@PathVariable Long id,
                              @Valid @ModelAttribute StudentData studentData,
                              BindingResult bindingResult,
                              RedirectAttributes redirectAttributes,
                              Model model) {

        if (bindingResult.hasErrors()) {
            model.addAttribute("errorMessage", Messages.VALIDATION_ERROR);
            return "students/edit";
        }

        try {
            Student student = studentData.toEntity();
            student.setId(id);
            studentRepository.save(student);

            String message = String.format(Messages.STUDENT_UPDATED, student.getName());
            redirectAttributes.addFlashAttribute("successMessage", message);

            return "redirect:/students";

        } catch (OptimisticLockingFailureException e) {
            model.addAttribute("errorMessage", Messages.OPTIMISTIC_LOCK_ERROR);
            return "students/edit";

        } catch (Exception e) {
            model.addAttribute("errorMessage", Messages.SYSTEM_ERROR);
            return "students/edit";
        }
    }
}

    📝 補足：よくある質問

    Q1: Flash属性とModelAttributeの使い分けは？

    A1: 画面遷移の種類で使い分けます。

    // リダイレクト時：Flash属性を使用
redirectAttributes.addFlashAttribute("message", "成功しました");
return "redirect:/students";  // 別のURLに遷移

// 同一画面返却時：ModelAttributeを使用
model.addAttribute("errorMessage", "エラーが発生しました");
return "students/edit";  // 同じテンプレートを返却

    Q2: メッセージが表示されない場合の確認ポイントは？

    A2: 以下の点を確認してください。

    1. Flash属性の設定: addFlashAttribute() を使用しているか
    2. リダイレクトの実行: return "redirect:..." になっているか
    3. View側の条件分岐: th:if="${successMessage}" が正しいか
    4. セッション設定: Spring Bootのセッション設定が有効か

    Q3: 楽観ロックエラー時はなぜModelAttributeを使うのか？

    A3: フォームの入力値を保持するためです。

    // 楽観ロックエラー時：フォーム値を保持して同じ画面に戻る
catch (OptimisticLockingFailureException e) {
    model.addAttribute("errorMessage", "他のユーザーが更新しました");
    return "students/edit";  // リダイレクトしない
}

    リダイレクトすると入力データが失われてしまうため、エラー時は同一画面に戻します。

    Q4: 国際化対応は必須ですか？

    A4: 今回は基盤作りとして学習しましたが、日本語のみのシステムであれば定数クラスでも十分です。

    将来的な拡張可能性を考慮して、MessageSource の基本的な使い方を理解しておくことをお勧めします。
