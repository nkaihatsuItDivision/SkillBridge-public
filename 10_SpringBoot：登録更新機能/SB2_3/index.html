    📚 Bean Validation技術の理解

    Bean Validationとは？

    Bean Validationは、Javaの標準仕様（JSR 303/349/380）として定められた、オブジェクトのフィールド値を自動検証する技術です。

    📘 Bean Validation技術の理解

    従来の手動バリデーション（問題の多い方法）

    @PostMapping("/students")
public String createStudent(@ModelAttribute StudentData studentData, Model model) {
    // 手動でのバリデーション - 問題だらけ！
    List<String> errors = new ArrayList<>();

    if (studentData.getName() == null || studentData.getName().trim().isEmpty()) {
        errors.add("名前は必須です");
    }
    if (studentData.getName() != null && studentData.getName().length() > 50) {
        errors.add("名前は50文字以下で入力してください");
    }
    if (studentData.getEmail() == null || !studentData.getEmail().contains("@")) {
        errors.add("正しいメールアドレスを入力してください");
    }
    // このような手動チェックが延々と続く...

    if (!errors.isEmpty()) {
        model.addAttribute("errors", errors);
        return "students/form";
    }

    // 保存処理
    return "redirect:/students";
}

    手動バリデーションの問題点： - ❌ コードが冗長: バリデーション処理がControllerを圧迫 - ❌ 保守性が低い: バリデーション条件変更時の修正箇所が多数 - ❌ 一貫性がない: 開発者によってチェック方法が異なる - ❌ テストが困難: バリデーション処理の単体テストが複雑

    Bean Validation（推奨される方法）

    @PostMapping("/students")
public String createStudent(@Valid @ModelAttribute StudentData studentData, 
                          BindingResult bindingResult, Model model) {

    if (bindingResult.hasErrors()) {
        // バリデーションエラーがある場合、フォーム画面に戻る
        return "students/form";
    }

    // バリデーション成功時の処理
    System.out.println("バリデーション成功: " + studentData);
    return "redirect:/students";
}

    Bean Validationの利点： - ✅ シンプルなコード: @Validだけでバリデーション実行 - ✅ 宣言的な設計: アノテーションによる分かりやすい検証ルール - ✅ 再利用性: 同じDTOクラスを他の場所でも活用可能 - ✅ 標準仕様: Java標準のため学習コストが低い

        System.out.println("=== バリデーション実行 ===");
        System.out.println("受信データ: " + studentData);
        System.out.println("エラー有無: " + bindingResult.hasErrors());

        if (bindingResult.hasErrors()) {
            System.out.println("バリデーションエラーが発生しました");
            System.out.println("エラー数: " + bindingResult.getErrorCount());

            System.out.println("エラー詳細:");
            bindingResult.getAllErrors().forEach(error -> 
                System.out.println("- " + error.getDefaultMessage())
            );

            System.out.println("フォーム画面に戻ります");
            return "students/form";
        }

        System.out.println("バリデーション成功!");
        System.out.println("学生データ: " + studentData);

        // Week 9.5でDB保存処理を実装予定
        System.out.println("DB保存処理は Week 9.5 で実装予定");

        return "redirect:/students";
    }
}
