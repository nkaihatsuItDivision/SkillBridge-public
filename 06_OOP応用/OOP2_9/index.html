    🚀 現代Java開発の課題：ボイラープレートコード

    実際のJava開発でよくある問題

    Java開発では、同じようなコードを何度も書く必要があります：

    public class Student {
    private Long id;
    private String name;
    private Integer score;

    // コンストラクタ
    public Student() {}

    public Student(Long id, String name, Integer score) {
        this.id = id;
        this.name = name;
        this.score = score;
    }

    // getter メソッド（3つ）
    public Long getId() { return id; }
    public String getName() { return name; }
    public Integer getScore() { return score; }

    // setter メソッド（3つ）
    public void setId(Long id) { this.id = id; }
    public void setName(String name) { this.name = name; }
    public void setScore(Integer score) { this.score = score; }

    // toString メソッド
    @Override
    public String toString() {
        return "Student{" +
                "id=" + id +
                ", name=&apos;" + name + &apos;\&apos;&apos; +
                ", score=" + score +
                &apos;}&apos;;
    }

    // equals メソッド
    @Override
    public boolean equals(Object o) {
        if (this == o) return true;
        if (o == null || getClass() != o.getClass()) return false;
        Student student = (Student) o;
        return Objects.equals(id, student.id) &&
               Objects.equals(name, student.name) &&
               Objects.equals(score, student.score);
    }

    // hashCode メソッド
    @Override
    public int hashCode() {
        return Objects.hash(id, name, score);
    }
}

    問題点： - 50行超のコードがたった3つのフィールドのために必要 - 同じパターンの繰り返しで開発時間がかかる - フィールド追加時にすべてのメソッドを手動更新する必要 - ヒューマンエラーの発生リスク

    💡 Lombok：革命的なコード自動生成ライブラリ

    Lombokとは

    Lombokは、アノテーションによってJavaのボイラープレートコードを自動生成するライブラリです。

    📘 Lombokの概要図

    Lombokの画期的な効果

    上記の50行のコードが、Lombokを使うとたった7行になります：

    import lombok.Data;

@Data
public class Student {
    private Long id;
    private String name;
    private Integer score;
}

    驚異的な効果： - コード量: 50行 → 7行（86%削減） - 開発時間: 大幅短縮 - 保守性: 自動更新でヒューマンエラー防止 - 可読性: 本質的なコードに集中

    Lombokが自動生成するもの

    @Dataアノテーション1つで以下がすべて自動生成されます：

    • getter/setterメソッド: 全フィールド分
    • toString()メソッド: 見やすい文字列表現
    • equals()メソッド: オブジェクト比較
    • hashCode()メソッド: ハッシュ値計算
    • 引数なしコンストラクタ: デフォルトコンストラクタ

    🔧 主要なLombokアノテーション

    1. @Data：最も使用頻度の高いアノテーション

    実践例：学生クラスの定義

    import lombok.Data;

@Data
public class Student {
    private Long id;
    private String name;
    private Integer score;

// 現代的なサービスクラス
@Slf4j
public class ModernStudentService {

    public Student registerStudent(String name, Integer score, String email) {
        log.info("=== 学生登録処理開始 ===");
        log.debug("入力データ - 名前: {}, 点数: {}, メール: {}", name, score, email);

        try {
            // Builderパターンで学生オブジェクト作成
            Student student = Student.builder()
                .name(name)
                .score(score)
                .email(email)
                .registerDate(LocalDateTime.now())
                .build();

            // バリデーション処理
            validateStudent(student);
            log.info("学生データの検証が完了しました: {}", student.getName());

            // ID自動生成（実際のDBでは自動採番）
            student.setId(System.currentTimeMillis());

            // 登録完了ログ
            log.info("学生登録が正常に完了しました - ID: {}, 名前: {}, 成績: {}", 
                     student.getId(), student.getName(), student.getGrade());

            return student;

        } catch (IllegalArgumentException e) {
            log.warn("入力データに問題があります: {}", e.getMessage());
            throw new RuntimeException("データ検証エラー: " + e.getMessage(), e);
        } catch (Exception e) {
            log.error("学生登録中に予期しないエラーが発生しました: {}", e.getMessage(), e);
            throw new RuntimeException("システムエラーが発生しました", e);
        }
    }

    public CompletableFuture<String> processStudentAsync(Student student) {
        return CompletableFuture.supplyAsync(() -> {
            log.info("非同期学生処理開始: {}", student.getName());

            try {
                // 重い処理のシミュレーション
                Thread.sleep(1000);

                String result = String.format("学生 %s (ID: %s) の処理が完了しました。成績: %s", 
                                               student.getName(), student.getId(), student.getGrade());
                log.info("非同期処理完了: {}", result);
                return result;

            } catch (InterruptedException e) {
                log.error("非同期処理が中断されました: {}", e.getMessage());
                Thread.currentThread().interrupt();
                throw new RuntimeException("処理中断エラー", e);
            } catch (Exception e) {
                log.error("非同期処理でエラーが発生: {}", e.getMessage(), e);
                throw new RuntimeException("非同期処理エラー", e);
            }
        });
    }

    private void validateStudent(Student student) {
        if (student.getName() == null || student.getName().trim().isEmpty()) {
            throw new IllegalArgumentException("学生名が入力されていません");
        }

        if (student.getScore() == null) {
            throw new IllegalArgumentException("点数が入力されていません");
        }

        if (student.getScore() < 0 || student.getScore() > 100) {
            throw new IllegalArgumentException("点数は0-100の範囲で入力してください");
        }

        if (student.getEmail() == null || !student.getEmail().contains("@")) {
            throw new IllegalArgumentException("有効なメールアドレスを入力してください");
        }
    }
}

    A2: 現代的Java開発の標準技術です
