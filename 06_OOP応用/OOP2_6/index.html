    レッスン5.5で習得した抽象クラス技術

    前回のレッスンでは、抽象クラスによる具体化を強制する設計技術を習得しました：

    // レッスン5.5で学んだ抽象クラス
public abstract class Animal {
    protected String name;

    public Animal(String name) {
        this.name = name;
    }

    // 共通の具体的なメソッド
    public void sleep() {
        System.out.println(name + "が眠っています。");
    }

    // 抽象メソッド（子クラスでの実装を強制）
    public abstract void makeSound();
    public abstract void move();
}

    抽象クラスの特徴（復習）

    • 一部抽象、一部具体: 抽象メソッドと具体メソッドの混在
    • 継承の制限: 1つのクラスからのみ継承可能
    • 実装の強制: abstract メソッドは子クラスで必ず実装
    • 共通処理の提供: 共通のフィールドやメソッドを提供

    🤝 契約による設計：インターフェースの本質

    契約の概念

    現実世界では「契約」によって役割と責任が明確に定められます：

    📘 インターフェースの契約概念

    【例：配送サービスの契約】
契約内容：
- 荷物を指定住所まで配送する
- 配送時間は24時間以内
- 配送完了の通知を行う

実際の配送会社：
├── ヤマト運輸（トラックで配送）
├── 佐川急便（バイクで配送）
└── Amazon配送（ドローンで配送）

重要：依頼者は「配送方法」を知る必要がない
      「契約通りに配送される」ことだけを期待

    プログラムでの契約：インターフェース

    // 配送サービスの契約（インターフェース）
public interface DeliveryService {
    // 契約で定められた機能（抽象メソッド）
    void deliver(String address, String item);
    String getDeliveryTime();
    void notifyCompletion();
}

// 各社はこの契約に従って実装（多様な実装方法）
public class YamatoDelivery implements DeliveryService {
    @Override
    public void deliver(String address, String item) {
        System.out.println("ヤマト運輸：トラックで" + item + "を" + address + "に配送します");
    }

    @Override
    public String getDeliveryTime() {
        return "明日の午前中";
    }

    @Override
    public void notifyCompletion() {
        System.out.println("ヤマト運輸：配送完了をメールで通知しました");
    }
}

    インターフェースの重要な特徴

    1. 完全な抽象化: 全てのメソッドが抽象（実装を持たない）
    2. 契約の定義: 「何をするか」のみを定義、「どうやるか」は実装クラスに委ねる
    3. 多重実装: 1つのクラスが複数のインターフェースを実装可能
    4. 統一インターフェース: 実装方法に関係なく同じ方法で使用可能

    📘 抽象クラスとインターフェースの比較

    Comparableインターフェース：並び替え機能

    Java標準ライブラリのインターフェースを実装して、実用的な機能を実現：

    // 学生クラスにComparableインターフェースを実装
public class Student implements Comparable<Student> {
    private String name;
    private int score;
    private String subject;

    public Student(String name, int score, String subject) {
        this.name = name;
        this.score = score;
        this.subject = subject;
    }

    // Comparableインターフェースの契約を実装
    @Override
    public int compareTo(Student other) {
        // 得点の高い順で並び替え
        return Integer.compare(other.score, this.score);
    }

    public void showInfo() {
        System.out.println(name + "（" + subject + "：" + score + "点）");
    }

    // getter メソッド
    public String getName() { return name; }
    public int getScore() { return score; }
    public String getSubject() { return subject; }
}

    完成したインターフェース設計システム 
    // === インターフェース定義 ===
public interface Evaluable {
    double calculateGPA();
    String getGradeLevel();
}

public interface Reportable {
    void generateReport();
    void exportToFile(String filename);
}

public interface Analyzable {
    Map<String, Double> getSubjectAnalysis();
    List<String> getWeakSubjects();
}

// === 実装クラス ===
public class RegularStudent implements Evaluable {
    protected String name;
    protected List<Grade> grades;

    public RegularStudent(String name) {
        this.name = name;
        this.grades = new ArrayList<>();
    }

    @Override
    public double calculateGPA() {
        return grades.stream()
                    .mapToDouble(Grade::getScore)
                    .average()
                    .orElse(0.0);
    }

    @Override
    public String getGradeLevel() {
        double gpa = calculateGPA();
        if (gpa >= 90) return "優秀";
        if (gpa >= 80) return "良好";
        if (gpa >= 70) return "普通";
        return "要改善";
    }
}

public class HonorStudent extends RegularStudent implements Reportable {
    public HonorStudent(String name) {
        super(name);
    }

    @Override
    public void generateReport() {
        System.out.println("=== " + name + "の優等生レポート ===");
        System.out.println("GPA: " + calculateGPA());
        System.out.println("レベル: " + getGradeLevel());
        System.out.println("優等生特典の詳細...");
    }

    @Override
    public void exportToFile(String filename) {
        System.out.println(filename + "に優等生レポートを出力しました");
    }
}

public class ResearchStudent extends RegularStudent 
    implements Reportable, Analyzable {

    public ResearchStudent(String name) {
        super(name);
    }

    // Reportable実装
    @Override
    public void generateReport() {
        System.out.println("=== " + name + "の研究生レポート ===");
        System.out.println("GPA: " + calculateGPA());
        System.out.println("研究進捗: 75%");
        System.out.println("弱点分析: " + getWeakSubjects());
    }

    @Override
    public void exportToFile(String filename) {
        System.out.println(filename + "に研究生レポートを出力しました");
    }

    // Analyzable実装
    @Override
    public Map<String, Double> getSubjectAnalysis() {
        Map<String, Double> analysis = new HashMap<>();
        for (Grade grade : grades) {
            analysis.put(grade.getSubject(), grade.getScore());
        }
        return analysis;
    }

    @Override
    public List<String> getWeakSubjects() {
        return grades.stream()
                    .filter(g -> g.getScore() < 80)
                    .map(Grade::getSubject)
                    .collect(Collectors.toList());
    }
}

// === システム管理クラス ===
public class InterfaceStudentManager {
    public void processAllCapabilities(List<RegularStudent> students) {
        for (RegularStudent student : students) {
            System.out.println("\n=== " + student.name + "の処理 ===");

            // 全員が持つ評価機能
            if (student instanceof Evaluable) {
                Evaluable evaluable = (Evaluable) student;
                System.out.println("GPA: " + evaluable.calculateGPA());
                System.out.println("レベル: " + evaluable.getGradeLevel());
            }

            // 報告機能を持つ学生
            if (student instanceof Reportable) {
                Reportable reportable = (Reportable) student;
                reportable.generateReport();
            }

            // 分析機能を持つ学生
            if (student instanceof Analyzable) {
                Analyzable analyzable = (Analyzable) student;
                System.out.println("弱点科目: " + analyzable.getWeakSubjects());
            }
        }
    }
}

    💡 よくある質問

    Q1: 抽象クラスとインターフェース、どちらを使うべきですか？

    A: 以下の判断基準を参考にしてください：

    インターフェースを選ぶ場合： - 能力・契約を表現したい（can-do関係） - 多重実装が必要 - 完全な抽象化が必要 - 異なるクラス階層でも同じ機能を提供したい

    抽象クラスを選ぶ場合： - 共通の実装を提供したい - is-a関係を表現したい - 部分的な実装で十分 - フィールドや具体メソッドを共有したい

    Q2: なぜ多重継承ではなく多重実装なのですか？

    A: 多重継承はダイヤモンド問題という複雑な問題を引き起こします：

    // 問題のある例（Javaでは許可されない）
class A { void method() { ... } }
class B extends A { void method() { ... } }
class C extends A { void method() { ... } }
class D extends B, C { // どちらのmethod()を使う？ }

    インターフェースによる多重実装では、実装クラスが明確に責任を持つため、この問題を回避できます。

    Q3: interfaceのメソッドはなぜpublicなのですか？

    A: インターフェースは契約なので、外部から使用できる必要があります：

    public interface Drawable {
    // 暗黙的にpublic abstract
    void draw();

    // 以下と同じ意味
    public abstract void draw();
}

    Q4: 実務でよく使われるJava標準インターフェースは？

    A: 以下のインターフェースは頻繁に使用されます：

    // 比較・並び替え
Comparable<T>, Comparator<T>

// 関数型インターフェース（Java 8以降）
Runnable, Supplier<T>, Consumer<T>, Function<T,R>

// コレクション操作
Iterable<T>, Iterator<T>

// 入出力
Serializable, Closeable
