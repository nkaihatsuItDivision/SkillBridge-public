    1. ポリモーフィズムとは何か？

    基本概念：「同じインターフェースで異なる動作」

    ポリモーフィズム（多態性）は、同じメソッド呼び出しでオブジェクトの種類に応じた異なる動作を実現する仕組みです。

    📘 ポリモーフィズムの基本概念

    日常生活での例：リモコン

    リモコンの「電源ボタン」を押すと： - テレビのリモコン: テレビの電源が切り替わる - エアコンのリモコン: エアコンの電源が切り替わる - オーディオのリモコン: オーディオの電源が切り替わる

    同じ操作（ボタンを押す）で、対象によって異なる動作が実行される

    プログラミングでの実現

    // 同じメソッド呼び出し
student1.displayInfo();  // 学部生の表示形式
student2.displayInfo();  // 大学院生の表示形式

// student1とstudent2の実際の型によって、異なるdisplayInfo()が実行される

    これがポリモーフィズムの本質です！

    2. ポリモーフィズムの動作メカニズム

    2.1 is-a関係による型の互換性

    継承関係では「is-a関係」が成立します：

    UndergraduateStudent undergraduate = new UndergraduateStudent(...);
GraduateStudent graduate = new GraduateStudent(...);

// is-a関係により、以下の代入が可能
Student student1 = undergraduate;  // 学部生 is-a 学生
Student student2 = graduate;       // 大学院生 is-a 学生

// 型は Student だが、実際のオブジェクトは異なる

    📘 IS-A関係の説明

    2.2 動的バインディング（実行時メソッド選択）

    public class PolymorphismDemo {
    public static void main(String[] args) {
        // 宣言時の型は Student
        Student student1 = new UndergraduateStudent("田中太郎", 20, "コンピュータ科学");
        Student student2 = new GraduateStudent("佐藤花子", 24, "数学", "機械学習");

        System.out.println("=== 動的バインディングの実演 ===");

        // 実行時に実際のオブジェクトの型が判定され、適切なメソッドが呼び出される
        student1.displayInfo();  // UndergraduateStudent.displayInfo() が実行
        System.out.println();

        student2.displayInfo();  // GraduateStudent.displayInfo() が実行
        System.out.println();

        // 学習活動でも同様
        System.out.println("--- 学習活動の違い ---");
        student1.study();  // 学部生固有の学習活動
        student2.study();  // 大学院生固有の研究活動
    }

    3.2 条件分岐を排除するパターン

    従来のif文による条件分岐を、ポリモーフィズムで置き換えることで、コードがより柔軟で保守性が高くなります：

    public class ConditionalEliminationDemo {

    // ❌ 従来の方法：条件分岐による処理
    public static void processStudentOldWay(Student student) {
        if (student.getStudentType().equals("学部生")) {
            System.out.println("学部生として処理");
            // 学部生固有の処理...
        } else if (student.getStudentType().equals("大学院生")) {
            System.out.println("大学院生として処理");
            // 大学院生固有の処理...
        }
        // 新しい学生種別が追加されるたびにif文を追加する必要がある
    }

    // ✅ ポリモーフィズムによる方法：条件分岐なし
    public static void processStudentNewWay(Student student) {
        student.displayInfo();  // オブジェクトの種類に応じて自動的に適切な処理
        student.study();        // 新しい学生種別が追加されても変更不要
    }

    public static void main(String[] args) {
        Student undergrad = new UndergraduateStudent("田中", 20, "情報科学");
        Student grad = new GraduateStudent("佐藤", 25, "数学", "統計学");

        System.out.println("=== ポリモーフィズムの威力 ===");

        // 同じメソッド呼び出しで異なる処理が実行される
        processStudentNewWay(undergrad);
        System.out.println();
        processStudentNewWay(grad);
    }
}

public class PolymorphicSearchSystem {

    public static List<Student> searchByMajor(List<Student> students, String major) {
        List<Student> results = new ArrayList<>();

        for (Student student : students) {
            if (student.getMajor().equals(major)) {
                results.add(student);
            }
        }

        return results;
    }

    public static List<Student> searchByAgeRange(List<Student> students, int minAge, int maxAge) {
        List<Student> results = new ArrayList<>();

        for (Student student : students) {
            if (student.getAge() >= minAge && student.getAge() <= maxAge) {
                results.add(student);
            }
        }

        return results;
    }

    public static void displaySearchResults(List<Student> results, String searchCriteria) {
        System.out.println("=== 検索結果 ===");
        System.out.println("検索条件: " + searchCriteria);
        System.out.println("該当者数: " + results.size() + "名\n");

        if (results.isEmpty()) {
            System.out.println("該当する学生は見つかりませんでした。");
            return;
        }

        for (int i = 0; i < results.size(); i++) {
            Student student = results.get(i);
            System.out.println((i + 1) + ". " + student.getName() + 
                             " (" + student.getStudentType() + " - " + student.getMajor() + ")");
            student.displayInfo();
            System.out.println();
        }
    }
}

                System.out.println("・評価: 研究が着実に進展しています");
            } else {
                System.out.println("・評価: 論文完成に向けて順調に進んでいます");

    public static void generateOverallReport(List<Student> students) {
        System.out.println("📊 全体統計レポート");
        System.out.println("==========================================");

        int graduateCount = 0;
        double totalGPA = 0.0;

                undergraduateCount++;

                graduateCount++;

        System.out.println("総学生数: " + students.size() + "名");
        System.out.println("学部生: " + undergraduateCount + "名");
        System.out.println("大学院生: " + graduateCount + "名");

            System.out.println("学部生比率: " + String.format("%.1f", (double) undergraduateCount / students.size() * 100) + "%");
        }
        if (graduateCount > 0) {
            System.out.println("大学院生比率: " + String.format("%.1f", (double) graduateCount / students.size() * 100) + "%");
        }

        System.out.println("==========================================\n");
    }

    6. ポリモーフィズムの設計原則

    6.1 開放閉鎖原則（Open-Closed Principle）

    ポリモーフィズムを活用することで、拡張に対して開放的で、変更に対して閉鎖的なシステム設計が可能になります：

    // ❌ 拡張性の低い設計（条件分岐による処理）
public void processStudentOldWay(Student student) {
    if (student.getStudentType().equals("学部生")) {
        // 学部生固有の処理
    } else if (student.getStudentType().equals("大学院生")) {
        // 大学院生固有の処理
    }
    // 新しい学生種別が追加されるたびにこのメソッドを変更する必要がある
}

// ✅ 拡張性の高い設計（ポリモーフィズムによる処理）
public void processStudentNewWay(Student student) {
    student.displayInfo();  // 新しい学生種別が追加されても変更不要
    student.study();        // 各種別のオブジェクトが適切な処理を実行
}

    6.2 依存関係逆転原則（Dependency Inversion Principle）

    具体的なクラスではなく、抽象的なクラス（親クラス）に依存することで、柔軟なシステム設計が実現できます：

    // ✅ 抽象に依存した設計
public class StudentProcessor {

    // 具体的なクラス（UndergraduateStudent, GraduateStudent）ではなく、
    // 抽象的なクラス（Student）に依存
    public void processStudents(List<Student> students) {
        for (Student student : students) {
            student.displayInfo();  // 抽象メソッドの呼び出し
        }
    }
}

    💡 よくある質問

    Q1: ポリモーフィズムと継承の違いは何ですか？

    A: 継承は仕組み、ポリモーフィズムは効果です： - 継承: 親クラスの機能を子クラスが受け継ぐ仕組み - ポリモーフィズム: 継承により実現される「同じインターフェースで異なる動作」という効果

    Q2: instanceof演算子はいつ使うべきですか？

    A: 必要最小限に留めることが重要です： - ✅ 使用推奨: 型固有の情報にアクセスする必要がある場合 - ❌ 使用注意: 条件分岐による処理の振り分け（ポリモーフィズムで解決すべき）

    // ✅ 適切な使用例：型固有の情報アクセス
if (student instanceof UndergraduateStudent) {
    UndergraduateStudent undergrad = (UndergraduateStudent) student;
    int credits = undergrad.getEarnedCredits();  // 学部生特有の情報
}

// ❌ 避けるべき使用例：処理の振り分け
if (student instanceof UndergraduateStudent) {
    // 学部生用の処理
} else if (student instanceof GraduateStudent) {
    // 大学院生用の処理  ← これはポリモーフィズムで解決すべき
}

    Q3: ポリモーフィズムはいつ使うべきですか？

    A: 以下の状況でポリモーフィズムが威力を発揮します： - 複数の似た処理を統一的に扱いたい場合 - 新しい種類の追加が将来的に予想される場合 - 条件分岐が多いコードをシンプル化したい場合 - 統一インターフェースで異なる実装を扱いたい場合

    Q4: ポリモーフィズムを使うデメリットはありますか？

    A: 適切に使用すればデメリットは少ないですが、注意点があります： - 学習コスト: 初心者には理解が難しい概念 - デバッグの複雑さ: 実行時にどのメソッドが呼ばれるかが分かりにくい場合 - 過度な抽象化: 不必要な場面での使用は逆に複雑化を招く

    対策: 明確な目的がある場合にのみ使用し、適切な設計原則に従う

    Q5: メソッドオーバーライドとポリモーフィズムの関係は？

    A: メソッドオーバーライドがポリモーフィズムの基盤です： - メソッドオーバーライド: 子クラスで親クラスのメソッドを再定義する仕組み - ポリモーフィズム: オーバーライドされたメソッドが実行時に動的に選択される現象

    // Student student = new UndergraduateStudent(...);
// student.displayInfo(); ← UndergraduateStudentでオーバーライドされたメソッドが実行される
