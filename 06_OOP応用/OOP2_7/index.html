    1. なぜ例外処理が設計の一部なのか？

    📘 例外処理がシステムに与える影響

    例外処理の重要性

    例外処理は単なる「エラー対応」ではありません。システムの品質と信頼性を決定する設計要素です。

    実例：銀行システムでの例外処理

    // ❌ 例外を考慮しない危険な設計
public class BankAccount {
    private double balance;

    public void withdraw(double amount) {
        balance -= amount;  // 残高不足でも引き落とし？
    }
}

// ✅ 例外を考慮した堅牢な設計
public class SafeBankAccount {
    private double balance;

    public void withdraw(double amount) throws InsufficientFundsException {
        if (amount > balance) {
            throw new InsufficientFundsException("残高不足: " + amount + " > " + balance);
        }
        balance -= amount;
    }
}

    例外安全設計の効果

    1. 信頼性向上：異常状態でもシステムが安全に動作
    2. 保守性向上：エラーの原因が明確で修正しやすい
    3. 使いやすさ向上：呼び出し側で適切な対応が可能
    4. 契約の完全性：正常・異常両方の動作を明確に定義

    2. カスタム例外クラスの設計

    2-1. 基本的なカスタム例外の作成

    業務に特化した例外を作成することで、エラーの原因を明確化できます。

    // 学生管理システム用のカスタム例外
public class StudentNotFoundException extends Exception {
    private String studentId;

    public StudentNotFoundException(String studentId) {
        super("学生が見つかりません: ID=" + studentId);
        this.studentId = studentId;
    }

    public String getStudentId() {
        return studentId;
    }
}

public class InvalidGpaException extends Exception {
    private double invalidGpa;

    public InvalidGpaException(double gpa) {
        super("無効なGPA値です: " + gpa + " (0.0-4.0の範囲で入力してください)");
        this.invalidGpa = gpa;
    }

    public double getInvalidGpa() {
        return invalidGpa;
    }
}

    2-2. カスタム例外を活用したサービスクラス

    import java.util.ArrayList;
import java.util.List;

public class StudentService {
    private List<Student> students = new ArrayList<>();

    // 学生追加（例外安全）
    public void addStudent(String name, int age, double gpa) 
            throws InvalidGpaException {

        // GPAの妥当性チェック
        if (gpa < 0.0 || gpa > 4.0) {
            throw new InvalidGpaException(gpa);
        }

        students.add(new Student(name, age, gpa));
        System.out.println("学生を追加しました: " + name + " (GPA: " + gpa + ")");
    }

    // 学生検索（例外安全）
    public Student findStudent(String studentId) 
            throws StudentNotFoundException {

        for (Student student : students) {
            if (student.getStudentId().equals(studentId)) {
                return student;
            }
        }

        // 見つからない場合は例外をスロー
        throw new StudentNotFoundException(studentId);
    }

    // GPAの更新（複数例外の組み合わせ）
    public void updateGpa(String studentId, double newGpa) 
            throws StudentNotFoundException, InvalidGpaException {

        Student student = findStudent(studentId);  // StudentNotFoundException の可能性

        if (newGpa < 0.0 || newGpa > 4.0) {        // InvalidGpaException の可能性
            throw new InvalidGpaException(newGpa);
        }

        student.setGpa(newGpa);
        System.out.println("GPAを更新しました: " + studentId + " -> " + newGpa);
    }
}

    3. 例外階層の設計

    3-1. 例外の継承関係

    適切な例外階層を設計することで、例外の分類と処理が効率的になります。

    // 基底例外クラス（全ての学生管理例外の親）
public class StudentManagementException extends Exception {
    protected String errorCode;
    protected long timestamp;

    public StudentManagementException(String message, String errorCode) {
        super(message);
        this.errorCode = errorCode;
        this.timestamp = System.currentTimeMillis();
    }

    public String getErrorCode() {
        return errorCode;
    }

    public long getTimestamp() {
        return timestamp;
    }
}

// データ関連例外
public class StudentDataException extends StudentManagementException {
    public StudentDataException(String message) {
        super(message, "DATA_ERROR");
    }

    4. try-catch-finallyの高度活用

    4-1. 基本的なtry-catch-finally

    public class BasicExceptionHandling {

    public static void processStudentData(String fileName) {
        BufferedReader reader = null;

        try {
            // リソースの取得
            reader = new BufferedReader(new FileReader(fileName));

            String line;
            while ((line = reader.readLine()) != null) {
                // データ処理
                processLine(line);
            }

            System.out.println("ファイル処理が完了しました");

        } catch (FileNotFoundException e) {
            System.err.println("ファイルが見つかりません: " + fileName);

        } catch (IOException e) {
            System.err.println("ファイル読み込みエラー: " + e.getMessage());

        } finally {
            // リソースの確実なクリーンアップ
            if (reader != null) {
                try {
                    reader.close();
                    System.out.println("ファイルをクローズしました");
                } catch (IOException e) {
                    System.err.println("ファイルクローズエラー: " + e.getMessage());
                }
            }
        }
    }

    private static void processLine(String line) throws IOException {
        if (line.trim().isEmpty()) {
            throw new IOException("空行は処理できません");
        }
        System.out.println("処理中: " + line);
    }
}

    public class InMemoryStudentRepository implements StudentRepository {
    private Map<String, Student> students = new HashMap<>();

    @Override
    public void addStudent(Student student) 
            throws DuplicateStudentException, InvalidStudentDataException {

        // データ検証
        validateStudentData(student);

        // 重複チェック
        if (students.containsKey(student.getStudentId())) {
            throw new DuplicateStudentException(student.getStudentId());
        }

        // 安全に追加
        students.put(student.getStudentId(), student);
        System.out.println("学生を追加: " + student.getName());
    }

    @Override
    public Student findStudent(String studentId) 
            throws StudentNotFoundException {

        Student student = students.get(studentId);
        if (student == null) {
            throw new StudentNotFoundException(studentId);
        }

        return student;
    }

    @Override
    public void updateStudent(String studentId, Student newData) 
            throws StudentNotFoundException, InvalidStudentDataException {

        // 存在チェック
        Student existing = findStudent(studentId);  // StudentNotFoundException の可能性

        // データ検証
        validateStudentData(newData);  // InvalidStudentDataException の可能性

        // 安全に更新
        students.put(studentId, newData);
        System.out.println("学生情報を更新: " + studentId);
    }

    private void validateStudentData(Student student) 
            throws InvalidStudentDataException {

        if (student == null) {
            throw new InvalidStudentDataException("学生データがnullです");
        }

        if (student.getName() == null || student.getName().trim().isEmpty()) {
            throw new InvalidStudentDataException("学生名が無効です");
        }

        if (student.getAge() < 0 || student.getAge() > 150) {
            throw new InvalidStudentDataException("年齢が無効です: " + student.getAge());
        }

        if (student.getGpa() < 0.0 || student.getGpa() > 4.0) {
            throw new InvalidStudentDataException("GPAが無効です: " + student.getGpa());
        }
    }
}

// 例外安全なサービスクラス
class ExceptionSafeStudentService {
    private Map<String, Student> students = new HashMap<>();
    private static int nextId = 1;

    public String addStudent(String name, int age, double gpa) 
            throws DuplicateStudentException, InvalidStudentDataException {

        // データ検証
        validateName(name);
        validateAge(age);
        validateGpa(gpa);

        // IDの生成と重複チェック
        String studentId = "S" + String.format("%03d", nextId++);
        if (students.containsKey(studentId)) {
            throw new DuplicateStudentException(studentId);
        }

        // 学生の作成と追加
        Student student = new Student(studentId, name, age, gpa);
        students.put(studentId, student);

        System.out.println("学生を追加しました: " + name + " (ID: " + studentId + ")");
        return studentId;
    }

    public Student findStudent(String studentId) 
            throws StudentNotFoundException {

        Student student = students.get(studentId);
        if (student == null) {
            throw new StudentNotFoundException(studentId);
        }

        return student;
    }

    public void updateGpa(String studentId, double newGpa) 
            throws StudentNotFoundException, InvalidStudentDataException {

        Student student = findStudent(studentId);  // 存在チェック
        validateGpa(newGpa);                       // データ検証

        double oldGpa = student.getGpa();
        student.setGpa(newGpa);

        System.out.println("GPA更新: " + studentId + " (" + oldGpa + " -> " + newGpa + ")");
    }

    public void removeStudent(String studentId) 
            throws StudentNotFoundException {

        Student student = findStudent(studentId);  // 存在チェック
        students.remove(studentId);

        System.out.println("学生を削除しました: " + student.getName() + " (ID: " + studentId + ")");
    }

    public void displayAllStudents() {
        if (students.isEmpty()) {
            System.out.println("登録されている学生はいません");
            return;
        }

        System.out.println("=== 全学生一覧 ===");
        for (Student student : students.values()) {
            System.out.println(student);
        }
    }

    // データ検証メソッド群
    private void validateName(String name) throws InvalidStudentDataException {
        if (name == null || name.trim().isEmpty()) {
            throw new InvalidStudentDataException("name", name);
        }
    }

    private void validateAge(int age) throws InvalidStudentDataException {
        if (age < 0 || age > 150) {
            throw new InvalidStudentDataException("age", age);
        }
    }

    private void validateGpa(double gpa) throws InvalidStudentDataException {
        if (gpa < 0.0 || gpa > 4.0) {
            throw new InvalidStudentDataException("gpa", gpa);
        }
    }
}

    💡 よくある質問

    Q1: checked例外とunchecked例外の使い分けは？

    A: 以下の基準で判断しましょう：

    // Checked例外（必ず処理が必要）
// - 回復可能なエラー
// - 呼び出し側で対処すべきエラー
public void saveStudent(Student student) throws IOException {
    // ファイル保存：IOExceptionは回復可能
}

// Unchecked例外（RuntimeException）
// - プログラムのバグ
// - 回復不可能なエラー
public void setAge(int age) {
    if (age < 0) {
        throw new IllegalArgumentException("年齢は負の値にできません");
    }
}

    Q2: 例外のメッセージはどう書けばよいですか？

    A: 以下のポイントを意識しましょう：

    // ❌ 悪い例：原因が分からない
throw new Exception("エラーです");

// ✅ 良い例：具体的で対処法が分かる
throw new InvalidGpaException(
    "GPA値が無効です: " + gpa + " (0.0-4.0の範囲で入力してください)"
);

    Q3: finallyとtry-with-resourcesはどう使い分けますか？

    A: 基本的にはtry-with-resourcesを推奨します：

    // ✅ try-with-resources（推奨）
try (FileWriter writer = new FileWriter("data.txt")) {
    writer.write("データ");
} catch (IOException e) {
    // エラー処理
}

// ❌ finally（非推奨だが、古いライブラリでは必要な場合も）
FileWriter writer = null;
try {
    writer = new FileWriter("data.txt");
    writer.write("データ");
} finally {
    if (writer != null) {
        writer.close();
    }
}

    Q4: 例外をキャッチして何もしないのは問題ですか？

    A: 非常に危険です。必ず適切な処理をしましょう：

    // ❌ 最悪の例：例外を握りつぶす
try {
    riskyOperation();
} catch (Exception e) {
    // 何もしない：バグの温床
}

// ✅ 最低限の対応
try {
    riskyOperation();
} catch (Exception e) {
    System.err.println("エラーが発生しました: " + e.getMessage());
    e.printStackTrace();  // デバッグ情報を出力
}
