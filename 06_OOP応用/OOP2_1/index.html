    1. なぜ継承が必要なのか？

    現実世界の問題

    大学システムを考えてみましょう：

    • 学部生: 卒業要件単位数、指導教員なし
    • 大学院生: 研究テーマ、指導教員あり、論文提出要件

    これらを別々のクラスで作ると…

    // 学部生クラス
public class UndergraduateStudent {
    private String studentId;     // 重複！
    private String name;          // 重複！
    private int age;              // 重複！
    private String major;         // 重複！
    private int requiredCredits;  // 学部生固有

    public void displayInfo() {   // 重複！
        System.out.println("=== 学生情報 ===");
        System.out.println("学生ID: " + studentId);
        System.out.println("氏名: " + name);
        // ...
    }
}

// 大学院生クラス
public class GraduateStudent {
    private String studentId;     // 重複！
    private String name;          // 重複！
    private int age;              // 重複！
    private String major;         // 重複！
    private String researchTopic; // 大学院生固有
    private String supervisor;    // 大学院生固有

    public void displayInfo() {   // 重複！
        System.out.println("=== 学生情報 ===");
        System.out.println("学生ID: " + studentId);
        System.out.println("氏名: " + name);
        // ...
    }
}

    問題点： - コードの重複が大量発生 - 修正が困難（共通部分の変更時、複数箇所修正が必要） - 設計が複雑になる

    📘 継承に関する問題点の図解

    継承による解決

    継承を使うと、共通部分を親クラスにまとめ、固有部分だけを子クラスで追加できます：

    // 親クラス（すべての学生の共通部分）
public class Student {
    protected String studentId;
    protected String name;
    protected int age;
    protected String major;

    public Student(String studentId, String name, int age, String major) {
        this.studentId = studentId;
        this.name = name;
        this.age = age;
        this.major = major;
    }

    public void displayInfo() {
        System.out.println("=== 学生情報 ===");
        System.out.println("学生ID: " + studentId);
        System.out.println("氏名: " + name);
        System.out.println("年齢: " + age);
        System.out.println("専攻: " + major);
    }
}

// 子クラス（学部生固有の機能）
public class UndergraduateStudent extends Student {
    private int requiredCredits;

    public UndergraduateStudent(String studentId, String name, int age, 
                               String major, int requiredCredits) {
        super(studentId, name, age, major);  // 親クラスのコンストラクタ呼び出し
        this.requiredCredits = requiredCredits;
    }

    public void displayRequiredCredits() {
        System.out.println("必要単位数: " + requiredCredits);
    }
}

// 子クラス（大学院生固有の機能）
public class GraduateStudent extends Student {
    private String researchTopic;
    private String supervisor;

    public GraduateStudent(String studentId, String name, int age, 
                          String major, String researchTopic, String supervisor) {
        super(studentId, name, age, major);  // 親クラスのコンストラクタ呼び出し
        this.researchTopic = researchTopic;
        this.supervisor = supervisor;
    }

    public void displayResearchInfo() {
        System.out.println("研究テーマ: " + researchTopic);
        System.out.println("指導教員: " + supervisor);
    }
}

    継承の効果： - コードの重複が解消された - 共通部分の修正が一箇所で済む - 設計がシンプルになった

    📘 継承による解決策の図解

    2. 継承の基本構文

    extendsキーワード

    public class 子クラス名 extends 親クラス名 {
    // 子クラス固有の内容
}

    例： 

    public class UndergraduateStudent extends Student {
    // UndergraduateStudent は Student を継承
    // Student の全てのメンバーを使用可能
}

    protectedアクセス修飾子

    継承では、親クラスのフィールドを子クラスからアクセスできるようにする必要があります：

    public class Student {
    protected String studentId;  // protected: 子クラスからアクセス可能
    protected String name;
    protected int age;
    protected String major;

    // privateだと子クラスからアクセスできない
    // publicだと外部からも自由にアクセスできてしまう
    // protectedは「継承関係にあるクラスのみアクセス可能」
}

    アクセス修飾子の整理： - private: そのクラス内のみ - protected: 同じパッケージ + 継承関係 - public: どこからでも - パッケージプライベート: 同じパッケージ内のみ

    superキーワード

    superは親クラスを参照するキーワードです：

    public class UndergraduateStudent extends Student {
    public UndergraduateStudent(String studentId, String name, int age, 
                               String major, int requiredCredits) {
        // super(): 親クラスのコンストラクタを呼び出し
        super(studentId, name, age, major);
        this.requiredCredits = requiredCredits;
    }

    @Override
    public void displayInfo() {
        // super.メソッド名(): 親クラスのメソッドを呼び出し
        super.displayInfo();
        System.out.println("必要単位数: " + requiredCredits);
    }
}

    3. 実際に継承を使ってみよう

    ステップ1: 基本的な継承クラスの作成

    新しいプロジェクトを作成し、以下のクラスを作成してください：

    親クラス（Student.java）： 

    public class Student {
    protected String studentId;
    protected String name;
    protected int age;
    protected String major;

    public Student(String studentId, String name, int age, String major) {
        this.studentId = studentId;
        this.name = name;
        this.age = age;

    5. 継承設計のベストプラクティス

    Is-A関係の重要性

    継承は「Is-A関係」（〜は〜である）が成り立つ場合に使用します：

    ✅ 適切な継承： - UndergraduateStudent is a Student → ◯ - GraduateStudent is a Student → ◯ - Dog is a Animal → ◯

    ❌ 不適切な継承： - Car is a Engine → ✗（Car has a Engine が正しい） - Student is a School → ✗（Student belongs to School が正しい）

    継承 vs コンポジション

    継承を使うべき場合： - Is-A関係が明確 - 親クラスの機能をそのまま使いたい - 子クラスが親クラスの特殊化

    コンポジション（Has-A関係）を使うべき場合： - Has-A関係（持っている関係） - 機能の組み合わせ - より柔軟な設計が必要

    継承階層の設計原則

    // ✅ 良い継承階層
public class Animal {          // 抽象的
    protected String name;
    public void eat() { ... }
}

public class Mammal extends Animal {  // より具体的
    public void breathe() { ... }
}

public class Dog extends Mammal {     // さらに具体的
    public void bark() { ... }
}

    設計のポイント： - 上位は抽象的、下位は具体的 - 単一責任の原則を維持 - 深すぎる階層は避ける（3-4レベルまで）

    💡 よくある質問

    Q1: privateフィールドは継承されないのですか？

    A1: privateフィールドは継承されますが、子クラスから直接アクセスできません。

    public class Parent {
    private String privateField = "アクセス不可";
    protected String protectedField = "アクセス可能";
}

public class Child extends Parent {
    public void test() {
        // System.out.println(privateField);     // ❌ コンパイルエラー
        System.out.println(protectedField);      // ✅ アクセス可能
    }
}

    Q2: 複数のクラスから継承はできますか？

    A2: Javaでは多重継承はできません。1つの親クラスからのみ継承可能です。

    // ❌ Javaでは不可
public class Child extends Parent1, Parent2 {  // コンパイルエラー
}

// ✅ インターフェースを使った多重実装は可能（Week 5後半で学習）
public class Child extends Parent implements Interface1, Interface2 {
}

    Q3: コンストラクタは継承されますか？

    A3: コンストラクタは継承されません。子クラスで明示的に定義する必要があります。

    public class Parent {
    public Parent(String name) {
        // 親クラスのコンストラクタ
    }
}

public class Child extends Parent {
    public Child(String name) {
        super(name);  // 親クラスのコンストラクタを明示的に呼び出し
    }
}

    Q4: superの呼び出しは必須ですか？

    A4: 親クラスにデフォルトコンストラクタがない場合は必須です。

    public class Parent {
    public Parent(String name) {  // 引数ありコンストラクタのみ
    }
}

public class Child extends Parent {
    public Child(String name) {
        super(name);  // ✅ 必須（これがないとコンパイルエラー）
    }
}
