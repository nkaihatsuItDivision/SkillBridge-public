    1. 抽象クラスとは何か？

    🎯 ステップ1: 抽象クラスの必要性を実感する

    レッスン5.4で作成した学生管理システムを思い出してください。現在のStudentクラスには一つの問題があります。

    現在の設計の問題点

    // レッスン5.4のStudentクラス（問題のある設計）
public class Student {
    protected String studentId;
    protected String name;
    protected String major;

    // 問題：このメソッドは具体的な実装を持っているが...
    public void study() {
        System.out.println(name + "が一般的な学習をしています。");
    }
}

public class UndergraduateStudent extends Student {
    @Override
    public void study() {
        System.out.println(name + "が学部レベルの学習をしています。");
    }
}

    何が問題なのでしょうか？

    // 問題の発生例
public static void main(String[] args) {
    // 直接Studentクラスのインスタンスを作成できてしまう
    Student student = new Student();
    student.setName("一般学生");
    student.study();  // "一般学生が一般的な学習をしています。"

    // でも、実際には「一般学生」は存在しない概念
    // 学生は必ず「学部生」「大学院生」「研究生」のいずれか
}

    実行結果： 

    一般学生が一般的な学習をしています。

    問題の本質： - Studentクラスは「学生の概念」を表すが、「具体的な学生」ではない - しかし、現在の設計では直接インスタンス化できてしまう - study()メソッドの実装も曖昧で、本来は各学生種別で決めるべき

    📘 抽象クラスの概念図

    抽象クラスの概念図：抽象メソッドと具体メソッドの混在、インスタンス化不可の表現

    抽象クラスの特徴

    1. インスタンス化不可: 直接 new Student() はできない

    public abstract void study();

        int score = generateRandomScore(60, 100); // 60-100点
        grades.put(subject, score);
        System.out.println("・" + subject + "の理解度テスト: " + score + "点");
    }

    @Override
    public double calculateGPA() {
        // 学部生のGPA計算ロジック
        if (grades.isEmpty()) return 0.0;

        double total = 0;
        for (int score : grades.values()) {
            total += convertToGPA(score);
        }
        return total / grades.size();
    }

    @Override
    public boolean canTakeAdvancedCourse(String courseName) {
        return currentYear >= 3 && calculateGPA() >= 3.0;
    }

    private double convertToGPA(int score) {
        if (score >= 90) return 4.0;
        if (score >= 80) return 3.0;
        if (score >= 70) return 2.0;
        if (score >= 60) return 1.0;
        return 0.0;
    }
}

        System.out.println(student.getStudentType() + " " + student.getName() + "を登録");
    }

    public void conductBatchLearningSession(String subject, int duration) {
        System.out.println("=== 全学生対象：" + subject + "一斉学習セッション ===\n");

        for (Student student : students) {
            student.conductLearningSession(subject, duration);
        }
    }

    public void generateAdvancedCourseReport() {
        System.out.println("=== 高度科目履修可能性レポート ===");

        for (Student student : students) {
            System.out.println(student.getName() + " (" + student.getStudentType() + "):");
            System.out.println("  GPA: " + String.format("%.2f", student.calculateGPA()));
            System.out.println("  高度科目履修: " + 
                (student.canTakeAdvancedCourse("高度アルゴリズム") ? "可能" : "不可"));
            System.out.println();
        }
    }
}

    💡 よくある質問

    Q1: 抽象クラスと通常のクラスの違いがよく分からない

    A: 以下の比較で理解しましょう：

    項目 通常のクラス 抽象クラス
    インスタンス化 new Student() 可能 new Student() 不可
    目的 具体的なオブジェクト生成 共通機能の提供＋実装強制
    メソッド 全て具体実装 抽象メソッド＋具体メソッドの混在
    継承の役割 コードの再利用 設計の強制＋コードの再利用 
    // 通常のクラス：直接使用可能
Student student = new Student(); // OK

// 抽象クラス：継承してから使用
Student student = new Student(); // エラー！
Student student = new UndergraduateStudent(); // OK

    Q2: いつ抽象クラスを使うべき？

    A: 以下の条件が揃った時に使用を検討してください：

    1. 概念的なクラス: 直接インスタンス化しない概念（動物、図形、学生など）
    2. 共通機能あり: 子クラスで共有する具体的な機能がある
    3. 実装の強制: 子クラスで必ず実装してほしいメソッドがある
    4. テンプレート化: 処理の流れは共通だが、一部は個別実装したい

    Q3: 抽象メソッドを実装し忘れるとどうなる？

    A: コンパイルエラーが発生し、プログラムが実行できません：

    // 抽象メソッドの実装を忘れた場合
public class UndergraduateStudent extends Student {
    // study()メソッドの実装を忘れた！

    @Override
    public String getStudentType() {
        return "学部生";
    }
    // その他のメソッドも実装必要...
}

// エラーメッセージ例：
// "UndergraduateStudent is not abstract and does not override abstract method study() in Student"

    これがコンパイル時の安全性です！ 実行前にミスを発見できます。

    Q4: final修飾子がついたメソッドの意味は？

    A: final がついたメソッドは「オーバーライド禁止」を意味します：

    public abstract class Student {
    // テンプレートメソッド：手順は変更禁止
    public final void conductLearningSession() {
        prepareSession();  // 手順1
        study();          // 手順2（個別実装）
        recordProgress(); // 手順3
    }

    public abstract void study(); // これはオーバーライド必須
}

// 子クラスで以下はエラー
public class UndergraduateStudent extends Student {
    @Override
    public void conductLearningSession() { // エラー！finalメソッドはオーバーライド不可
        // ...
    }
}
