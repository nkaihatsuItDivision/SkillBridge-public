    // 親クラス
public class Student {
    protected String studentId;
    protected String name;
    protected int age;
    protected String major;

    public Student(String studentId, String name, int age, String major) {
        this.studentId = studentId;
        this.name = name;
        this.age = age;
        this.major = major;
    }

    public void displayInfo() {
        System.out.println("=== 学生情報 ===");
        System.out.println("学生ID: " + studentId);
        System.out.println("氏名: " + name);
        System.out.println("年齢: " + age + "歳");
        System.out.println("専攻: " + major);
    }

    public void study() {
        System.out.println(name + "が勉強しています");
    }
}

    // 子クラス1：学部生
public class UndergraduateStudent extends Student {
    private int requiredCredits;

    public UndergraduateStudent(String studentId, String name, int age, 
                               String major, int requiredCredits) {
        super(studentId, name, age, major);
        this.requiredCredits = requiredCredits;
    }

    public void displayRequiredCredits() {
        System.out.println("必要単位数: " + requiredCredits + "単位");
    }
}

    // 子クラス2：大学院生
public class GraduateStudent extends Student {
    private String researchTopic;
    private String supervisor;

    public GraduateStudent(String studentId, String name, int age, String major,
                          String researchTopic, String supervisor) {
        super(studentId, name, age, major);
        this.researchTopic = researchTopic;
        this.supervisor = supervisor;
    }

    public void displayResearchInfo() {
        System.out.println("研究テーマ: " + researchTopic);
        System.out.println("指導教員: " + supervisor);
    }
}

    🔧 メソッドオーバーライドとは

    概念の理解

    メソッドオーバーライドとは、継承した親クラスのメソッドを子クラスで再定義（上書き）する技術です。

    📘 メソッドオーバーライドの概念

    オーバーライドの特徴

    1. 同じメソッド名・引数：親クラスと全く同じシグネチャを持つ
    2. 動作の変更：親クラスの動作を子クラス向けにカスタマイズ
    3. 動的選択：実行時にオブジェクトの実際の型に応じて適切なメソッドが呼ばれる
    4. 透明性：呼び出し側は親クラスのメソッドを呼んでいるつもりでも、実際は子クラスのメソッドが実行される

    @Override注解の重要性

    オーバーライドする際は、必ず@Override注解を付けます：

    public class UndergraduateStudent extends Student {
    // その他のフィールドとコンストラクタ...

    @Override  // この注解が重要！
    public void displayInfo() {
        // 親クラスのメソッドをオーバーライド
    }
}

    @Override注解の利点

    1. コンパイル時チェック：メソッド名やシグネチャの間違いを検出
    2. 意図の明示：このメソッドがオーバーライドであることを明確化
    3. 保守性向上：親クラスの変更時に問題を早期発見
    4. 可読性向上：コードの意図が明確になる

    💡 よくある質問

    Q1: @Override注解を付けないとどうなりますか？

    A: @Override注解がなくても動作しますが、以下の問題が発生する可能性があります：

    public class UndergraduateStudent extends Student {
    // @Override注解なし - 危険！
    public void displayInf() {  // メソッド名を間違えている！
        // 実装...
    }
}

    問題点： - コンパイルエラーにならない：新しいメソッドとして認識される - オーバーライドされない：親クラスのメソッドがそのまま実行される - 意図しない動作：期待した結果が得られない - デバッグ困難：問題の原因が分かりにくい

    @Override注解があれば： 

    @Override
public void displayInf() {  // コンパイルエラー！メソッド名が間違っている
    // 実装...
}

    結論：@Override注解は必須です。安全性・保守性のために必ず付けましょう。

    Q2: super.メソッド名()はいつ使うべきですか？

    A: 以下の場合に使用することをお勧めします：

    // ✅ 良い例：親クラスの機能を活用して拡張
@Override
public void displayInfo() {
    super.displayInfo();  // 親の基本情報表示
    // 子クラス固有の情報を追加
    System.out.println("学生種別: 学部生");
}

// ❌ 避けるべき例：完全に独自実装で重複コード
@Override
public void displayInfo() {
    // 親クラスと同じコードを再実装（重複！）
    System.out.println("学生ID: " + studentId);
    System.out.println("氏名: " + name);
    // ...
    System.out.println("学生種別: 学部生");
}

    使用基準： - 拡張型：親の機能 + 追加機能 → super.メソッド名()を使用 - 置換型：完全に異なる動作 → super.メソッド名()は不要

    Q3: オーバーライドできないメソッドはありますか？

    A: はい、以下のメソッドはオーバーライドできません：

    public class Student {
    // ❌ finalメソッド：オーバーライド不可
    public final void validateStudentId() {
        // この実装は変更不可
    }

    // ❌ staticメソッド：オーバーライド不可（隠蔽は可能）
    public static void getUniversityInfo() {
        // クラスレベルのメソッド
    }

    // ❌ privateメソッド：継承されないのでオーバーライド不可
    private void internalProcess() {
        // 外部からアクセス不可
    }

    // ✅ 通常のメソッド：オーバーライド可能
    public void displayInfo() {
        // オーバーライド可能
    }
}

    Q4: メソッドオーバーライドとメソッドオーバーロードの違いは？

    A: 全く異なる概念です：

    // メソッドオーバーライド：継承関係で親のメソッドを再定義
public class Student {
    public void displayInfo() { /* 親の実装 */ }
}

public class UndergraduateStudent extends Student {
    @Override
    public void displayInfo() { /* 子の実装で上書き */ }
}

// メソッドオーバーロード：同じクラス内で引数違いの同名メソッド
public class Student {
    public void displayInfo() { /* 引数なし */ }
    public void displayInfo(boolean detailed) { /* boolean引数 */ }
    public void displayInfo(String format) { /* String引数 */ }
}
    項目 オーバーライド オーバーロード
    関係 継承関係（親子クラス） 同一クラス内
    シグネチャ 完全に同じ 引数が異なる
    目的 動作の変更・カスタマイズ 利便性向上
    選択時期 実行時（動的） コンパイル時（静的）
