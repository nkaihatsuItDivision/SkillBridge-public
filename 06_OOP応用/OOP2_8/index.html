    1. ログ出力と例外処理の統合

    🔍 なぜログ出力が重要なのか？

    レッスン5.7では例外をキャッチして適切に処理しましたが、実際のシステム運用では、「どこで何が起きたのか」を記録することが必要不可欠です。

    ログなしの例外処理の問題

    try {
    processStudent(studentData);
} catch (StudentValidationException e) {
    // 例外をキャッチしたが、記録がない
    return ResponseStatus.VALIDATION_ERROR;
}

    問題点： - 原因の特定困難：どの学生データで失敗したかわからない - デバッグの効率低下：エラーの再現が困難 - 運用時の問題：システム管理者が状況を把握できない

    ログ出力統合版の例外処理

    Java標準のjava.util.loggingを使用した実装例：

    import java.util.logging.Logger;
import java.util.logging.Level;

public class StudentService {
    private static final Logger logger = Logger.getLogger(StudentService.class.getName());

    public ResponseStatus processStudent(StudentData studentData) {
        try {
            logger.info("学生データ処理開始: ID=" + studentData.getId());

            validateStudentData(studentData);
            saveStudent(studentData);

            logger.info("学生データ処理完了: ID=" + studentData.getId());
            return ResponseStatus.SUCCESS;

        } catch (StudentValidationException e) {
            // ログレベル：WARN（データ品質の問題）
            logger.warning("学生データバリデーションエラー: ID=" + 
                          studentData.getId() + ", エラー内容=" + e.getMessage());
            return ResponseStatus.VALIDATION_ERROR;

        } catch (DatabaseException e) {
            // ログレベル：SEVERE（システムレベルの問題）
            logger.severe("データベース処理エラー: ID=" + 
                         studentData.getId() + ", 詳細=" + e.getMessage());
            return ResponseStatus.SYSTEM_ERROR;

        } catch (Exception e) {
            // ログレベル：SEVERE（予期しないエラー）
            logger.log(Level.SEVERE, "予期しないエラー: ID=" + studentData.getId(), e);
            return ResponseStatus.UNKNOWN_ERROR;
        }
    }
}

    ⚡ CompletableFutureでの例外処理

    現代のJavaでは、非同期処理にCompletableFutureを使用します。しかし、非同期処理では例外処理も複雑になります。

    基本的な非同期例外処理

    public class AsyncStudentService {
    private StudentRepository repository = new StudentRepository();
    private EmailService emailService = new EmailService();

    // 非同期での学生登録処理
    public CompletableFuture<String> registerStudentAsync(StudentData studentData) {
        return CompletableFuture
            .supplyAsync(() -> {
                try {
                    // 1. バリデーション
                    validateStudentData(studentData);
                    return studentData;
                } catch (StudentValidationException e) {
                    throw new RuntimeException("バリデーションエラー", e);
                }
            })
            .thenCompose(validatedData -> {
                // 2. データベース保存（非同期）
                return CompletableFuture.supplyAsync(() -> {
                    try {
                        return repository.save(validatedData);
                    } catch (DataAccessException e) {
                        throw new RuntimeException("保存エラー", e);
                    }
                });
            })
            .thenCompose(savedStudent -> {
                // 3. メール送信（非同期）
                return CompletableFuture.supplyAsync(() -> {
                    try {
                        emailService.sendWelcomeEmail(savedStudent.getEmail());
                        return "登録完了: " + savedStudent.getId();
                    } catch (EmailException e) {
                        // メールエラーは警告ログのみ（登録は成功とする）
                        logger.warning("ウェルカムメール送信失敗: " + e.getMessage());
                        return "登録完了（メール送信失敗）: " + savedStudent.getId();
                    }
                });
            })
            .exceptionally(throwable -> {
                // すべての例外をここでキャッチ
                logger.severe("学生登録処理エラー: " + throwable.getMessage());
                return "登録失敗: " + throwable.getCause().getMessage();
            });
    }
}

    ⚡ 例外処理のコストを理解する

    例外処理のコスト測定

    public class PerformanceMeasurement {
    public static void main(String[] args) {
        measureExceptionCost();
        measureValidationApproaches();
    }

    // 例外処理のコスト測定
    private static void measureExceptionCost() {
        int iterations = 100000;

        // 1. 正常処理の時間測定
        long startTime = System.nanoTime();
        for (int i = 0; i < iterations; i++) {
            normalProcessing(10); // 正常な値
        }
        long normalTime = System.nanoTime() - startTime;

        // 2. 例外処理の時間測定
        startTime = System.nanoTime();
        for (int i = 0; i < iterations; i++) {
            try {
                exceptionProcessing(-1); // 例外が発生する値
            } catch (IllegalArgumentException e) {
                // 例外をキャッチ
            }
        }
        long exceptionTime = System.nanoTime() - startTime;

        System.out.println("=== 例外処理のパフォーマンス比較 ===");
        System.out.println("正常処理: " + (normalTime / 1000000) + "ms");
        System.out.println("例外処理: " + (exceptionTime / 1000000) + "ms");
        System.out.println("例外処理は " + (exceptionTime / normalTime) + "倍遅い");
    }

    private static int normalProcessing(int value) {
        return value * 2; // 正常な処理
    }

    private static int exceptionProcessing(int value) {
        if (value < 0) {
            throw new IllegalArgumentException("負の値は処理できません");
        }
        return value * 2;
    }
}

            // データベース保存をシミュレート
            if (Math.random() < 0.1) { // 10%の確率でエラー
                throw new RuntimeException("データベース接続エラー");
            }

            students.put(student.getId(), student);
            logger.info("学生データ保存完了: ID=" + student.getId());
            return student;

        } catch (Exception e) {
            throw new DataAccessException("学生データ保存エラー: ID=" + student.getId(), e);
        }
    }

    public Student findById(String id) throws DataAccessException {
        try {
            Student student = students.get(id);
            if (student == null) {
                logger.warning("学生が見つかりません: ID=" + id);
                return null;
            }
            return student;
        } catch (Exception e) {
            throw new DataAccessException("学生データ取得エラー: ID=" + id, e);
        }
    }

    public boolean existsById(String id) {
        return students.containsKey(id);
    }
}

// ===== Service層 =====
class StudentService {
    private static final Logger logger = Logger.getLogger(StudentService.class.getName());
    private StudentRepository repository = new StudentRepository();
    private AsyncNotificationService notificationService = new AsyncNotificationService();

    public String registerStudent(String name, int age, String email) throws ServiceException {
        String studentId = "S" + String.format("%03d", new Random().nextInt(1000));

        try {
            logger.info("学生登録処理開始: " + name);

            // 1. 高速バリデーション
            ValidationResult validation = validateStudent(name, age, email);
            if (!validation.isValid()) {
                String errorMsg = "バリデーションエラー: " + String.join(", ", validation.getErrors());
                logger.warning(errorMsg);
                throw new ServiceException(errorMsg, null);
            }

            // 2. 学生データ作成・保存
            Student student = new Student(studentId, name, age, email);
            repository.save(student);

            // 3. 非同期通知送信
            notificationService.sendWelcomeNotification(student)
                .thenAccept(success -> {
                    if (success) {
                        logger.info("通知送信完了: " + studentId);
                    } else {
                        logger.warning("通知送信失敗（登録は完了）: " + studentId);
                    }
                });

            logger.info("学生登録処理完了: " + studentId);
            return studentId;

        } catch (DataAccessException e) {
            logger.severe("学生登録エラー: " + e.getMessage());
            throw new ServiceException("学生登録サービスエラー", e);
        }
    }

    public Student getStudent(String id) throws ServiceException {
        try {
            // 事前チェックによる最適化
            if (id == null || id.trim().isEmpty()) {
                logger.warning("無効な学生ID: " + id);
                return null;
            }

            if (!repository.existsById(id)) {
                logger.info("学生が見つかりません: ID=" + id);
                return null;
            }

            return repository.findById(id);

        } catch (DataAccessException e) {
            logger.severe("学生取得エラー: " + e.getMessage());
            throw new ServiceException("学生取得サービスエラー", e);
        }
    }

    private ValidationResult validateStudent(String name, int age, String email) {
        List<String> errors = new ArrayList<>();

        if (name == null || name.trim().isEmpty()) {
            errors.add("名前は必須です");
        }

        if (age < 0 || age > 150) {
            errors.add("年齢は0-150の範囲で入力してください");
        }

        if (email == null || !email.contains("@")) {
            errors.add("正しいメールアドレスを入力してください");
        }

        return new ValidationResult(errors.isEmpty(), errors);
    }
}

// ===== 非同期通知サービス =====
class AsyncNotificationService {
    private static final Logger logger = Logger.getLogger(AsyncNotificationService.class.getName());

    public CompletableFuture<Boolean> sendWelcomeNotification(Student student) {
        return CompletableFuture
            .supplyAsync(() -> {
                try {
                    // メール送信をシミュレート（1秒）
                    Thread.sleep(1000);

                    // 10%の確率で送信失敗
                    if (Math.random() < 0.1) {
                        throw new RuntimeException("メールサーバーエラー");
                    }

                    logger.info("ウェルカムメール送信完了: " + student.getEmail());
                    return true;

                } catch (Exception e) {
                    throw new RuntimeException("メール送信エラー", e);
                }
            })
            .orTimeout(3, TimeUnit.SECONDS) // 3秒でタイムアウト
            .exceptionally(throwable -> {
                logger.warning("メール送信失敗: " + throwable.getMessage());
                return false;
            });
    }
}

    💡 よくある質問

    Q1: いつログを出力すべきですか？

    A: 以下の場面でログ出力が重要です：

    必須のログ出力： - 処理の開始・完了：重要な処理の境界 - 例外の発生：すべての例外をログに記録 - 外部システム連携：データベースやAPIとの通信

    任意のログ出力： - デバッグ情報：開発時のみ有効 - パフォーマンス情報：処理時間の記録 - ビジネスイベント：重要な業務処理の記録

    // 例：適切なログ出力
public String processOrder(Order order) {
    logger.info("注文処理開始: " + order.getId()); // 開始ログ

    try {
        validateOrder(order);
        String result = saveOrder(order);

        logger.info("注文処理完了: " + order.getId()); // 完了ログ
        return result;

    } catch (ValidationException e) {
        logger.warning("注文バリデーションエラー: " + e.getMessage()); // 例外ログ
        throw e;
    }
}

    Q2: 例外変換はいつ行うべきですか？

    A: レイヤー境界で例外変換を行います：

    例外変換のタイミング： - Repository → Service：技術例外をビジネス例外に変換 - Service → Controller：ビジネス例外をプレゼンテーション例外に変換 - 外部API呼び出し：外部例外を自社例外に変換

    // Repository層での例外変換
catch (SQLException e) {
    throw new DataAccessException("データアクセスエラー", e); // 技術→データ例外
}

// Service層での例外変換
catch (DataAccessException e) {
    throw new BusinessException("ビジネス処理エラー", e); // データ→ビジネス例外
}

    Q3: 非同期処理での例外はどう扱うべきですか？

    A: メイン処理への影響度で判断します：

    クリティカルな処理（同期待ち）： 

    CompletableFuture<String> result = asyncProcess()
    .exceptionally(throwable -> {
        logger.severe("クリティカルエラー: " + throwable.getMessage());
        throw new RuntimeException(throwable); // メイン処理に伝播
    });

return result.get(); // 例外で処理停止

    オプション処理（非同期続行）： 

    asyncNotification()
    .exceptionally(throwable -> {
        logger.warning("通知送信失敗: " + throwable.getMessage());
        return false; // メイン処理に影響なし
    });

    Q4: パフォーマンス最適化はどこまでやるべきですか？

    A: 測定結果に基づいて判断します：

    最適化の優先順位： 1. 頻繁に実行される処理：ループ内、リクエスト処理 2. ユーザー体験に影響する処理：レスポンス時間が重要 3. リソース集約的な処理：メモリやCPUを多用

    // 測定用コード
long startTime = System.nanoTime();
// 処理実行
long endTime = System.nanoTime();
logger.info("処理時間: " + (endTime - startTime) / 1000000 + "ms");

    最適化の基準： - 10ms以下：通常は最適化不要 - 100ms以上：最適化を検討 - 1秒以上：最適化必須
