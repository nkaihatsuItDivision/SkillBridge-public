    レッスン5.2での達成内容

    前回のレッスンでは、メソッドオーバーライドによる動的なカスタマイズ技術を習得しました：

    // 親クラス（Student）
public class Student {
    protected String studentId;
    protected String name;
    protected int age;
    protected String major;

    public void displayInfo() {
        System.out.println("=== 学生情報 ===");
        System.out.println("学生ID: " + studentId);
        System.out.println("氏名: " + name);
        System.out.println("年齢: " + age + "歳");
        System.out.println("専攻: " + major);
    }
}

// 子クラス（UndergraduateStudent）
public class UndergraduateStudent extends Student {
    private int requiredCredits;

    @Override
    public void displayInfo() {
        super.displayInfo();  // 親クラスの機能を活用
        System.out.println("必要単位数: " + requiredCredits + "単位");
        System.out.println("学生種別: 学部生");
    }
}

    レッスン5.2の成果： - ✅ @Override注解の活用：メソッドオーバーライドの明示と安全性確保 - ✅ super.メソッド名()の活用：親クラスの機能を継承しながら拡張 - ✅ 動的メソッド選択：実行時に適切なメソッドが自動選択される仕組み - ✅ 学生種別の専用処理：UndergraduateStudent、GraduateStudentでの固有機能実装

    継承技術による改善： 

    // 継承による学生種別の自然な表現
public abstract class Student { // 共通部分

    💡 ヒント1：抽象親クラスの設計 
    // Week 4のStudentクラスを継承対応に改良
public abstract class Student {
    // Week 4からの継承フィールド
    protected String studentId;
    protected String name;
    protected int age;
    protected String major;
    protected List<Grade> grades;

    // Week 4からの継承コンストラクタ
    public Student(String studentId, String name, int age, String major) {
        this.studentId = studentId;
        this.name = name;
        this.age = age;
        this.major = major;
        this.grades = new ArrayList<>();
    }

    // Week 4からの継承メソッド（共通機能）
    public void addGrade(Grade grade) {
        grades.add(grade);
    }

    public double calculateGPA() {
        if (grades.isEmpty()) return 0.0;
        double total = 0.0;
        for (Grade grade : grades) {
            total += grade.getGradePoint();
        }
        return total / grades.size();
    }

    // 継承技術：子クラスで異なる実装を強制
    public abstract void displayInfo();
    public abstract String getStudentType();
    public abstract void study();

    // getter/setterはWeek 4から継承
    public String getStudentId() { return studentId; }
    public String getName() { return name; }
    public int getAge() { return age; }
    public String getMajor() { return major; }
    public List<Grade> getGrades() { return new ArrayList<>(grades); }
}
    **設計のポイント**： - Week 4の実用的な機能を完全継承 - `protected`で子クラスからのアクセスを許可 - `abstract`メソッドで子クラス固有の機能を強制 - 既存システムとの互換性を維持

    }

    public Student findStudentById(String studentId) {

                      .filter(s -> s.getStudentId().equals(studentId))

                      .orElse(null);
    }

    // 継承技術活用：動的メソッド選択によるレポート生成
    public void generateStudentReport(String studentId) {

            // 動的メソッド選択：実行時に適切なdisplayInfo()が呼ばれる

    // 継承技術活用：すべての学生の統一的な管理
    public void generateAllReports() {

        for (Student student : students) {
            // 動的メソッド選択：各学生の種別に応じた適切な処理
            student.displayInfo();
            System.out.println(); // 区切り
        }
    }

    // 継承技術活用：学生種別ごとの統計
    public void showStudentTypeStatistics() {
        System.out.println("=== 学生種別統計 ===");

        long undergraduateCount = students.stream()
                                         .filter(s -> s instanceof UndergraduateStudent)
                                         .count();
        long graduateCount = students.stream()
                                    .filter(s -> s instanceof GraduateStudent)
                                    .count();

        System.out.println("学部生: " + undergraduateCount + "名");
        System.out.println("大学院生: " + graduateCount + "名");
        System.out.println("総学生数: " + students.size() + "名");

                                           .mapToDouble(Student::calculateGPA)
                                           .average()
                                           .orElse(0.0);

        double graduateAvgGPA = students.stream()
                                      .filter(s -> s instanceof GraduateStudent)
                                      .mapToDouble(Student::calculateGPA)
                                      .average()
                                      .orElse(0.0);

        System.out.println("学部生平均GPA: " + String.format("%.2f", undergraduateAvgGPA));
        System.out.println("大学院生平均GPA: " + String.format("%.2f", graduateAvgGPA));

            System.out.println("\n--- " + student.getName() + "さんの学習活動 ---");
            // 動的メソッド選択：各学生種別の適切なstudy()メソッドが実行される
            student.study();

    }

    public List<Student> findStudentsByMajor(String major) {
        return students.stream()
                      .filter(s -> s.getMajor().equals(major))
                      .collect(Collectors.toList());
    }

    public List<Student> findHighPerformers(double minGPA) {
        return students.stream()
                      .filter(s -> s.calculateGPA() >= minGPA)
                      .sorted((s1, s2) -> Double.compare(s2.calculateGPA(), s1.calculateGPA()))
                      .collect(Collectors.toList());
    }

    💡 よくある質問

    Q1: Week 4のシステムコードをどの程度変更する必要がありますか？

    A: 最小限の変更で統合できます：

    // Week 4での変更点
// 変更前：concrete クラス
public class Student { ... }

// 変更後：abstract クラス（継承対応）
public abstract class Student { 
    // 既存のフィールド・メソッドはそのまま維持
    // 新たにabstractメソッドを追加
    public abstract void displayInfo();
}

// StudentManagementSystemクラス：変更なし
// List<Student>で統一的に扱えるため、既存コードがそのまま動作

    Q2: 継承を使わない場合と比べて何が改善されるのですか？

    A: 以下の点で大幅に改善されます：

    // ❌ 継承なしの場合（Week 4レベル）
public class Student {
    private String studentType; // 文字列で管理

    public void displayInfo() {
        // if文による分岐処理（保守性が低い）
        if ("学部生".equals(studentType)) {
            // 学部生用処理
        } else if ("大学院生".equals(studentType)) {
            // 大学院生用処理
        }
        // 新しい学生種別を追加するたびにif文を追加（保守困難）
    }
}

// ✅ 継承ありの場合（今回習得）
public abstract class Student {
    public abstract void displayInfo(); // 各子クラスで実装を強制
}

public class UndergraduateStudent extends Student {
    @Override
    public void displayInfo() {
        // 学部生専用処理（if文不要）
    }
}
// 新しい学生種別を追加する場合、新しい子クラスを作るだけ（拡張容易）

    Q3: 動的メソッド選択はどのような仕組みで動作するのですか？

    A: Java仮想マシンが実行時に適切なメソッドを選択します：

    // コンパイル時：StudentクラスのdisplayInfo()を呼び出し予定
Student student = new UndergraduateStudent(...);

// 実行時：Javaが実際のオブジェクトの型を確認
// → UndergraduateStudentのdisplayInfo()を呼び出し
student.displayInfo();

// この仕組みにより、以下が可能
List<Student> students = Arrays.asList(
    new UndergraduateStudent(...),  // 実行時にUndergraduateStudentのメソッド
    new GraduateStudent(...)        // 実行時にGraduateStudentのメソッド
);

for (Student student : students) {
    student.displayInfo(); // 各オブジェクトに応じた適切なメソッドが実行される
}

    Q4: protectedアクセス修飾子の意味と使い分けを教えてください

    A: アクセス修飾子の使い分けは以下の通りです：

    public class Student {
    public String publicField;       // どこからでもアクセス可能
    protected String protectedField; // 同パッケージ + 子クラスからアクセス可能
    String packageField;             // 同パッケージからのみアクセス可能
    private String privateField;     // 同クラス内からのみアクセス可能
}

public class UndergraduateStudent extends Student {
    public void someMethod() {
        // ✅ publicField = "OK";      // 親クラスのpublicフィールド
        // ✅ protectedField = "OK";   // 親クラスのprotectedフィールド（継承でアクセス可能）
        // ❌ privateField = "NG";     // 親クラスのprivateフィールド（アクセス不可）
    }
}

    推奨: 継承で使用するフィールドはprotected、外部に公開しないフィールドはprivate
