    1. なぜString型は特別なのか？

    プリミティブ型の動作確認

    まず、レッスン7で学んだプリミティブ型の動作を確認してみましょう：

    public class PrimitiveComparison {
    public static void main(String[] args) {
        int a = 10;
        int b = 10;

        System.out.println("a = " + a);
        System.out.println("b = " + b);
        System.out.println("a == b: " + (a == b));  // 結果は？
    }
}

    実行結果： 

    a = 10
b = 10
a == b: true

    当然の結果ですね。同じ値なのでtrueになります。

    String型で同じことをやってみると...

    今度は、String型で同じようなテストをしてみましょう：

    public class StringComparison {
    public static void main(String[] args) {
        String name1 = "田中";
        String name2 = "田中";

        System.out.println("name1 = " + name1);
        System.out.println("name2 = " + name2);
        System.out.println("name1 == name2: " + (name1 == name2));  // 結果は？
    }
}

    この結果はどうなると思いますか？

    実行結果： 

    name1 = 田中
name2 = 田中
name1 == name2: true

    あれ？同じような結果になりました。でも、実はこれは偶然なんです！

    問題が現れるケース

    以下のコードを実行してみてください：

    public class StringProblem {
    public static void main(String[] args) {
        // ユーザーからの入力をシミュレート
        String input = new String("田中");
        String stored = "田中";

        System.out.println("input = " + input);
        System.out.println("stored = " + stored);
        System.out.println("input == stored: " + (input == stored));  // 結果は？
        System.out.println("input.equals(stored): " + (input.equals(stored)));  // 結果は？
    }
}

    実行結果： 

    input = 田中
stored = 田中
input == stored: false  ← なぜ？
input.equals(stored): true

    なぜこんなことが起こるのでしょうか？

    2. プリミティブ型 vs 参照型の根本的違い

    メモリ上での保存方法の違い

    プリミティブ型と参照型では、メモリ上でのデータの保存方法が全く異なります。

    📘 プリミティブ型と参照型のメモリ比較

    プリミティブ型の場合

    int a = 10;
int b = 10;

    メモリ上の状態： - 変数aは直接値「10」を保存 - 変数bも直接値「10」を保存 - a == bは値同士を比較するのでtrue

    参照型（String）の場合

    String name1 = new String("田中");
String name2 = new String("田中");

    メモリ上の状態： - 変数name1は「田中」が保存されている場所の住所を保存 - 変数name2は別の「田中」が保存されている場所の住所を保存 - name1 == name2は住所同士を比較するのでfalse - name1.equals(name2)は中身を比較するのでtrue

    身近な例で理解しよう

    これを身近な例で考えてみましょう：

    プリミティブ型 = 現金 

    田中さんの財布: 1000円札
佐藤さんの財布: 1000円札
→ 同じ価値なので「等しい」

    参照型 = 住所が書かれた紙 

    田中さんが持つ紙: 「東京都新宿区1-1-1の家」
佐藤さんが持つ紙: 「東京都新宿区2-2-2の家」
→ 紙に書かれた住所は違う（==はfalse）
→ でも、家の中身は同じかもしれない（.equals()でチェック）

    3. ==と.equals()の正しい使い分け

    📘 equalsと==の違い

    基本ルール

    // ❌ 文字列の比較でこれはダメ
if (name1 == name2) {
    System.out.println("同じ名前です");
}

// ✅ 文字列の比較はこれが正解
if (name1.equals(name2)) {
    System.out.println("同じ名前です");
}

    実践的な例：パスワード認証

    public class PasswordCheck {
    public static void main(String[] args) {
        String correctPassword = "secret123";
        String userInput = new String("secret123");  // ユーザー入力をシミュレート

        // ❌ 間違った方法
        if (userInput == correctPassword) {
            System.out.println("ログイン成功（間違った方法）");
        } else {
            System.out.println("ログイン失敗（間違った方法）");
        }

        // ✅ 正しい方法
        if (userInput.equals(correctPassword)) {
            System.out.println("ログイン成功（正しい方法）");
        } else {
            System.out.println("ログイン失敗（正しい方法）");
        }
    }
}

    実行結果： 

    ログイン失敗（間違った方法）
ログイン成功（正しい方法）

    重要なポイント： - ==は住所（参照）を比較 - .equals()は中身（内容）を比較 - 文字列の比較では常に.equals()を使う

    4. null概念の理解

    nullとは？

    nullは「何も参照していない」ことを表す特別な値です。

    String name = null;  // 「住所が書かれていない状態」

    📘 nullの概念図

    nullの安全な扱い方

    public class NullHandling {
    public static void main(String[] args) {
        String name = null;

        // ❌ 危険：NullPointerExceptionが発生
        // System.out.println(name.length());

        // ✅ 安全：nullチェックを先に行う
        if (name != null) {
            System.out.println("名前の長さ: " + name.length());
        } else {
            System.out.println("名前が設定されていません");
        }

        // ✅ さらに安全：メソッドでnullチェック
        printNameLength(name);
        printNameLength("田中太郎");
    }

    // nullに対応した安全なメソッド
    public static void printNameLength(String name) {
        if (name == null) {
            System.out.println("名前がnullです");
        } else if (name.equals("")) {
            System.out.println("名前が空文字列です");
        } else {
            System.out.println(name + "の文字数: " + name.length());
        }
    }
}

名前がnullです

    5. String不変性の概念

    Stringは変更できない

    📘 Stringの不変性の図解

    実は、String型のデータは一度作成されると変更できません。

    public class StringImmutability {
    public static void main(String[] args) {
        String message = "Hello";
        System.out.println("元の文字列: " + message);

        // 文字列を「変更」する
        message = message + " World";
        System.out.println("変更後: " + message);

        // 実際には新しいStringオブジェクトが作成される
    }
}

    何が起こっているか： 1. "Hello"という文字列オブジェクトが作成される 2. message + " World"で新しい"Hello World"オブジェクトが作成される 3. 元の"Hello"オブジェクトはそのまま残る（ガベージコレクションされるまで）

    効率的な文字列操作

    大量の文字列操作が必要な場合は、StringBuilderを使うと効率的です：

    public class EfficientString {
    public static void main(String[] args) {
        // ❌ 非効率な方法（大量の処理の場合）
        String result1 = "";
        for (int i = 1; i <= 5; i++) {
            result1 = result1 + i + " ";  // 毎回新しいオブジェクト作成
        }
        System.out.println("非効率な方法: " + result1);

        // ✅ 効率的な方法
        StringBuilder sb = new StringBuilder();
        for (int i = 1; i <= 5; i++) {
            sb.append(i).append(" ");  // 同じオブジェクトに追加
        }
        String result2 = sb.toString();
        System.out.println("効率的な方法: " + result2);
    }
}

    public class NameValidator {
    public static void main(String[] args) {
        // テストケース
        System.out.println("テスト結果:");
        System.out.println("田中太郎: " + validateName("田中太郎"));
        System.out.println("null: " + validateName(null));
        System.out.println("空文字: " + validateName(""));
        System.out.println("空白のみ: " + validateName("   "));
        System.out.println("長すぎる名前: " + validateName("これは非常に長い名前でテストケースとして使用します"));
        System.out.println("正常な名前: " + validateName("山田花子"));
    }

    /**
     * 名前の有効性を検証するメソッド
     * @param name 検証する名前
     * @return 有効ならtrue、無効ならfalse
     */
    public static boolean validateName(String name) {
        // nullチェック
        if (name == null) {
            return false;
        }

        // 空文字列チェック
        if (name.equals("")) {
            return false;
        }

        // 空白文字だけかどうかチェック
        if (name.trim().length() == 0) {
            return false;
        }

        // 長さチェック（20文字以下）
        if (name.length() > 20) {
            return false;
        }

        // すべての条件をクリア
        return true;
    }
}

    課題2：文字列比較の安全なメソッド

    まずは自分で考えてみよう！

    以下の要件でメソッドを作成してください：

    要件： 1. safeEqualsメソッドを作成 2. 引数：String型を2つ 3. 戻り値：boolean型 4. 動作：どちらかがnullでも安全に比較できる

    💡 ヒント1: 両方がnullの場合 

    if (str1 == null && str2 == null) {
    return true;  // 両方nullなら等しい
}

    💡 ヒント2: どちらか一方がnullの場合 

    if (str1 == null || str2 == null) {
    return false;  // 片方だけnullなら異なる
}

    💡 ヒント3: 両方ともnullでない場合 

    return str1.equals(str2);

    完成例

    public class SafeStringComparison {
    public static void main(String[] args) {
        // テストケース
        System.out.println("テスト結果:");
        System.out.println("同じ文字列: " + safeEquals("Hello", "Hello"));
        System.out.println("異なる文字列: " + safeEquals("Hello", "World"));
        System.out.println("片方null: " + safeEquals("Hello", null));
        System.out.println("両方null: " + safeEquals(null, null));
        System.out.println("逆順null: " + safeEquals(null, "Hello"));
    }

    /**
     * nullセーフな文字列比較メソッド
     * @param str1 比較する文字列1
     * @param str2 比較する文字列2
     * @return 等しければtrue、異なればfalse
     */
    public static boolean safeEquals(String str1, String str2) {
        // 両方nullの場合は等しい
        if (str1 == null && str2 == null) {
            return true;
        }

        // どちらか一方がnullの場合は異なる
        if (str1 == null || str2 == null) {
            return false;
        }

        // 両方ともnullでない場合は通常の比較
        return str1.equals(str2);
    }
}

    💡 よくある質問

    Q1: なぜStringだけ特別な扱いなのですか？

    A: Stringは参照型だからです。Javaには以下の2種類のデータ型があります： - プリミティブ型：値を直接保存（int, double, boolean等） - 参照型：オブジェクトの場所（住所）を保存（String, 配列, 自作クラス等）

    Q2: 文字列リテラル（"Hello"）とnew String("Hello")の違いは？

    A: 

    String s1 = "Hello";          // 文字列プールから取得（効率的）
String s2 = "Hello";          // 同じオブジェクトを参照
String s3 = new String("Hello"); // 新しいオブジェクトを強制作成

System.out.println(s1 == s2); // true（同じオブジェクト）
System.out.println(s1 == s3); // false（異なるオブジェクト）

    Q3: nullと空文字列("")の違いは？

    A: - null: 「何も参照していない」状態 - 空文字列(""): 「長さ0の文字列オブジェクト」を参照している状態

    String s1 = null;    // 何も参照していない
String s2 = "";      // 長さ0の文字列を参照

// s1.length(); // ❌ NullPointerException
// s2.length(); // ✅ 0が返る

    Q4: StringBuilder以外に効率的な方法はありますか？

    A: Java 8以降ではString.join()やStream.collect()も便利です： 

    List<String> words = Arrays.asList("Hello", "World", "Java");
String result = String.join(" ", words); // "Hello World Java"
