    1. なぜ高度な演算子が必要なのか？

    問題：複雑な条件判定が書きにくい

    例えば、以下のような条件を判定したい場合を考えてみましょう：

    「年齢が18歳以上65歳未満で、かつ運転免許を持っている人は運転できる」

    今までの知識では、こう書く必要がありました：

    int age = 25;
boolean hasLicense = true;

if (age >= 18) {
    if (age < 65) {
        if (hasLicense) {
            System.out.println("運転できます");
        } else {
            System.out.println("免許が必要です");
        }
    } else {
        System.out.println("年齢制限があります");
    }
} else {
    System.out.println("年齢制限があります");
}

    問題点： - ネストが深くて読みにくい - 同じメッセージを複数箇所に書いている - 条件が複雑になると管理が困難

    解決策：論理演算子による簡潔な記述

    新しい演算子を使うと、このように書けます：

    int age = 25;
boolean hasLicense = true;

if (age >= 18 && age < 65 && hasLicense) {
    System.out.println("運転できます");
} else {
    System.out.println("運転できません");
}

    たった5行で同じ判定ができました！

    📘 論理演算子による条件判定の改善

    2. 論理演算子（&&, ||, !）

    論理演算子とは？

    論理演算子は、複数の条件を組み合わせるための演算子です。日常会話の「かつ」「または」「ではない」に対応します。

    演算子 意味 例 結果
    && かつ（AND） true && false false
    || または（OR） true || false true
    ! ではない（NOT） !true false

    &&演算子（AND：かつ）

    両方の条件がtrueの時だけtrueになります。

    public class LogicalAnd {
    public static void main(String[] args) {
        int age = 22;
        boolean hasJob = true;

        // 年齢が20歳以上で、かつ仕事がある場合
        if (age >= 20 && hasJob) {
            System.out.println("クレジットカードが作れます");
        } else {
            System.out.println("条件を満たしていません");
        }

        // 条件の詳細確認
        System.out.println("年齢条件: " + (age >= 20));  // true
        System.out.println("仕事条件: " + hasJob);        // true
        System.out.println("総合判定: " + (age >= 20 && hasJob)); // true
    }
}

    実行結果： 

    クレジットカードが作れます
年齢条件: true
仕事条件: true
総合判定: true

    ||演算子（OR：または）

    どちらか一方がtrueならtrue**になります。

    public class LogicalOr {
    public static void main(String[] args) {
        boolean isWeekend = false;
        boolean isHoliday = true;

        // 週末または祝日の場合
        if (isWeekend || isHoliday) {
            System.out.println("お出かけしましょう！");
        } else {
            System.out.println("お仕事頑張りましょう");
        }

        // 条件の詳細確認
        System.out.println("週末: " + isWeekend);           // false
        System.out.println("祝日: " + isHoliday);           // true  
        System.out.println("お休み: " + (isWeekend || isHoliday)); // true
    }
}

    実行結果： 

    お出かけしましょう！
週末: false
祝日: true
お休み: true

    !演算子（NOT：ではない）

    条件を反転させます。trueをfalseに、falseをtrueにします。

    public class LogicalNot {
    public static void main(String[] args) {
        boolean isRaining = false;

        // 雨が降っていない場合
        if (!isRaining) {
            System.out.println("散歩に行きましょう");
        } else {
            System.out.println("家にいましょう");
        }

        // NOT演算子の動作確認
        System.out.println("雨が降っている: " + isRaining);     // false
        System.out.println("雨が降っていない: " + !isRaining);  // true
    }
}

    短絡評価（ショートサーキット）

    論理演算子には短絡評価という重要な特徴があります：

    public class ShortCircuit {
    public static void main(String[] args) {
        int a = 5;
        int b = 0;

        // &&の短絡評価：左がfalseなら右は評価されない
        if (b != 0 && a / b > 2) {  // b != 0 がfalseなので、a / b は実行されない
            System.out.println("条件を満たしています");
        } else {
            System.out.println("条件を満たしていません");
        }

        // ||の短絡評価：左がtrueなら右は評価されない
        if (a > 0 || a / b > 2) {  // a > 0 がtrueなので、a / b は実行されない
            System.out.println("正の数です");
        }
    }
}

    これにより、ゼロ除算エラーを回避できます！

    3. 複合代入演算子（+=, -=, *=, /=, %=）

    複合代入演算子とは？

    複合代入演算子は、演算と代入を同時に行う演算子です。コードが簡潔になります。

    演算子 従来の書き方 複合代入
    += x = x + 5 x += 5
    -= x = x - 3 x -= 3
    *= x = x * 2 x *= 2
    /= x = x / 4 x /= 4
    %= x = x % 3 x %= 3

    実際の使用例

    public class CompoundAssignment {
    public static void main(String[] args) {
        int score = 80;
        int bonus = 10;
        int penalty = 5;

        System.out.println("初期点数: " + score);

        // ボーナス点を加算
        score += bonus;  // score = score + bonus と同じ
        System.out.println("ボーナス後: " + score);

        // ペナルティを減算
        score -= penalty;  // score = score - penalty と同じ
        System.out.println("ペナルティ後: " + score);

        // 2倍ボーナス
        score *= 2;  // score = score * 2 と同じ
        System.out.println("2倍ボーナス後: " + score);

        // 切り捨て処理
        score /= 10;  // score = score / 10 と同じ
        System.out.println("10で割った結果: " + score);
    }
}

    実行結果： 

    初期点数: 80
ボーナス後: 90
ペナルティ後: 85
2倍ボーナス後: 170
10で割った結果: 17

    文字列との組み合わせ

    public class StringCompound {
    public static void main(String[] args) {
        String message = "Java";

        message += "プログラミング";  // message = message + "プログラミング"
        System.out.println(message);  // Javaプログラミング

        message += "は楽しい！";
        System.out.println(message);  // Javaプログラミングは楽しい！
    }
}

    4. インクリメント・デクリメント演算子（++, --）

    インクリメント・デクリメント演算子とは？

    インクリメント（++）：変数の値を1増やす デクリメント（--）：変数の値を1減らす

    int count = 5;
count++;  // count = count + 1 と同じ（countは6になる）
count--;  // count = count - 1 と同じ（countは5になる）

    前置と後置の違い

    重要なポイントは、++や--を変数の前に置くか後に置くかで動作が変わることです：

    public class IncrementDecrement {
    public static void main(String[] args) {
        int a = 5;
        int b = 5;

        // 前置インクリメント：先に値を増やしてから使用
        int result1 = ++a;  // aを先に6にしてから、result1に6を代入
        System.out.println("前置: a=" + a + ", result1=" + result1);

        // 後置インクリメント：値を使用してから増やす
        int result2 = b++;  // bの現在値5をresult2に代入してから、bを6にする
        System.out.println("後置: b=" + b + ", result2=" + result2);
    }
}

    実行結果： 

    前置: a=6, result1=6
後置: b=6, result2=5

    for文での活用（復習）

    レッスン5で学んだfor文でも、この演算子を使っていました：

    for (int i = 1; i <= 5; i++) {  // i++は後置インクリメント
    System.out.println(i + "回目");
}

    5. 三項演算子（? :）

    三項演算子とは？

    三項演算子は、if-else文を1行で表現できる演算子です：

    結果 = 条件 ? true の場合の値 : false の場合の値;

    if-else文との比較

    従来のif-else文： 

    int score = 75;
String result;

if (score >= 60) {
    result = "合格";
} else {
    result = "不合格";
}
System.out.println(result);

    三項演算子： 

    int score = 75;
String result = (score >= 60) ? "合格" : "不合格";
System.out.println(result);

    同じ結果が1行で書けました！

    実用的な例

    public class TernaryOperator {
    public static void main(String[] args) {
        // 年齢による料金計算
        int age = 17;
        int price = (age >= 18) ? 1000 : 500;  // 大人1000円、子供500円
        System.out.println("料金: " + price + "円");

        // 偶数・奇数判定
        int number = 7;
        String type = (number % 2 == 0) ? "偶数" : "奇数";
        System.out.println(number + "は" + type + "です");

        // 最大値を求める
        int a = 15, b = 23;
        int max = (a > b) ? a : b;
        System.out.println("最大値: " + max);

        // 三項演算子を変数代入以外でも使用可能
        System.out.println("天気: " + (true ? "晴れ" : "雨"));
    }
}

    実行結果： 

    料金: 500円
7は奇数です
最大値: 23
天気: 晴れ

    6. 演算子の優先順位完全版

    なぜ優先順位が重要？

    複雑な式では、どの演算子が先に計算されるかで結果が変わります：

    boolean result = 5 + 3 * 2 > 10 && true || false;
// この式はどう計算される？

    主要な演算子の優先順位（高い順）

    📘 演算子の優先順位完全版

    優先順位 演算子 例
    1 括弧 (a + b)
    2 単項演算子 ++a, --a, !flag
    3 乗除余 *, /, %
    4 加減 +, -
    5 比較 <, >, <=, >=
    6 等価 ==, !=
    7 論理AND &&
    8 論理OR ||
    9 三項演算子 ? :
    10 代入 =, +=, -=

    実際の計算例

    public class OperatorPrecedence {
    public static void main(String[] args) {
        // 複雑な式の計算順序を確認
        int a = 5, b = 3, c = 2;
        boolean flag = true;

        // 式1: 5 + 3 * 2 > 10 && true
        boolean result1 = a + b * c > 10 && flag;
        System.out.println("計算過程:");
        System.out.println("5 + 3 * 2 = " + (a + b * c));      // 乗算が先：11
        System.out.println("11 > 10 = " + (a + b * c > 10));   // 比較：true
        System.out.println("true && true = " + result1);        // 論理AND：true

        // 式2: 三項演算子との組み合わせ
        String result2 = a > b && flag ? "条件成立" : "条件不成立";
        System.out.println("結果: " + result2);

        // 括弧で優先順位を明確化
        boolean result3 = (a + b) * c > (10 + 5) && (!flag || true);
        System.out.println("括弧使用: " + result3);
    }
}

    迷った時の対策

    括弧を積極的に使いましょう！

    // わかりにくい
boolean result = a + b * c > d && flag || !otherFlag;

// わかりやすい
boolean result = ((a + (b * c)) > d && flag) || (!otherFlag);

    7. 実践課題：学生評価システム

    課題内容

    学生の総合評価を行うプログラムを作成してください。複数の条件を組み合わせて判定を行います。

    まずは自分で考えてみよう！

    要件： 1. 学生の情報（名前、数学、英語、出席率）を設定 2. 以下の条件で総合評価を判定： - 優秀：両科目80点以上かつ出席率90%以上 - 良好：両科目60点以上かつ出席率80%以上 - 要努力：それ以外 3. 各科目の合格/不合格も表示（60点以上で合格） 4. 出席率による特別メッセージも表示（95%以上で皆勤賞）

    💡 ヒント1: 必要な変数 

    String name = "田中太郎";
int mathScore = 85;
int englishScore = 72;
double attendanceRate = 92.5;  // パーセント

    💡 ヒント2: 使用する演算子 - 論理演算子（&&, ||）：複数条件の組み合わせ - 三項演算子（? :）：合格/不合格の簡潔な判定 - 比較演算子（>=）：点数・出席率の比較

    💡 ヒント3: 判定の構造 

    // 総合評価の条件組み合わせ

boolean isGood = (mathScore >= 60 && englishScore >= 60 && attendanceRate >= 80);

// 三項演算子での評価決定

    完成例

    public class StudentEvaluation {
    public static void main(String[] args) {
        // 学生情報
        String name = "田中太郎";
        int mathScore = 85;
        int englishScore = 72;
        double attendanceRate = 92.5;

        // 各科目の合格判定（三項演算子使用）
        String mathResult = (mathScore >= 60) ? "合格" : "不合格";
        String englishResult = (englishScore >= 60) ? "合格" : "不合格";

        // 総合評価判定（論理演算子使用）
        boolean isExcellent = (mathScore >= 80 && englishScore >= 80 && attendanceRate >= 90);
        boolean isGood = (mathScore >= 60 && englishScore >= 60 && attendanceRate >= 80);

        // 三項演算子での評価決定
        String evaluation = isExcellent ? "優秀" : (isGood ? "良好" : "要努力");

        // 皆勤賞判定
        String attendanceMessage;
        if (attendanceRate >= 95) {
            attendanceMessage = "皆勤賞おめでとうございます！";
        } else if (attendanceRate >= 90) {
            attendanceMessage = "惜しい！皆勤賞まであと少し";
        } else {
            attendanceMessage = "出席率向上を目指しましょう";
        }

        // 結果表示
        System.out.println("=== 学生総合評価システム ===");
        System.out.println("学生名: " + name);
        System.out.println("数学: " + mathScore + "点 (" + mathResult + ")");
        System.out.println("英語: " + englishScore + "点 (" + englishResult + ")");
        System.out.println("出席率: " + attendanceRate + "%");
        System.out.println();
        System.out.println("総合評価: " + evaluation);
        System.out.println("特別メッセージ: " + attendanceMessage);
    }
}

    💡 よくある質問

    Q1: &&と||はどちらを先に使いますか？

    A: 論理演算子にも優先順位があります： - &&（AND）が先に評価される - ||（OR）が後に評価される

    // この式は (a && b) || c として評価される
boolean result = a && b || c;

// 明確にするには括弧を使用
boolean result = (a && b) || c;

    Q2: 三項演算子は複数回使えますか？

    A: 使えますが、読みにくくなるので注意が必要です：

    // 可能だが読みにくい
String result = score >= 90 ? "優秀" : score >= 60 ? "合格" : "不合格";

// こちらの方が読みやすい
String result;
if (score >= 90) {
    result = "優秀";
} else if (score >= 60) {
    result = "合格";
} else {
    result = "不合格";
}

    Q3: ++iとi++の使い分けはありますか？

    A: for文では通常どちらでも問題ありませんが、代入で使用する場合は注意が必要です：

    // for文では通常どちらでもOK
for (int i = 0; i < 10; i++) { }   // 一般的
for (int i = 0; i < 10; ++i) { }   // これも正しい

// 代入で使用する場合は違いがある
int a = 5;
int b = a++;  // bは5、aは6
int c = ++a;  // cは7、aは7

    Q4: 複合代入演算子はすべての演算子で使えますか？

    A: 主要な算術演算子では使えます：

    x += 5;   // 加算代入
x -= 3;   // 減算代入  
x *= 2;   // 乗算代入
x /= 4;   // 除算代入
x %= 3;   // 余り代入

// ただし比較演算子や論理演算子では使えません
// x >= 5;  // エラー！
// x &&= true;  // エラー！

    Q5: 演算子の優先順位を全て覚える必要がありますか？

    A: 全て覚える必要はありません。重要なのは： 1. 基本的な優先順位（算術 → 比較 → 論理）を理解する 2. 迷った時は括弧を使う習慣をつける 3. 可読性を重視したコードを書く
