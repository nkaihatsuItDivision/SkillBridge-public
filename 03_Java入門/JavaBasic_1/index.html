    📚 1. Eclipse（Pleiades）の起動と基本操作

    Eclipseとは？

    Eclipse（イクリプス） は、Javaプログラムを効率的に開発するためのツールです。テキストエディターのようにコードを書くだけでなく、以下の便利な機能があります：

    • 自動補完: コードを途中まで打つと、続きを提案してくれる
    • エラー検出: 間違いがあると赤い線で教えてくれる
    • 実行機能: ボタン一つでプログラムを実行できる
    • デバッグ機能: プログラムの動作を詳しく調べられる

    Eclipseの起動

    1. デスクトップの 「Eclipse」 アイコンをダブルクリック
    2. ワークスペースの選択画面が表示されます

    📘 Eclipseワークスペース選択画面

    ワークスペースとは、あなたのJavaプロジェクト（作品）を保存する場所のことです。

    1. 初回は既定の場所でOKなので、「起動」 をクリック

    Eclipse画面の見方

    Eclipseが起動すると、以下のような画面が表示されます：

    📘 Eclipseのメイン画面構成

    重要な部分： - ①パッケージエクスプローラー: プロジェクト（作品フォルダ）一覧 - ②エディター: コードを書く場所 - ③コンソール: プログラムの実行結果が表示される場所

    📁 2. 最初のJavaプロジェクト作成

    プロジェクトとは？

    プロジェクトは、一つのアプリケーション（例：計算機、ゲーム、管理システム）に関連するファイルをまとめる「フォルダ」のようなものです。

    プロジェクト作成手順

    ステップ1: 新規プロジェクト作成開始

    1. 「ファイル」 → 「新規」 → 「プロジェクト」 をクリック

    📘 Eclipse新規プロジェクト作成ステップ1

    ステップ2: Javaプロジェクトを選択

    1. 「Java」 フォルダを展開
    2. 「Javaプロジェクト」 を選択
    3. 「次へ」 をクリック

    📘 Eclipse新規プロジェクト作成ステップ2

    ステップ3: プロジェクト名の設定

    1. プロジェクト名 に HelloWorldProject と入力
    2. 「完了」 をクリック

    重要なポイント： - クラス名は大文字で始める - ファイル名とクラス名は同じにする - main メソッドは「プログラムの開始地点」

    自動生成されたコード

    以下のコードが自動的に作成されます：

    public class HelloWorld {

    public static void main(String[] args) {
        // TODO 自動生成されたメソッド・スタブ

    }

}

    コードの意味を理解しよう

    public class HelloWorld {              // ← クラス（プログラムの入れ物）の定義
    public static void main(String[] args) {  // ← メインメソッド（開始地点）
        // ここにプログラムの処理を書く
    }                                  // ← メインメソッドの終わり
}                                      // ← クラスの終わり

    各部分の説明：

    部分 意味 例え
    public class HelloWorld 「HelloWorld」という名前の公開クラス 本のタイトル
    { と } クラスやメソッドの範囲を示す 本の表紙と裏表紙
    public static void main プログラムの開始地点 本の第1章
    String[] args プログラム実行時の引数（今は使わない） 付録（今は気にしない）
    // コメント（メモ書き、実行されない） 本の注釈

    👋 4. Hello Worldプログラムの作成

    コードの入力

    自動生成されたコードの中身を編集して、実際にプログラムを書いてみましょう。

    ステップ1: コメント行を削除

    // TODO 自動生成されたメソッド・スタブ
    この行を削除します。

    ステップ2: Hello World表示のコードを追加

    main メソッドの { と } の間に以下を入力：

    System.out.println("Hello, World!");

    完成したコード：

    public class HelloWorld {

    public static void main(String[] args) {
        System.out.println("Hello, World!");
    }

}

    コードの説明

    System.out.println("Hello, World!");

    各部分の意味： - System.out.println : 画面に文字を表示する命令 - "Hello, World!" : 表示したい文字（文字列は"で囲む） - ; : 命令の終わりを示す（必須）

    重要なルール： - Javaでは各命令の最後に ;（セミコロン）が必要 - 文字列は " で囲む - 大文字・小文字は区別される（System ≠ system）

    ⚙️ 6. コンパイルと実行の仕組み

    プログラムが動く仕組み

    Javaプログラムが実行されるまでには、以下のステップがあります：

    画像

    ステップ1: ソースコード作成

    public class HelloWorld {
    public static void main(String[] args) {
        System.out.println("Hello, World!");
    }
}
    人間が読める形のプログラム（HelloWorld.java）

    ステップ2: コンパイル

    Eclipseが自動的にコンパイル（翻訳）を行い、コンピューターが理解できる形（HelloWorld.class）に変換します。

    ステップ3: 実行

    コンピューターが.classファイルを読み込んで、プログラムを実行します。

    Eclipseの便利な点

    • 自動コンパイル: ファイルを保存すると自動的にコンパイルされる
    • エラー表示: コンパイルに失敗すると、赤い線やエラーマークで教えてくれる
    • 実行環境: 複雑な設定なしで、ボタン一つで実行できる

    📋 8. よくあるエラーと解決方法

    エラー1: セミコロンがない

    System.out.println("Hello, World!")  // ← セミコロンがない

    エラーメッセージ: 「;が必要です」 解決方法: 行末に ; を追加

    エラー2: 文字列の引用符が閉じていない

    System.out.println("Hello, World!);  // ← 引用符が足りない

    エラーメッセージ: 「文字列リテラルが閉じられていません」 解決方法: 文字列の最後に " を追加

    エラー3: 大文字・小文字の間違い

    system.out.println("Hello, World!");  // ← systemが小文字

    エラーメッセージ: 「systemを解決できません」 解決方法: System に修正（大文字のS）

    エラー4: クラス名とファイル名が異なる

    public class HelloWorld {  // ← クラス名
    ファイル名が Hello.java の場合

    エラーメッセージ: 「公開されたクラスHelloWorldは独自のファイルで定義する必要があります」 解決方法: ファイル名を HelloWorld.java に変更

    トラブルシューティングのコツ

    1. エラーメッセージを読む: Eclipseは親切にエラー内容を教えてくれます
    2. 赤い線を確認: エラーがある場所に赤い線が表示されます
    3. 一つずつ修正: 複数エラーがある場合は、上から順番に修正
    4. 保存して再実行: 修正後は必ず保存してから実行

    💡 よくある質問

    Q1: なぜpublic static void mainというおまじないが必要なの？

    A: 書けますが、Eclipseがあると作業がとても楽になります。自動補完、エラー検出、実行機能など、初心者にとって強力な味方です。

    Q3: プログラムが実行されない場合は？

    A: 以下を確認してください： 1. ファイルが保存されているか（Ctrl + S） 2. エラー（赤い線）がないか 3. main メソッドが正しく書かれているか
