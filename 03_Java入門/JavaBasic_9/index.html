    1. なぜ型変換が必要なのか？

    日常生活での「変換」

    私たちは日常的に単位や形式を変換しています：

    • 距離: 1.5km → 1500m （小数から整数への変換）
    • 時間: 90分 → 1時間30分 （単位の変換）
    • 通貨: 1ドル → 150円 （レートによる変換）

    プログラムでも、同じようにデータ型の変換が必要になる場面があります！

    プログラムでよくある場面

    // 問題1: 異なる型同士の計算
int totalStudents = 30;
int presentStudents = 25;
// 出席率を計算したい → 小数点も表示したい

// 問題2: ユーザー入力の処理
String userInput = "25";  // ユーザーが入力した文字列
// これを数値として計算に使いたい

// 問題3: 計算結果の表示
double average = 85.7;
// 「平均点: 85点」のように整数で表示したい

    これらの問題を解決するのが型変換です！

    📘 型変換の基本概念

    2. 暗黙的型変換（自動変換）

    暗黙的型変換とは？

    暗黙的型変換は、Javaが自動的に行ってくれる安全な型変換です。

    public class ImplicitConversion {
    public static void main(String[] args) {
        // int型からdouble型への自動変換
        int intValue = 10;
        double doubleValue = intValue;  // 自動的にdouble型に変換

        System.out.println("元の値（int）: " + intValue);
        System.out.println("変換後（double）: " + doubleValue);
    }
}

    実行結果： 

    元の値（int）: 10
変換後（double）: 10.0

    暗黙的変換のルール

    小さい型 → 大きい型 への変換は自動的に行われます：

    📘 データ型変換のルール

    // ✅ 自動変換される例（安全な変換）
byte b = 10;
short s = b;    // byte → short
int i = s;      // short → int
long l = i;     // int → long
float f = l;    // long → float
double d = f;   // float → double

System.out.println("byte: " + b);
System.out.println("short: " + s);
System.out.println("int: " + i);
System.out.println("long: " + l);
System.out.println("float: " + f);
System.out.println("double: " + d);

    なぜ自動変換が可能？

    理由: 大きい型は小さい型の値を完全に表現できるからです。

    int smallNumber = 100;
double bigNumber = smallNumber;  // 100.0になる、情報は失われない

    3. 明示的キャスト（手動変換）

    明示的キャストとは？

    明示的キャストは、プログラマーが明確に指示する型変換です。

    public class ExplicitCast {
    public static void main(String[] args) {
        // double型からint型への手動変換
        double doubleValue = 3.14;
        int intValue = (int) doubleValue;  // 明示的にint型に変換

        System.out.println("元の値（double）: " + doubleValue);
        System.out.println("変換後（int）: " + intValue);
    }
}

    実行結果： 

    元の値（double）: 3.14
変換後（int）: 3

    キャストの書き方

    (変換したい型) 変数名

    明示的キャストが必要な場面

    大きい型 → 小さい型 の変換では明示的キャストが必要です：

    // ❌ コンパイルエラーになる例
double d = 3.14;
int i = d;  // エラー！

// ✅ 正しい書き方
double d = 3.14;
int i = (int) d;  // OK（3になる）

    精度損失に注意！

    明示的キャストでは情報が失われることがあります：

    public class PrecisionLoss {
    public static void main(String[] args) {
        double originalValue = 123.456;
        int convertedValue = (int) originalValue;

        System.out.println("元の値: " + originalValue);
        System.out.println("変換後: " + convertedValue);
        System.out.println("失われた情報: " + (originalValue - convertedValue));
    }
}

    実行結果： 

    元の値: 123.456
変換後: 123
失われた情報: 0.456

    4. String⇔数値変換

    なぜString⇔数値変換が重要？

    ユーザー入力は常に文字列として受け取られます：

    // ユーザーが「25」と入力した場合
String userInput = "25";  // これは文字列の"25"
int age = userInput;      // ❌ エラー！直接代入はできない

    String → 数値への変換

    int型への変換

    public class StringToNumber {
    public static void main(String[] args) {
        String numberString = "123";
        int number = Integer.parseInt(numberString);

        System.out.println("文字列: " + numberString);
        System.out.println("数値: " + number);
        System.out.println("計算結果: " + (number * 2));
    }
}

    実行結果： 

    文字列: 123
数値: 123
計算結果: 246

    double型への変換

    String decimalString = "3.14";
double decimal = Double.parseDouble(decimalString);

System.out.println("文字列: " + decimalString);
System.out.println("数値: " + decimal);

    数値 → Stringへの変換

    public class NumberToString {
    public static void main(String[] args) {
        int age = 25;
        double height = 170.5;

        // 数値 → 文字列への変換
        String ageString = String.valueOf(age);
        String heightString = String.valueOf(height);

        System.out.println("年齢（数値）: " + age);
        System.out.println("年齢（文字列）: " + ageString);
        System.out.println("身長（数値）: " + height);
        System.out.println("身長（文字列）: " + heightString);
    }
}

    変換メソッドのまとめ

    変換方向 メソッド 例
    String → int Integer.parseInt() Integer.parseInt("123")
    String → double Double.parseDouble() Double.parseDouble("3.14")
    int → String String.valueOf() String.valueOf(123)
    double → String String.valueOf() String.valueOf(3.14)

    5. NumberFormatExceptionの対処

    エラーが起こる場面

    不正な文字列を数値に変換しようとするとエラーが発生します：

    public class ConversionError {
    public static void main(String[] args) {
        String invalidString = "abc";  // 数値ではない文字列

        try {
            int number = Integer.parseInt(invalidString);
            System.out.println("変換成功: " + number);
        } catch (NumberFormatException e) {
            System.out.println("変換エラー: " + invalidString + " は数値ではありません");
        }
    }
}

    実行結果： 

    変換エラー: abc は数値ではありません

    安全な変換方法

    public class SafeConversion {
    public static void main(String[] args) {
        String[] testStrings = {"123", "abc", "45.67", ""};

        for (String str : testStrings) {
            System.out.println("文字列: \"" + str + "\"");

            try {
                int number = Integer.parseInt(str);
                System.out.println("  → 変換成功: " + number);
            } catch (NumberFormatException e) {
                System.out.println("  → 変換失敗: 数値ではありません");
            }
            System.out.println();
        }
    }
}

    💡 よくある質問

    Q1: なぜ (double) をつけるのですか？

    A: int同士の割り算は結果もintになり、小数点以下が切り捨てられるからです。

    int a = 5, b = 2;
System.out.println(a / b);           // 2（小数点切り捨て）
System.out.println((double) a / b);  // 2.5（正確な結果）

    Q2: String.valueOf() 以外に数値→文字列変換はありますか？

    A: はい、いくつかの方法があります：

    int number = 123;
String str1 = String.valueOf(number);  // 推奨
String str2 = "" + number;             // 簡単だが読みにくい
String str3 = Integer.toString(number); // 型が明確

    Q3: キャストで型を間違えるとどうなりますか？

    A: コンパイルエラーになります：

    String text = "hello";
int number = (int) text;  // ❌ コンパイルエラー

    Q4: 精度損失を防ぐ方法はありますか？

    A: 変換前に値の範囲をチェックしたり、適切な型を選択したりします：

    double d = 123.456;
if (d == (int) d) {
    System.out.println("整数変換しても情報は失われません");
} else {
    System.out.println("小数点以下の情報が失われます: " + (d - (int) d));
}

    Q5: NumberFormatExceptionはどんな時に起こりますか？

    A: 以下のような場合に発生します：

    // ❌ エラーになる例
Integer.parseInt("abc");    // 英字
Integer.parseInt("");       // 空文字
Integer.parseInt("12.34");  // 小数（int変換時）
Integer.parseInt("1 2 3");  // スペース含む
Integer.parseInt(null);     // null
