    1. 算術演算子の基礎

    算術演算子とは？

    算術演算子は、数学の計算をプログラムで行うための記号です。小学校で学んだ計算が、Javaでもできるんです！

    📘 算術演算子の一覧表

    基本的な算術演算子

    演算子 意味 例 結果
    + 足し算 10 + 3 13
    - 引き算 10 - 3 7
    * 掛け算 10 * 3 30
    / 割り算 10 / 3 3
    % 余り 10 % 3 1

    注意: プログラミングでは掛け算は「×」ではなく「*」（アスタリスク）を使います！

    実際にプログラムで試してみよう

    ステップ1: 新しいクラスを作成

    前回と同じように、新しいクラスBasicOperationを作成しましょう。

    1. Eclipse で右クリック → 新規 → クラス
    2. クラス名：BasicOperation
    3. mainメソッドにチェック
    4. 完了

    ステップ2: 基本的な計算プログラム

    public class BasicOperation {
    public static void main(String[] args) {
        // 計算に使う数値を変数に格納
        int a = 10;
        int b = 3;

        // 各種計算を実行
        System.out.println("a = " + a + ", b = " + b);
        System.out.println("a + b = " + (a + b));  // 足し算
        System.out.println("a - b = " + (a - b));  // 引き算
        System.out.println("a * b = " + (a * b));  // 掛け算
        System.out.println("a / b = " + (a / b));  // 割り算
        System.out.println("a % b = " + (a % b));  // 余り
    }
}

    ステップ3: 実行して結果確認

    📘 Eclipseでの計算処理例

    結果はこのようになります： 

    a = 10, b = 3
a + b = 13
a - b = 7
a * b = 30
a / b = 3
a % b = 1

    重要なポイント：括弧の使い方

    上のプログラムで、なぜ (a + b) のように括弧をつけたのでしょうか？

    // 括弧なし
System.out.println("a + b = " + a + b);  // 結果: "a + b = 103" 

// 括弧あり
System.out.println("a + b = " + (a + b));  // 結果: "a + b = 13"

    理由: 括弧がないと、文字列連結として処理されてしまうからです！

    📘 括弧の効果に関する説明

    割り算の注意点

    整数同士の割り算では、小数点以下が切り捨てられます：

    int result1 = 10 / 3;  // 結果: 3 (3.333...ではない)
int result2 = 7 / 2;   // 結果: 3 (3.5ではない)

System.out.println("10 / 3 = " + result1);
System.out.println("7 / 2 = " + result2);

    余り（%）演算子の活用

    余り演算子は、割り算の余りを求めます。これは意外に便利で、よく使われます：

    // 偶数・奇数の判定に使える
int number = 7;
int remainder = number % 2;
System.out.println(number + " を 2 で割った余り: " + remainder);
// 7 % 2 = 1 → 奇数

// 10で割った余りで一の位を取得
int score = 87;
int lastDigit = score % 10;
System.out.println("点数の一の位: " + lastDigit);  // 7

    2. 比較演算子の基礎

    比較演算子とは？

    比較演算子は、2つの値を比べて、「正しい（true）」か「間違い（false）」かを判定する演算子です。

    📘 比較演算子の説明

    基本的な比較演算子

    演算子 意味 例 結果
    == 等しい 10 == 10 true
    != 等しくない 10 != 3 true
    > より大きい 10 > 3 true
    < より小さい 10 < 3 false
    >= 以上 10 >= 10 true
    <= 以下 10 <= 3 false

    重要: 「等しい」は == （イコール2個）です。= （イコール1個）は代入に使うので混同しないように！

    3. boolean型の導入

    boolean型とは？

    比較演算子の結果は、boolean型という特別な型で表されます。

    boolean result = (10 > 3);
System.out.println(result);  // true

    boolean型が持てる値は2つだけ： - true：正しい（真） - false：間違い（偽）

    📘 boolean型の表示例

    実際にプログラムで試してみよう

    public class ComparisonExample {
    public static void main(String[] args) {
        int score1 = 85;
        int score2 = 70;

        // 等しいかどうか
        boolean isEqual = (score1 == score2);
        System.out.println("score1 == score2: " + isEqual);  // false

        // 異なるかどうか
        boolean isDifferent = (score1 != score2);
        System.out.println("score1 != score2: " + isDifferent);  // true

        // 大小比較
        boolean isGreater = (score1 > score2);
        System.out.println("score1 > score2: " + isGreater);  // true

        boolean isLessEqual = (score1 <= 100);
        System.out.println("score1 <= 100: " + isLessEqual);  // true
    }
}

    5. 演算子の優先順位

    なぜ優先順位が重要？

    数学と同じように、プログラミングでも計算には順序があります：

    int result = 2 + 3 * 4;  // 結果は？
System.out.println(result);  // 14 (20ではない！)

    理由: 掛け算が先に計算されるから → 3 * 4 = 12 → 2 + 12 = 14

    📘 演算子の優先順位の説明

    基本的な優先順位

    1. 括弧 ( )
    2. 算術演算子 *, /, %
    3. 算術演算子 +, -
    4. 比較演算子 <, >, <=, >=
    5. 比較演算子 ==, !=

    迷ったときの対策

    括弧を使って明確にしましょう！

    // わかりにくい
int result1 = 2 + 3 * 4;

// わかりやすい
int result2 = 2 + (3 * 4);  // 掛け算を先にしたい
int result3 = (2 + 3) * 4;  // 足し算を先にしたい

    6. 実践：総合演習プログラム

    課題：お小遣い管理プログラム

    今月のお小遣いと支出を管理するプログラムを作ってみましょう。

    まずは自分で考えてみよう！

    以下のヒントを参考に、まずは自分でプログラムを作成してみてください：

    💡 ヒント1: 必要な変数を考えよう 

    // どんな情報が必要でしょうか？
int monthlyAllowance = ？;  // 今月のお小遣い
int bookCost = ？;          // 本代
// 他にも支出項目を追加してみよう

    💡 ヒント2: 計算に必要な演算子 - 合計支出 → どの演算子を使う？（複数の金額を合計） - 残金 → どの演算子を使う？（お小遣いから支出を引く）

    💡 ヒント3: 判定に必要な比較演算子 - 予算オーバーしてる？ → 支出 ？ お小遣い - まだ1000円以上ある？ → 残金 ？ 1000 - 半分以上使った？ → 支出 ？ お小遣いの半分

    完成例

    自分で挑戦してみた後、以下の完成例と比較してみてください：

    public class PocketMoneyManager {
    public static void main(String[] args) {
        // 今月の状況
        int monthlyAllowance = 5000;  // 月のお小遣い
        int bookCost = 1200;          // 本代
        int snackCost = 800;          // お菓子代
        int gameCost = 2500;          // ゲーム代

        // 合計支出を計算（算術演算子 + を使用）
        int totalSpent = bookCost + snackCost + gameCost;

        // 残金を計算（算術演算子 - を使用）
        int remaining = monthlyAllowance - totalSpent;

        // 各種判定（比較演算子を使用）
        boolean isOverBudget = (totalSpent > monthlyAllowance);  // 予算オーバー？
        boolean canBuyMore = (remaining >= 1000);                // まだ1000円以上買い物できる？
        boolean spentHalf = (totalSpent >= monthlyAllowance / 2); // 半分以上使った？

        // 結果表示
        System.out.println("=== 今月のお小遣い管理 ===");
        System.out.println("お小遣い: " + monthlyAllowance + "円");
        System.out.println("支出合計: " + totalSpent + "円");
        System.out.println("残金: " + remaining + "円");
        System.out.println();
        System.out.println("予算オーバー？ " + isOverBudget);
        System.out.println("1000円以上残ってる？ " + canBuyMore);
        System.out.println("半分以上使った？ " + spentHalf);
    }
}

    💡 よくある質問

    Q1: = と == の違いがよくわかりません

    A: - = は代入：変数に値を入れる（int age = 20;） - == は比較：2つの値が同じかチェック（age == 20）

    int a = 5;      // 代入：aに5を入れる
boolean b = (a == 5);  // 比較：aが5と等しいかチェック

    Q2: 割り算の結果が思ったものと違います

    A: 整数同士の割り算は、小数点以下が切り捨てられます。

    int result = 7 / 2;  // 結果は3 (3.5ではない)

    これは次回以降のレッスンで詳しく学習します。

    Q3: 比較演算子の結果がtrue/falseで表示されるのはなぜ？

    A: 比較の結果はboolean型になり、JavaではtrueFalseで表示されます。これは「正しい」「間違い」を表す専用の型です。

    Q4: 複雑な計算の順序がわからなくなります

    A: 迷ったときは積極的に括弧を使いましょう！括弧は何重にしても大丈夫です。

    int result = ((a + b) * c) / (d - e);  // 括弧で順序を明確に
