    1. なぜ対話型プログラムが重要なのか？

    単発処理の限界

    これまで作成したプログラムは、一度実行すると終了してしまいました：

    // 従来のプログラム
public static void main(String[] args) {
    // 何かの処理
    System.out.println("結果: " + result);
    // プログラム終了
}

    問題点： - 別の処理をしたい場合、プログラムを再起動する必要がある - 複数の機能を使い分けることができない - 実用的なアプリケーションとは言えない

    対話型プログラムの価値

    // 対話型プログラム
public static void main(String[] args) {
    while (true) {
        // メニュー表示
        // ユーザーの選択に応じて処理
        // 継続するかユーザーに確認
    }
}

    メリット： - 一度起動すれば複数の処理を実行できる - ユーザーが必要な機能を選択できる - 実際のソフトウェアに近い使用感

    身近な対話型プログラムの例

    • ATM: メニューから「残高照会」「出金」「入金」を選択
    • コンビニレジ: 商品登録、会計、レシート印刷を繰り返し
    • カーナビ: 目的地設定、ルート検索、案内開始を選択

    2. while文による継続的な処理ループ

    while文の基本構文

    while (条件) {
    // 条件がtrueの間、繰り返し実行される処理
}

    for文との違い

    // for文（回数が決まっている場合）
for (int i = 1; i <= 5; i++) {
    System.out.println(i + "回目");
}

// while文（条件によって継続する場合）
boolean continueProgram = true;
while (continueProgram) {
    // 何かの処理
    // continueProgram = false; でループを終了
}

    実用的なwhile文の例

    public class SimpleLoop {
    public static void main(String[] args) {
        Scanner scanner = new Scanner(System.in);
        String answer;

        do {
            System.out.println("簡単な計算をします");
            System.out.print("1つ目の数値: ");
            int num1 = scanner.nextInt();
            System.out.print("2つ目の数値: ");
            int num2 = scanner.nextInt();

            int sum = num1 + num2;
            System.out.println("合計: " + sum);

            System.out.print("もう一度計算しますか？ (y/n): ");
            scanner.nextLine(); // 改行文字を消費
            answer = scanner.nextLine();

        } while (answer.equalsIgnoreCase("y"));

        System.out.println("プログラムを終了します");
        scanner.close();
    }
}

2つ目の数値: 15
合計: 20

    3. switch-case文によるメニューシステム

    switch-case文の基本構文

    switch (変数) {
    case 値1:
        // 値1の場合の処理
        break;
    case 値2:
        // 値2の場合の処理
        break;
    default:
        // どのcaseにも該当しない場合の処理
        break;
}

    if-else文との比較

    // if-else文での選択処理
if (choice == 1) {
    System.out.println("機能1を実行");

    なぜ配列が必要なのか？

    これまでは一つずつの値しか扱えませんでした：

    問題点： - 学生数が増えると変数がどんどん増える - 処理が複雑になる - 柔軟性がない

    配列による解決

    // 解決：配列を使用
int[] scores = {85, 92, 78, 88, 95};

// 平均を計算
int sum = 0;
for (int i = 0; i < scores.length; i++) {
    sum += scores[i];
}
double average = sum / (double) scores.length;

System.out.printf("平均点: %.1f点%n", average);

import java.util.Scanner;

public class StudentManager {

    public static void main(String[] args) {
        Scanner scanner = new Scanner(System.in);

        // 学生情報を保存する配列（最大10人）

    /**
     * メニューを表示する

        System.out.println("    学生情報管理システム");
        System.out.println("=".repeat(30));
        System.out.println("1. 学生一覧表示");

    /**
     * 有効な選択肢を取得する（エラーハンドリング付き）
     */
    public static int getValidChoice(Scanner scanner) {
        while (true) {
            try {
                int choice = scanner.nextInt();
                scanner.nextLine(); // 改行文字を消費
                return choice;
            } catch (InputMismatchException e) {
                System.out.print("数値を入力してください: ");
                scanner.nextLine(); // 不正入力をクリア
            }
        }
    }

    /**
     * 有効な点数を取得する（0-100の範囲チェック付き）
     */
    public static int getValidScore(Scanner scanner) {
        while (true) {
            try {
                System.out.print("点数を入力してください (0-100): ");
                int score = scanner.nextInt();
                scanner.nextLine(); // 改行文字を消費

                if (score < 0 || score > 100) {
                    System.out.println("エラー: 0から100までの値を入力してください");
                    continue;
                }

                return score;
            } catch (InputMismatchException e) {
                System.out.println("エラー: 数値を入力してください");
                scanner.nextLine(); // 不正入力をクリア
            }
        }
    }

        if (count >= names.length) {
            System.out.println("エラー: これ以上学生を登録できません（最大" + names.length + "人）");
            return count;
        }

        System.out.print("学生名を入力してください: ");
        String name = scanner.nextLine();

        if (name.trim().isEmpty()) {
            System.out.println("エラー: 学生名を入力してください");
            return count;
        }

        int score = getValidScore(scanner);

        names[count] = name;
        scores[count] = score;

        System.out.println("学生情報を登録しました");
        System.out.printf("登録内容: %s さん (%d点)%n", name, score);

        return count + 1;
    }

    /**
     * 統計情報を表示する
     */
    public static void showStatistics(String[] names, int[] scores, int count) {
        System.out.println("\n=== 統計情報 ===");

        if (count == 0) {
            System.out.println("統計を計算できません（学生が登録されていません）");
            return;
        }

        int sum = 0;
        int max = scores[0];
        int min = scores[0];
        String maxName = names[0];
        String minName = names[0];

        for (int i = 0; i < count; i++) {
            sum += scores[i];

            if (scores[i] > max) {
                max = scores[i];
                maxName = names[i];
            }

            if (scores[i] < min) {
                min = scores[i];
                minName = names[i];
            }
        }

        double average = sum / (double) count;

        System.out.printf("総学生数: %d人%n", count);
        System.out.printf("平均点: %.1f点%n", average);
        System.out.printf("最高点: %d点 (%s さん)%n", max, maxName);
        System.out.printf("最低点: %d点 (%s さん)%n", min, minName);
        System.out.printf("合計点: %d点%n", sum);
    }

    /**
     * 成績評価を表示する
     */
    public static void showGradeEvaluation(String[] names, int[] scores, int count) {
        System.out.println("\n=== 成績評価 ===");

        if (count == 0) {
            System.out.println("評価する学生がいません");
            return;
        }

        int excellentCount = 0, goodCount = 0, passCount = 0, failCount = 0;

        System.out.println("名前          | 点数 | 評価 | 判定");
        System.out.println("--------------|------|------|--------");

        for (int i = 0; i < count; i++) {
            String grade = getGrade(scores[i]);
            String status = getStatus(scores[i]);

            System.out.printf("%-12s | %4d | %4s | %s%n", 
                            names[i], scores[i], grade, status);

            if (scores[i] >= 90) excellentCount++;
            else if (scores[i] >= 75) goodCount++;
            else if (scores[i] >= 60) passCount++;
            else failCount++;
        }

        System.out.println("\n評価別人数:");
        System.out.printf("優 (90点以上): %d人%n", excellentCount);
        System.out.printf("良 (75-89点): %d人%n", goodCount);
        System.out.printf("可 (60-74点): %d人%n", passCount);
        System.out.printf("不可 (60点未満): %d人%n", failCount);
    }

    /**
     * 点数から成績評価を取得する
     */
    public static String getGrade(int score) {
        if (score >= 90) return "優";
        else if (score >= 75) return "良";
        else if (score >= 60) return "可";
        else return "不可";
    }

    /**
     * 点数から合否判定を取得する
     */
    public static String getStatus(int score) {

    }

    💡 よくある質問

    Q1: while文とfor文の使い分けがわかりません

    A: 以下の基準で判断してください： - for文: 繰り返し回数が決まっている場合（1から10まで、配列の全要素など） - while文: 条件によって継続する場合（ユーザーが終了を選ぶまで、正しい入力まで）

    // for文：配列の全要素を処理（回数が決まっている）
for (int i = 0; i < array.length; i++) {
    System.out.println(array[i]);
}

// while文：ユーザーが終了を選ぶまで（回数が不明）
boolean running = true;
while (running) {
    // メニュー処理
    if (choice == 0) running = false;
}

    Q2: switch-case文のbreakを忘れるとどうなりますか？

    A: フォールスルーという現象が起きて、次のcaseの処理も実行されてしまいます：

    switch (choice) {
    case 1:
        System.out.println("機能1");
        // breakがない！
    case 2:
        System.out.println("機能2");
        break;
}

// choice=1の場合の出力：
// 機能1
// 機能2  ← 意図しない出力

    対策: 必ず各caseの最後にbreak;を記述してください。

    Q3: 配列のインデックスを間違えやすいです

    A: 以下のことに注意してください： - 配列のインデックスは0から始まる - 最後の要素はarray[array.length - 1] - ArrayIndexOutOfBoundsExceptionに注意

    int[] scores = new int[5]; // インデックス0-4が有効

// ✅ 正しいアクセス
scores[0] = 100; // 最初の要素
scores[4] = 90;  // 最後の要素

// ❌ エラーになるアクセス
scores[5] = 80;  // ArrayIndexOutOfBoundsException

    Q4: mainメソッドが長くなってしまいます

    A: 以下の方針で改善してください： 1. 機能ごとにメソッドを作成 2. mainメソッドは処理の流れのみ記述 3. 一つのメソッドは20-30行程度に抑える

    // ✅ 良い例：mainは処理の流れのみ
public static void main(String[] args) {
    initializeSystem();
    while (isRunning()) {
        showMenuAndProcessChoice();
    }
    finalizeSystem();
}

    Q5: エラーハンドリングを忘れてしまいます

    A: 以下のチェックリストを使用してください： - [ ] Scanner.nextInt()を使う場所でInputMismatchExceptionを処理している - [ ] 数値の範囲チェックを行っている - [ ] 配列のインデックス範囲チェックを行っている - [ ] null文字列のチェックを行っている
