    1. なぜエラーハンドリングが必要なのか？

    現実のプログラムで起こる問題

    前回学んだ基本的な入力処理には、まだ解決されていない問題があります：

    public class BasicInputProblem {
    public static void main(String[] args) {
        Scanner scanner = new Scanner(System.in);

        System.out.print("年齢を入力してください: ");
        int age = scanner.nextInt();  // ここで問題が発生する可能性

        System.out.println("あなたは" + age + "歳ですね");
        scanner.close();
    }
}

    このプログラムで「abc」と入力すると... 

    Exception in thread "main" java.util.InputMismatchException

    プログラムがクラッシュしてしまいます！

    エラーハンドリングの効果

    同じプログラムに例外処理を追加すると：

    public class SafeInputExample {
    public static void main(String[] args) {
        Scanner scanner = new Scanner(System.in);

        System.out.print("年齢を入力してください: ");
        try {
            int age = scanner.nextInt();
            System.out.println("あなたは" + age + "歳ですね");
        } catch (InputMismatchException e) {
            System.out.println("エラー: 数値を入力してください");
        }

        scanner.close();
    }
}

    「abc」と入力した場合： 

    年齢を入力してください: abc
エラー: 数値を入力してください

    プログラムがクラッシュせずに、適切なエラーメッセージを表示できました！

    2. try-catch文の基本構文

    try-catch文とは？

    try-catch文は、プログラムでエラー（例外）が発生した時に、プログラムをクラッシュさせずに適切に対処する仕組みです。

    基本的な書き方

    try {
    // エラーが発生する可能性のあるコード
} catch (例外の種類 変数名) {
    // エラーが発生した時の処理
}

    実際の例

    public class TryCatchBasic {
    public static void main(String[] args) {
        Scanner scanner = new Scanner(System.in);

        System.out.print("数値を入力してください: ");

        try {
            int number = scanner.nextInt();
            System.out.println("入力された数値: " + number);
        } catch (InputMismatchException e) {
            System.out.println("エラー: 数値以外が入力されました");
        }

        scanner.close();
    }
}

    try-catch文の流れ

    📘 try-catchのフローチャート

    1. tryブロック内のコードを実行
    2. エラーが発生しなかった場合: catchブロックをスキップして次の処理へ
    3. エラーが発生した場合: 該当するcatchブロックの処理を実行

    3. InputMismatchException の対応

    InputMismatchException とは？

    InputMismatchExceptionは、Scannerで数値を期待している時に、数値以外のデータが入力された場合に発生する例外です。

    具体的な対応方法

    public class InputMismatchHandling {
    public static void main(String[] args) {
        Scanner scanner = new Scanner(System.in);

        System.out.print("点数を入力してください (0-100): ");

        try {
            int score = scanner.nextInt();

            // 正常に入力された場合の処理
            if (score >= 0 && score <= 100) {
                System.out.println("入力された点数: " + score);

                if (score >= 80) {
                    System.out.println("評価: 優秀");
                } else if (score >= 60) {
                    System.out.println("評価: 合格");
                } else {
                    System.out.println("評価: 不合格");
                }
            } else {
                System.out.println("エラー: 0から100の範囲で入力してください");
            }

        } catch (InputMismatchException e) {
            System.out.println("エラー: 数値を入力してください");
            scanner.nextLine(); // 不正な入力をクリア
        }

        scanner.close();
    }
}

    重要なポイント

    scanner.nextLine()でバッファクリア

    catch (InputMismatchException e) {
    System.out.println("エラー: 数値を入力してください");
    scanner.nextLine(); // ← これが重要！
}

    例外が発生した場合、Scannerのバッファに不正な入力が残ってしまいます。scanner.nextLine()で不正な入力をクリアすることで、次の入力処理に影響しないようにします。

    4. NumberFormatException の対応

    NumberFormatException とは？

    NumberFormatExceptionは、文字列を数値に変換しようとした時に、変換できない形式の文字列が渡された場合に発生する例外です。

    Integer.parseInt() での例外処理

    public class NumberFormatHandling {
    public static void main(String[] args) {
        Scanner scanner = new Scanner(System.in);

        System.out.print("年齢を入力してください: ");
        String input = scanner.nextLine();

        try {
            int age = Integer.parseInt(input);

            if (age >= 0 && age <= 150) {
                System.out.println("年齢: " + age + "歳");
            } else {
                System.out.println("エラー: 現実的な年齢を入力してください (0-150)");
            }

        } catch (NumberFormatException e) {
            System.out.println("エラー: 数値を入力してください");
            System.out.println("入力された値: \"" + input + "\"");
        }

        scanner.close();
    }
}

    Double.parseDouble() での例外処理

    public class DoubleParseHandling {
    public static void main(String[] args) {
        Scanner scanner = new Scanner(System.in);

        System.out.print("身長を入力してください (cm): ");
        String input = scanner.nextLine();

        try {
            double height = Double.parseDouble(input);

            if (height > 0 && height < 300) {
                System.out.printf("身長: %.1f cm%n", height);

                // BMI計算のための体重入力
                System.out.print("体重を入力してください (kg): ");
                String weightInput = scanner.nextLine();

                try {
                    double weight = Double.parseDouble(weightInput);
                    if (weight > 0 && weight < 500) {
                        double bmi = weight / ((height / 100) * (height / 100));
                        System.out.printf("BMI: %.1f%n", bmi);
                    } else {
                        System.out.println("エラー: 現実的な体重を入力してください");
                    }
                } catch (NumberFormatException e) {
                    System.out.println("エラー: 体重は数値で入力してください");
                }

            } else {
                System.out.println("エラー: 現実的な身長を入力してください");
            }

        } catch (NumberFormatException e) {
            System.out.println("エラー: 身長は数値で入力してください");
        }

        scanner.close();
    }
}

    5. do-while文を使った再試行ループ

    正しい入力まで繰り返す仕組み

    実用的なプログラムでは、エラーが発生しても再度入力を求めることが一般的です。

    do-while文の基本構文

    do {
    // 繰り返したい処理
} while (条件);

    実践例：堅牢な年齢入力システム

    public class RobustAgeInput {
    public static void main(String[] args) {
        Scanner scanner = new Scanner(System.in);
        int age = -1;  // 無効な初期値
        boolean validInput = false;

        do {
            System.out.print("年齢を入力してください (0-120): ");

            try {
                age = scanner.nextInt();

                // 年齢の範囲チェック
                if (age >= 0 && age <= 120) {
                    validInput = true;  // 正常な入力

    💡 よくある質問

    Q1: try-catch文とif文による事前チェックの違いは？

    A: - if文（事前チェック）: hasNextInt()のように、エラーが起きる前に確認 - try-catch文（例外処理）: エラーが起きてから対処

    // 事前チェック方式
if (scanner.hasNextInt()) {
    int number = scanner.nextInt();
} else {
    System.out.println("数値を入力してください");
}

// 例外処理方式
try {
    int number = scanner.nextInt();
} catch (InputMismatchException e) {
    System.out.println("数値を入力してください");
}

    どちらも有効ですが、例外処理の方がより汎用的で実用的です。

    Q2: catch文でscanner.nextLine()が必要な理由は？

    A: InputMismatchExceptionが発生すると、不正な入力がScannerのバッファに残ったままになります。これをクリアしないと、次の入力処理でも同じエラーが繰り返されます。

    // バッファクリアなし（問題のあるコード）
catch (InputMismatchException e) {
    System.out.println("エラー");
    // 不正な入力が残ったまま
}

// 正しいコード
catch (InputMismatchException e) {
    System.out.println("エラー");
    scanner.nextLine(); // 不正な入力をクリア
}

    Q3: なぜInteger.parseInt()とscanner.nextInt()を使い分けるのですか？

    A: - scanner.nextInt(): 直接入力から整数を読み取る - Integer.parseInt(): 文字列を整数に変換

    // scanner.nextInt()の場合
try {
    int age = scanner.nextInt();
} catch (InputMismatchException e) {
    // 処理
}

// Integer.parseInt()の場合
try {
    String input = scanner.nextLine();
    int age = Integer.parseInt(input);
} catch (NumberFormatException e) {
    // 処理
}

    Integer.parseInt()の方が文字列操作と組み合わせやすく、より細かい制御が可能です。

    Q4: do-while文とwhile文の違いは？

    A: - while文: 条件をチェックしてから実行（0回の場合あり） - do-while文: 実行してから条件をチェック（必ず1回は実行）

    // while文
while (条件) {
    // 条件がfalseなら1回も実行されない
}

// do-while文
do {
    // 必ず1回は実行される
} while (条件);

    入力処理では必ず1回は入力を求める必要があるため、do-while文が適しています。

    Q5: 例外処理はどこまで使うべきですか？

    A: 予期しないエラーや外部要因によるエラーに対して使用します。

    // 適切な使用例
try {
    int number = Integer.parseInt(userInput); // ユーザー入力は予測不可能
    File file = new File(filename); // ファイルが存在しない可能性
} catch (NumberFormatException | FileNotFoundException e) {
    // エラー処理
}

// 不適切な使用例
try {
    int result = 10 / 0; // 明らかにエラーになる処理
} catch (ArithmeticException e) {
    // このようなコードは書かない
}
