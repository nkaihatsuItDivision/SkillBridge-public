    // 1から5までの数字の偶数・奇数を判定
for (int i = 1; i <= 5; i++) {
    if (i % 2 == 0) {
        System.out.println(i + "は偶数です");

    for文は同じ処理の繰り返しに威力を発揮しました。 今回は、さらに一歩進んで同じ処理のまとまりを作る方法を学びます！

    1. なぜメソッドが必要なのか？

    for文では解決できない問題

    以下のようなプログラムを考えてみましょう：

    public class WithoutMethod {
    public static void main(String[] args) {
        // 太郎さんの情報を表示
        System.out.println("=== 学生情報 ===");
        System.out.println("名前: 太郎");
        System.out.println("年齢: 20");
        System.out.println("成績: 85点");
        System.out.println();

        // 花子さんの情報を表示
        System.out.println("=== 学生情報 ===");
        System.out.println("名前: 花子");
        System.out.println("年齢: 19");
        System.out.println("成績: 92点");
        System.out.println();

        // 次郎さんの情報を表示
        System.out.println("=== 学生情報 ===");
        System.out.println("名前: 次郎");
        System.out.println("年齢: 21");
        System.out.println("成績: 78点");
        System.out.println();
    }
}

    何か気づきませんか？

    • 似たような処理が3回も書かれている
    • for文は使えない（データが違うから）
    • コードが長くて読みにくい

    これが「同じ処理のまとまり」の問題です！

    📘 メソッドを使った課題解決の例

    メソッドによる解決

    同じ処理をメソッドにまとめることで、すっきりと書けます：

    public class WithMethod {
    public static void main(String[] args) {
        // メソッドを呼び出すだけ！
        showStudentInfo("太郎", 20, 85);
        showStudentInfo("花子", 19, 92);
        showStudentInfo("次郎", 21, 78);
    }

    // 学生情報を表示するメソッド
    public static void showStudentInfo(String name, int age, int score) {
        System.out.println("=== 学生情報 ===");
        System.out.println("名前: " + name);
        System.out.println("年齢: " + age);
        System.out.println("成績: " + score + "点");
        System.out.println();
    }
}

    メソッドの効果： - コードの重複がなくなった - 処理の意味が分かりやすい - 修正が一箇所で済む

    2. メソッドの基本構文

    メソッドの基本的な書き方

    public static 戻り値の型 メソッド名(引数) {
    // 処理内容
}

    各部分の説明： - public static：おまじない（詳しくは後で学習） - 戻り値の型：メソッドが返す値の型（何も返さない場合はvoid） - メソッド名：好きな名前（分かりやすい名前をつける） - 引数：メソッドに渡すデータ

    最も簡単なメソッド（引数なし、戻り値なし）

    public class SimpleMethod {
    public static void main(String[] args) {
        System.out.println("プログラム開始");
        greetUser(); // メソッドを呼び出し
        System.out.println("プログラム終了");
    }

    // 挨拶をするメソッド
    public static void greetUser() {
        System.out.println("こんにちは！");
        System.out.println("Javaプログラミング頑張りましょう！");
    }
}

    実行結果： 

    プログラム開始
こんにちは！
Javaプログラミング頑張りましょう！
プログラム終了

    ポイント： - メソッドはmainメソッドの外に書く - メソッドを呼び出すときはメソッド名()と書く - voidは「何も返さない」という意味

    3. 引数を使ったメソッド

    引数とは？

    引数（ひきすう）は、メソッドに渡すデータのことです。 同じ処理でも、データが変わる部分を引数で指定します。

    public class MethodWithParameter {
    public static void main(String[] args) {
        greetUser("田中さん");
        greetUser("佐藤さん");
        greetUser("山田さん");
    }

    // 名前を受け取って挨拶するメソッド
    public static void greetUser(String name) {
        System.out.println("こんにちは、" + name + "！");
        System.out.println("今日も一日頑張りましょう！");
        System.out.println();
    }
}

    実行結果： 

    こんにちは、田中さん！
今日も一日頑張りましょう！

こんにちは、佐藤さん！
今日も一日頑張りましょう！

こんにちは、山田さん！
今日も一日頑張りましょう！

    複数の引数を使う

    public class MultipleParameters {
    public static void main(String[] args) {
        calculateArea(5, 3);
        calculateArea(10, 7);
        calculateArea(2, 8);
    }

    // 長方形の面積を計算して表示するメソッド
    public static void calculateArea(int width, int height) {
        int area = width * height;
        System.out.println("幅" + width + "、高さ" + height + "の長方形");
        System.out.println("面積: " + area);
        System.out.println();
    }
}

    引数の種類： - int：数値を渡す - String：文字列を渡す
    - boolean：true/falseを渡す

    4. 戻り値を返すメソッド

    戻り値とは？

    戻り値（もどりち）は、メソッドが処理した結果を呼び出し元に返すことです。

    public class MethodWithReturn {
    public static void main(String[] args) {
        int result1 = calculateSum(10, 20);
        int result2 = calculateSum(5, 15);

        System.out.println("10 + 20 = " + result1);
        System.out.println("5 + 15 = " + result2);

        // メソッドの結果を直接使うこともできる
        System.out.println("30 + 40 = " + calculateSum(30, 40));
    }

    // 2つの数値の合計を計算して返すメソッド
    public static int calculateSum(int a, int b) {
        int sum = a + b;
        return sum; // 結果を返す
    }
}

    実行結果： 

    10 + 20 = 30
5 + 15 = 20
30 + 40 = 70

    ポイント： - 戻り値がある場合はvoidの代わりにデータ型を書く - return文で値を返す - 戻り値は変数で受け取るか、直接使用できる

    boolean型を返すメソッド

    前回学んだ偶数・奇数判定をメソッドにしてみましょう：

    public class BooleanReturn {
    public static void main(String[] args) {
        // 1から10までの数字を判定
        for (int i = 1; i <= 10; i++) {
            boolean isEven = checkEven(i);
            if (isEven) {
                System.out.println(i + "は偶数です");
            } else {
                System.out.println(i + "は奇数です");
            }
        }
    }

    // 偶数かどうかを判定するメソッド
    public static boolean checkEven(int number) {
        return number % 2 == 0;
    }
}

    💡 ヒント1: メソッドの戻り値はString型にして、判定結果の文字列を返しましょう

    💡 ヒント2: if文とelse if文を組み合わせて判定処理を書きましょう

    💡 ヒント3: レッスン4で学んだ比較演算子(&gt;=)を活用しましょう

    完成例

    public class GradeJudgment {
    public static void main(String[] args) {
        // 各学生の成績を判定
        System.out.println("田中さん(85点): " + judgeGrade(85));
        System.out.println("佐藤さん(92点): " + judgeGrade(92));
        System.out.println("山田さん(55点): " + judgeGrade(55));
        System.out.println("鈴木さん(78点): " + judgeGrade(78));
    }

    // 点数を受け取って成績を判定するメソッド
    public static String judgeGrade(int score) {
        if (score &gt;= 90) {
            return "優秀";
        } else if (score &gt;= 80) {
            return "良好";
        } else if (score &gt;= 60) {
            return "合格";
        } else {
            return "不合格";
        }
    }
}

    学習ポイント： - String型の戻り値：判定結果を文字列で返す - if-else if文：複数の条件を順番に判定 - メソッドの再利用：同じ判定処理を何度でも使える

    6. クラスとメソッドの関係

    クラスとは？

    クラスは、関連するメソッドをまとめるための「入れ物」です。

    📘 プログラム設計の基本概念

    public class Calculator {  // ← これがクラス

    // mainメソッド
    public static void main(String[] args) {
        // 処理を呼び出す
    }

    // 足し算メソッド
    public static int add(int a, int b) {
        return a + b;
    }

    // 引き算メソッド
    public static int subtract(int a, int b) {
        return a - b;
    }

    // 掛け算メソッド
    public static int multiply(int a, int b) {
        return a * b;
    }
}

    クラスの特徴： - 1つのファイル = 1つのクラス - クラス名 = ファイル名（Calculator.java） - 関連するメソッドをまとめる（電卓なら計算系メソッド）

    プログラムの実行の流れ

    1. Javaプログラム実行
    2. mainメソッドが最初に呼び出される
    3. mainメソッドから他のメソッドを呼び出し
    4. メソッドが処理を実行して結果を返す
    5. mainメソッドに戻って続きを実行

    💡 よくある質問

    Q1: public staticって何ですか？

    A: 現在は「おまじない」と考えてください。詳しくはWeek 3で学習します。 今は「メソッドを作るときに必要な決まり」程度の理解で十分です。

    Q2: メソッド名はどうやって決めればいいですか？

    A: 処理の内容が分かる名前をつけましょう： - ✅ 良い例：calculateSum, checkEven, judgeGrade - ❌ 悪い例：method1, abc, test

    Q3: 引数の順番を間違えるとどうなりますか？

    A: 思った通りの結果になりません： 

    calculateArea(5, 3); // 幅5、高さ3
calculateArea(3, 5); // 幅3、高さ5（異なる結果）
    引数の順番と意味をしっかり確認しましょう。

    Q4: returnを忘れるとどうなりますか？

    A: コンパイルエラーになります： 

    public static int calculateSum(int a, int b) {
    int sum = a + b;
    // return sum; ← これを忘れるとエラー
}
    戻り値がある場合は必ずreturn文が必要です。

    Q5: voidメソッドでreturnを使えますか？

    A: 使えますが、値は返せません： 

    public static void greetUser(String name) {
    if (name.equals("")) {
        System.out.println("名前が入力されていません");
        return; // ここで処理を終了
    }
    System.out.println("こんにちは、" + name + "さん！");
}
