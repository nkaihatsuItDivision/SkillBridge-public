    📚 前回までの復習

    前回の「Webの仕組み」で学んだ内容を思い出してみましょう：

    • クライアント（あなたのブラウザ） がサーバーにデータを送信
    • GET: データを取得する（検索など）
    • POST: データを送信する（フォーム送信など）
    • フォーム送信: ユーザーが入力したデータをサーバーに送る仕組み

    今回は、この理論を実際にHTMLのフォームで体験していきます！

    1. フォームとは何か？

    身近なフォームの例

    普段インターネットを使っていて、こんな場面はありませんか？

    • Googleで検索するとき → 検索ボックスに文字を入力
    • 会員登録するとき → 名前、メールアドレスを入力
    • お問い合わせするとき → 質問内容を入力

    これらすべてがフォームです！

    📘 フォーム送信と埋め込み表示の例

    フォームの役割

    フォームは、Webページでユーザーがデータを入力して、サーバーに送信するための仕組みです。

    ユーザー入力 → フォーム → サーバーに送信 → 処理結果が返ってくる

    2. form要素の基本構造

    HTMLでのフォーム作成

    HTMLでフォームを作るには、<form>タグを使います。

    <form action="送信先のURL" method="送信方法">
  <!-- ここに入力要素を配置 -->
</form>

    重要な属性

    action属性

    データをどこに送るかを指定します。

    <!-- 例1: 検索フォーム -->
<form action="https://www.google.com/search" method="get">

<!-- 例2: お問い合わせフォーム -->
<form action="/contact" method="post">

    method属性

    データをどのように送るかを指定します。

    • GET: データをURLに含めて送信（検索など）
    • POST: データを本文に含めて送信（個人情報など）

    3. input要素とtype属性の基本

    input要素とは？

    <input>は、ユーザーがデータを入力するための要素です。今まで学習したHTMLタグ（h1、p、aなど）とは少し違い、自己完結型のタグです。

    <!-- 今まで学んだタグ（開始タグと終了タグ） -->
<h1>見出し</h1>
<p>段落</p>

<!-- input要素（自己完結） -->
<input type="text" name="username">

    type属性で入力の種類を指定

    type属性を使って、どんな種類の入力なのかを指定します。

    📘 type属性比較フォーム

    テキスト入力（text）

    一番基本的な文字入力欄です。

    <input type="text" name="username" placeholder="ユーザー名を入力">

    属性の説明： - type="text": テキスト（文字）入力を指定 - name="username": データの名前（重要！後で詳しく説明） - placeholder: 入力ヒント（薄い文字で表示）

    メールアドレス入力（email）

    メールアドレス専用の入力欄です。

    <input type="email" name="email" placeholder="例: yamada@example.com">

    スマートフォンでは、自動的に@や.comが入力しやすいキーボードが表示されます！

    送信ボタン（submit）

    フォームを送信するためのボタンです。

    <input type="submit" value="送信">

    4. 実践：最初のフォームを作ってみよう

    ステップ1: HTMLファイル作成

    VSCodeで新しいファイルform-basic.htmlを作成しましょう。

    <!DOCTYPE html>
<html lang="ja">
<head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>フォーム練習</title>
</head>
<body>
    <h1>簡単なフォーム</h1>

    <form action="#" method="post">
        <p>
            <label>お名前:</label>
            <input type="text" name="name" placeholder="山田太郎">
        </p>

        <p>
            <label>メールアドレス:</label>
            <input type="email" name="email" placeholder="yamada@example.com">
        </p>

        <p>
            <input type="submit" value="送信">
        </p>
    </form>
</body>
</html>

    2. Live Serverで開く
    3. フォームに何か入力して「送信」ボタンを押してみる

    何が起こりましたか？

    現在はaction="#"なので、エラーになります。これは正常な動作です！

    5. name属性の重要性

    なぜname属性が必要？

    フォームでデータを送信するとき、サーバーは「どのデータが何なのか」を知る必要があります。

    <input type="text" name="username">
<input type="email" name="email">

    送信されるデータのイメージ： 

    username=山田太郎&amp;email=yamada@example.com

    📘 name属性の流れ確認フォーム

    name属性のルール

    • 英数字とアンダースコアを使用
    • 日本語は使わない
    • 分かりやすい名前をつける
    <!-- ✅ 良い例 -->
<input type="text" name="user_name">
<input type="email" name="email_address">

<!-- ❌ 悪い例 -->
<input type="text" name="名前">
<input type="email" name="a">

    6. 実際の動作を体験してみよう

    ステップ1: GETメソッドで送信体験

    先ほど作ったフォームのmethodをgetに変更してみましょう。

    <form action="#" method="get">

    ステップ2: 送信して結果確認

    フォームに入力して送信すると、URLが以下のように変わります：

    form-basic.html?name=山田太郎&amp;email=yamada@example.com

    これがGETメソッドでのデータ送信です！ 「Webの仕組み」で学んだ内容が実際に確認できましたね。

    7. よくある質問

    Q1: action="#"って何ですか？

    A: 現在のページ自身を指します。今回は練習なので、データを受け取るサーバーがないため、このような書き方をしています。

    Q2: placeholderの文字が薄いのはなぜ？

    A: placeholderは「入力ヒント」なので、実際の入力と区別するため、薄い色で表示されます。文字を入力すると消えます。

    Q3: submitボタンを押しても画面が変わらないことがあります

    A: action="#"の場合、同じページに戻ってくるので、一見何も起こらないように見えます。これは正常です！
