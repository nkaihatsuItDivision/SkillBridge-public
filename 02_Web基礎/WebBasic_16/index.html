    1. HTMLバリデーションとは？

    バリデーション（検証）の重要性

    バリデーションとは、ユーザーが入力したデータが正しい形式かどうかをチェックすることです。

    なぜバリデーションが必要？

    ユーザーの立場から： - 送信前にエラーが分かる → やり直しの手間が減る - 何を入力すればいいか明確 → 迷わない - エラーメッセージで正しい入力方法が分かる

    開発者の立場から： - 不正なデータの受信を防げる - サーバーへの負荷を軽減 - データベースの整合性を保てる

    HTMLバリデーションの特徴

    HTMLには標準でバリデーション機能が備わっています！

    <!-- JavaScriptを書かなくても、ブラウザが自動でチェック -->
<input type="email" required>

    このように簡単に設定でき、ブラウザが自動的にチェックしてくれます。

    2. required属性 - 必須入力の設定

    required属性の使い方

    必須入力にしたい項目には、required属性を追加します。

    <input type="text" name="name" required>
<input type="email" name="email" required>
<textarea name="message" required></textarea>

    実際の動作を見てみよう

    <form action="#" method="post">
    <p>
        <label>お名前（必須）:</label>
        <input type="text" name="name" placeholder="山田太郎" required>
    </p>

    <p>
        <label>メールアドレス（必須）:</label>
        <input type="email" name="email" placeholder="yamada@example.com" required>
    </p>

    <p>
        <label>電話番号（任意）:</label>
        <input type="tel" name="phone" placeholder="090-1234-5678">
    </p>

    <p>
        <input type="submit" value="送信">
    </p>
</form>

    requiredの効果

    1. 空欄のまま送信すると、ブラウザがエラーメッセージを表示
    2. 必須項目をすべて入力しないと送信できない
    3. ユーザーに分かりやすく必須項目を示せる

    ラベルで必須項目を明示

    <!-- ✅ 良い例：必須項目が分かりやすい -->
<label>お名前（必須）:</label>
<input type="text" name="name" required>

<label>メールアドレス（必須）:</label>
<input type="email" name="email" required>

<label>電話番号（任意）:</label>
<input type="tel" name="phone">

    3. pattern属性 - 入力パターンの制限

    pattern属性とは？

    pattern属性を使うと、正規表現で入力形式を制限できます。

    <input type="text" name="phone" pattern="[0-9]{3}-[0-9]{4}-[0-9]{4}" 
       placeholder="090-1234-5678">

    よく使うパターン集

    電話番号（ハイフンあり）

    <input type="tel" name="phone" 
       pattern="[0-9]{2,4}-[0-9]{2,4}-[0-9]{3,4}" 
       placeholder="090-1234-5678"
       title="電話番号をハイフン区切りで入力してください">

    郵便番号

    <input type="text" name="zipcode" 
       pattern="[0-9]{3}-[0-9]{4}" 
       placeholder="123-4567"
       title="郵便番号を123-4567の形式で入力してください">

    英数字のみ（ユーザーID）

    <input type="text" name="userid" 
       pattern="[a-zA-Z0-9]+" 
       placeholder="user123"
       title="英数字のみ使用できます">

    カタカナのみ

    <input type="text" name="kana" 
       pattern="[ァ-ヴ]+" 
       placeholder="ヤマダタロウ"
       title="カタカナで入力してください">

    title属性でヒントを提供

    title属性を追加すると、エラー時にカスタムメッセージを表示できます。

    <input type="text" name="phone" 
       pattern="[0-9]{3}-[0-9]{4}-[0-9]{4}"
       title="090-1234-5678の形式で入力してください">

    4. maxlength/minlength - 文字数制限

    文字数制限の設定

    最大文字数（maxlength）

    <!-- 最大20文字まで -->
<input type="text" name="name" maxlength="20" placeholder="お名前（20文字以内）">

<!-- 最大500文字まで -->
<textarea name="message" maxlength="500" placeholder="メッセージ（500文字以内）"></textarea>

    最小文字数（minlength）

    <!-- 最低8文字必要 -->
<input type="password" name="password" minlength="8" 
       placeholder="パスワード（8文字以上）">

    文字数制限の組み合わせ

    <!-- 8文字以上20文字以下 -->
<input type="password" name="password" 
       minlength="8" maxlength="20" 
       placeholder="パスワード（8-20文字）" required>

    文字数カウンター（応用）

    <label>自己紹介（200文字以内）:</label>
<textarea name="bio" maxlength="200" placeholder="自己紹介を入力してください..."></textarea>
<p><span id="charCount">0</span> / 200 文字</p>

    number型での数値範囲制限

    <!-- 年齢：18歳から100歳まで -->
<input type="number" name="age" min="18" max="100" 
       placeholder="年齢を入力" required>

<!-- 評価：1から5まで -->
<input type="number" name="rating" min="1" max="5" 
       placeholder="1-5で評価" required>

    5. placeholder属性の効果的な活用

    placeholderの基本

    placeholderは、入力欄に薄い文字で入力ヒントを表示します。

    <input type="text" placeholder="ここに入力してください">

    効果的なplaceholderの書き方

    ✅ 良い例：具体的で分かりやすい

    <input type="text" name="name" placeholder="例：山田太郎">
<input type="email" name="email" placeholder="例：yamada@example.com">
<input type="tel" name="phone" placeholder="例：090-1234-5678">
<input type="text" name="company" placeholder="例：株式会社サンプル">

    ❌ 悪い例：曖昧で分かりにくい

    <input type="text" name="name" placeholder="入力してください">
<input type="email" name="email" placeholder="メール">
<input type="tel" name="phone" placeholder="番号">

    placeholderのユーザビリティ向上テクニック

    1. 入力形式を明確に示す

    <input type="date" name="birthday" placeholder="1990-01-01">
<input type="time" name="meeting_time" placeholder="14:30">
<input type="url" name="website" placeholder="https://example.com">

    2. 文字数制限を示す

    <input type="text" name="title" maxlength="50" 
       placeholder="タイトル（50文字以内）">
<textarea name="description" maxlength="200" 
          placeholder="説明文（200文字以内）"></textarea>

    3. 必須・任意を明確に

    <input type="text" name="name" placeholder="お名前" required>
<input type="text" name="nickname" placeholder="ニックネーム（任意）">

    手順1: 基本構造から始めよう

    💡 確認ポイント: ブラウザで開いて、送信ボタンが表示されることを確認

    手順2: 1つずつ項目を追加

    最初は「お名前」だけ追加してみましょう：

    🚨 よくあるエラーと解決方法

    エラー1: バリデーションが効かない

    <!-- ❌ 間違い -->
<input type="text" required="true">

<!-- ✅ 正しい -->
<input type="text" required>

    エラー2: pattern属性で思った通りに動かない

    <!-- ❌ 間違い：引用符がない -->
<input type="text" pattern=[0-9]+>

<!-- ✅ 正しい：引用符で囲む -->
<input type="text" pattern="[0-9]+">

    エラー3: フォームが送信されない

    • name属性が設定されているか確認
    • required項目がすべて入力されているか確認
    • patternに一致する形式で入力されているか確認

    🎮 動作テストの手順

    フォームが完成したら、以下の方法で動作確認をしましょう：

    基本動作テスト

    1. 全項目空欄で送信 → 必須項目のエラーが表示される
    2. 1つずつ入力して送信 → エラーが消えていく
    3. すべて正しく入力して送信 → エラーが出ない

    バリデーション機能テスト

    1. メールアドレス: 「abc」→エラー、「abc@test.com」→OK
    2. フリガナ: 「やまだ」→エラー、「ヤマダ」→OK
    3. 電話番号: 「090123456789」→エラー、「090-1234-5678」→OK
    4. 年齢: 「200」→エラー、「25」→OK

    このフォームで使用したバリデーション機能

    1. required: 必須項目の設定
    2. maxlength: 文字数制限
    3. pattern: カタカナ入力制限、電話番号形式制限
    4. min/max: 年齢の数値範囲制限
    5. type="email": メールアドレス形式の自動チェック
    6. placeholder: 分かりやすい入力ヒント

    7. ユーザーフレンドリーなフォーム設計のポイント

    設計の基本原則

    1. 明確性（Clarity）

    • 何を入力すればいいか一目で分かる
    • 必須項目と任意項目を明確に区別
    • エラーメッセージが具体的

    2. 効率性（Efficiency）

    • 入力項目は最小限に
    • 適切な入力タイプを選択
    • タブキーでスムーズに移動できる

    3. 許容性（Forgiveness）

    • 入力ミスを事前に防ぐ
    • エラーが発生しても修正しやすい
    • ユーザーが迷わない設計

    ラベルとplaceholderの使い分け

    エラーメッセージのカスタマイズ

    8. よくある質問

    Q1: required属性とJavaScriptバリデーションの違いは？

    A: HTMLのrequired属性はブラウザ標準機能で、JavaScriptが無効でも動作します。JavaScriptバリデーションはより柔軟なカスタマイズが可能ですが、HTMLバリデーションの方が簡単で確実です。

    Q2: pattern属性の正規表現が難しいです

    A: 最初はよく使うパターンを覚えることから始めましょう： - [0-9]+ : 数字のみ - [a-zA-Z]+ : 英字のみ - [a-zA-Z0-9]+ : 英数字のみ - [ァ-ヴ]+ : カタカナのみ

    Q3: すべての入力項目にバリデーションを付けるべき？

    A: 必要な項目のみにバリデーションを設定しましょう。過度なバリデーションは、ユーザーにとって煩わしく感じられることがあります。

    Q4: placeholderのテキストが見えにくいです

    A: placeholderは入力ヒントなので、意図的に薄い色になっています。重要な情報はlabel要素に記載してください。
