    🏗️ MVCアーキテクチャとは何か

    💡 MVCアーキテクチャの基本概念

    MVCとは、Webアプリケーションを3つの役割に分離する設計パターンです：

    • Model（モデル）：データとビジネスロジック
    • View（ビュー）：ユーザーインターフェース
    • Controller（コントローラー）：リクエスト処理と制御

    📘 MVCアーキテクチャ概要

    🔍 なぜMVCアーキテクチャが重要なのか

    従来開発の問題点

    従来開発では全ての処理が1つのファイルに混在：
┌─────────────────────────────────┐
│ 1つのファイルに全て混在           │
├─────────────────────────────────┤
│ • ユーザー入力の処理              │
│ • データベースアクセス            │
│ • ビジネスロジック                │
│ • HTML生成処理                   │
│ • エラーハンドリング              │
│ • その他諸々...                  │
└─────────────────────────────────┘

    問題点： - コードの保守が困難 - バグの原因特定に時間がかかる - チーム開発での分担が難しい - 機能追加時の影響範囲が不明

    MVCアーキテクチャの解決

    責務を明確に分離：
┌──────────────┐ ┌──────────────┐ ┌──────────────┐
│    Model     │ │     View     │ │  Controller  │
├──────────────┤ ├──────────────┤ ├──────────────┤
│ • データ管理  │ │ • 画面表示    │ │ • 入力処理    │
│ • 計算処理    │ │ • UI設計     │ │ • 判定制御    │
│ • DB連携     │ │ • HTML生成   │ │ • データ受渡  │
└──────────────┘ └──────────────┘ └──────────────┘

    効果： - ✅ 保守しやすいコード - ✅ バグの原因が特定しやすい - ✅ チーム開発での分担が明確 - ✅ 機能追加の影響が限定的

    🎯 MVCアーキテクチャの価値

    MVCアーキテクチャを理解することで：

    1. Week 7までの技術が統合される：HTML、Java、SQLが適切な場所に配置される
    2. Spring Bootの簡潔さが理解できる：なぜ簡単に開発できるのかが分かる
    3. 実装の見通しが立つ：Week 8.3以降で何をするかが明確になる
    4. 現場での価値が実感できる：企業開発で重視される設計思想を習得

    🎯 Model-View-Controllerの詳細理解

    📊 Model（モデル）：データとビジネスロジック

    Modelの責務： - データの管理と操作 - ビジネスルールの実装 - データベースとの連携

    📘 コントローラー層の詳細

    🔄 3つの層の相互作用

    MVCの美しい協調：

    1. ユーザーがブラウザでアクセス → Controllerが受信
    2. Controllerが必要なModelを呼び出し
    3. Modelがデータベースからデータを取得
    4. Controllerが取得データをViewに渡す
    5. Viewが美しいHTMLを生成
    6. 完成したページがユーザーに表示

    各層の独立性： - Modelを変更してもViewに影響しない - Viewを変更してもModelに影響しない - Controllerが両者を適切に仲介

    これにより、保守しやすく、拡張しやすいシステムが実現されます。

    🚀 Spring BootでのMVC実装方法

    🌟 Spring BootがMVCを簡単にする理由

    Spring Bootは、MVCアーキテクチャの実装をアノテーションで簡単にします：

    Controller層の実装

    @Controller  // 「これはController」と宣言するだけ
public class StudentController {

    @GetMapping("/students")  // 「GETリクエストを受け取る」と宣言
    public String showStudents(Model model) {
        // 実際の処理を書くだけ
        return "students/index";  // 表示するViewを指定
    }
}

    Model層の実装

    @Service  // 「これはビジネスロジック」と宣言
public class StudentService {

    @Autowired  // 「自動で接続して」と宣言
    private StudentMapper studentMapper;

    public List<Student> findAll() {
        return studentMapper.findAll();  // MyBatisが自動実行
    }
}

@Mapper  // 「これはデータアクセス」と宣言
public interface StudentMapper {
    List<Student> findAll();  // XMLのSQLが自動実行される
}

    View層の実装（Thymeleaf）

    <!-- 通常のHTMLにちょっとだけThymeleaf記法を追加 -->
<div th:each="student : ${students}">
    <p th:text="${student.name}"></p>
</div>

    🎯 アノテーションの威力

    従来開発では複雑な設定ファイルが必要でしたが、Spring Bootでは：

    • @Controller → ControllerクラスとしてSpringが認識
    • @Service → Serviceクラスとしてビジネスロジックを管理
    • @Mapper → MyBatisでSQLを自動実行
    • @GetMapping → HTTPリクエストを自動受信
    • @Autowired → 必要なクラスを自動で接続

    結果：設定地獄から解放され、実装に集中できます！

    🌟 エラーハンドリングの基本概念

    MVCアーキテクチャでは、エラーも適切に分離されます：

    Model層のエラー：データベースアクセスエラー、データ不整合等
    View層のエラー：テンプレート構文エラー、表示データエラー等
    Controller層のエラー：リクエスト処理エラー、パラメータエラー等

    Spring Bootでは、これらのエラーを統一的に処理する仕組みが提供されています（Week 9で詳しく学習）。
