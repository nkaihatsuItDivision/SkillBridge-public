    🌟 Week 8.5からの技術発展

    これまでの学習成果

    Week 8.5では、学生管理システムのEntityクラス設計を完全にマスターしました：

    • ✅ @Entity、@Table、@Id、@Column: データベーステーブルマッピング
    • ✅ Lombok活用: @Data、@Builder等による効率的な実装
    • ✅ Bean Validation: @NotNull、@Email等によるデータ検証

    今回のMapper実装で実現すること

    Week 8.5のEntity技術を基盤として、以下を実現します：

    // これまで習得したEntity技術
Student student = Student.builder()
    .name("田中太郎")
    .email("tanaka@example.com")
    .birthDate(LocalDate.of(2001, 4, 1))
    .build();

    なぜXML設定が重要なのか？

    @Mapperアノテーションでインターフェースを定義しました。しかし、実際のSQL文はどこに記述するのでしょうか？

    MyBatisでは、SQL文をXMLファイルで管理します。これにより以下のメリットが得られます：

    1. SQL文の分離管理: JavaコードとSQL文を明確に分離
    2. 複雑なクエリ対応: 複数テーブル結合、条件分岐等の高度なSQL
    3. 保守性向上: SQL文の変更がJavaコードに影響しない
    4. DBA連携: データベース専門家との協業が容易

    動的SQLの主要タグ

    1. <if>タグ - 条件分岐

    <if test="name != null and name != ''">
    AND name LIKE CONCAT('%', #{name}, '%')
</if>
    - 用途: 条件が成立した場合のみSQL文を追加 - test属性: 条件式（Javaのboolean式）

    2. <where>タグ - WHERE句の自動調整

    <where>
    <if test="condition1">AND ...</if>
    <if test="condition2">AND ...</if>
</where>
    - 自動調整: 最初のANDを除去、条件がない場合WHERE句を除去

    3. <foreach>タグ - 繰り返し処理

    <foreach collection="students" item="student" separator=";">
    UPDATE students SET name = #{student.name} WHERE id = #{student.id}
</foreach>
    - collection: 繰り返し対象のコレクション - item: 各要素の変数名 - separator: 各処理の区切り文字

    エラーハンドリングの実装

    データベース操作では、様々なエラーが発生する可能性があります。適切なエラーハンドリングを実装しましょう：

    Serviceクラスでのエラーハンドリング例

    @Service
@Transactional
public class StudentService {

    @Autowired
    private StudentMapper studentMapper;

    public Student findById(Long id) {
        try {
            Student student = studentMapper.findById(id);
            if (student == null) {
                throw new StudentNotFoundException("学生ID " + id + " が見つかりません");
            }
            return student;
        } catch (DataAccessException e) {
            throw new DatabaseException("データベースアクセスエラーが発生しました", e);
        }
    }

    public void saveStudent(Student student) {
        try {
            // バリデーション（Week 8.5のEntity機能を活用）
            validateStudent(student);

            if (student.getId() == null) {
                studentMapper.insert(student);
            } else {
                studentMapper.update(student);
            }
        } catch (DuplicateKeyException e) {
            throw new StudentDuplicateException("同じメールアドレスの学生が既に存在します");
        } catch (DataAccessException e) {
            throw new DatabaseException("学生情報の保存に失敗しました", e);
        }
    }

    private void validateStudent(Student student) {
        // Week 8.5のBean Validation機能を活用
        Set<ConstraintViolation<Student>> violations = validator.validate(student);
        if (!violations.isEmpty()) {
            throw new ValidationException("入力データに不正があります");
        }
    }
}

    ステップ3: Serviceクラスの実装

    💡 ヒント3：エラーハンドリング付きService層 
    @Service
@Transactional
public class StudentService {

    @Autowired
    private StudentMapper studentMapper;

    @Transactional(readOnly = true)
    public List<Student> searchStudents(String name, Long departmentId, 
                                       Double minGpa, Double maxGpa) {
        try {
            return studentMapper.findByConditions(name, departmentId, minGpa, maxGpa);
        } catch (DataAccessException e) {
            throw new DatabaseException("学生検索中にエラーが発生しました", e);
        }
    }

    @Transactional(readOnly = true)
    public Student getStudentById(Long id) {
        if (id == null || id <= 0) {
            throw new IllegalArgumentException("無効な学生IDです");
        }

        Student student = studentMapper.findById(id);
        if (student == null) {
            throw new StudentNotFoundException("学生ID " + id + " が見つかりません");
        }
        return student;
    }

    public void saveStudent(Student student) {
        // Week 8.5のEntity機能を活用したバリデーション
        validateStudent(student);

        try {
            if (student.getId() == null) {
                studentMapper.insert(student);
            } else {
                Student existing = studentMapper.findById(student.getId());
                if (existing == null) {
                    throw new StudentNotFoundException("更新対象の学生が見つかりません");
                }
                studentMapper.update(student);
            }
        } catch (DuplicateKeyException e) {
            throw new StudentDuplicateException("同じメールアドレスの学生が既に存在します");
        }
    }

    private void validateStudent(Student student) {
        // Week 8.5のBean Validation活用
        if (student.getName() == null || student.getName().trim().isEmpty()) {
            throw new ValidationException("学生名は必須です");
        }
        if (student.getEmail() == null || !isValidEmail(student.getEmail())) {
            throw new ValidationException("有効なメールアドレスを入力してください");
        }
    }

    @Transactional(readOnly = true)
    public Map<String, Object> getStatistics() {
        Map<String, Object> stats = new HashMap<>();
        stats.put("totalStudents", studentMapper.countAll());
        stats.put("averageGpa", studentMapper.getAverageGpa());
        return stats;
    }
}
