    💔 従来のJava Web開発：開発者を苦しめる複雑さ

    なぜ多くの開発者がJava Web開発を「難しい」と感じるのか？

    Spring Bootが登場する前、Java Web開発は異常に複雑でした。

    従来開発の現実：endless設定地獄

    1. 膨大な設定ファイル群

    従来のJava Web開発では、以下のような設定ファイルを手動で作成・管理する必要がありました：

    <!-- web.xml: 数百行の複雑な設定 -->
<?xml version="1.0" encoding="UTF-8"?>
<web-app xmlns="http://xmlns.jcp.org/xml/ns/javaee"
         xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance"
         xsi:schemaLocation="..."
         version="4.0">

  <servlet>
    <servlet-name>StudentServlet</servlet-name>
    <servlet-class>com.example.StudentServlet</servlet-class>
  </servlet>

  <servlet-mapping>
    <servlet-name>StudentServlet</servlet-name>
    <url-pattern>/students</url-pattern>
  </servlet-mapping>

  <!-- さらに数十〜数百行の設定が必要... -->
</web-app>

    <!-- applicationContext.xml: フレームワーク設定 -->
<?xml version="1.0" encoding="UTF-8"?>
<beans xmlns="http://www.springframework.org/schema/beans">

  <bean id="dataSource" class="org.apache.commons.dbcp2.BasicDataSource">
    <property name="driverClassName" value="com.mysql.cj.jdbc.Driver"/>
    <property name="url" value="jdbc:mysql://localhost:3306/sampledb"/>
    <property name="username" value="root"/>
    <property name="password" value="password"/>
  </bean>

  <!-- さらに数十のBean定義が必要... -->
</beans>

    これらの設定ファイルだけで数百行〜千行に及ぶことも珍しくありませんでした。

    2. サーバー構築・デプロイの悪夢

    従来の開発では、以下の作業を手動で行う必要がありました：

    サーバー準備（毎回）: 1. Apache Tomcatサーバーを別途ダウンロード・インストール 2. 複雑なサーバー設定ファイルの編集 3. JDBCドライバの手動配置 4. メモリ設定・ポート設定の調整

    デプロイ作業（毎回）: 1. WARファイルの手動作成 2. サーバーへの手動デプロイ 3. サーバー再起動と動作確認 4. エラー発生時の複雑なログ調査

    📘 従来型とSpring Bootの比較

    3. 開発効率の深刻な低下

    1. 設定の完全自動化

    従来の複雑な設定: 

    <!-- 数百行のXML設定が必要だった -->

</bean>

    Spring Bootでの設定:

spring.datasource.url=jdbc:mysql://localhost:3306/sampledb

spring.datasource.password=password

    設定量が10分の1以下に削減！

    2. 組み込みサーバーによる簡単実行

    従来の実行方法: 1. Tomcatサーバーを別途起動 2. WARファイルを作成 3. サーバーにデプロイ 4. ブラウザで動作確認

    Spring Bootの実行方法: 

    // たった1つのコマンドで実行！
public static void main(String[] args) {

@Controller
public class StudentController {

    @PostMapping("/students")
    public String create(@Valid @ModelAttribute StudentForm form, 
                        BindingResult result, Model model) {
        if (result.hasErrors()) {
            return "students/new";
        }
        studentService.create(form);
        return "redirect:/students";
    }

    @PatchMapping("/students/{id}")
    public String update(@PathVariable Long id, 
                        @Valid @ModelAttribute StudentForm form,
                        BindingResult result) {
        if (result.hasErrors()) {
            return "students/edit";
        }
        studentService.update(id, form);
        return "redirect:/students";
    }
}
