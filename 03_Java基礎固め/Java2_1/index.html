    🔍 配列の仕組みを詳しく理解しよう

    配列ってそもそも何？

    配列の定義 配列とは、同じデータ型の値を連続したメモリ領域に格納するデータ構造です。

    📘 配列のメモリ図

    [メモリ上での配列のイメージ図]
int[] numbers = {10, 20, 30, 40, 50};

メモリアドレス: 1000  1004  1008  1012  1016
値:           [10]  [20]  [30]  [40]  [50]
インデックス:    0     1     2     3     4

    配列の特徴

    1. 連続メモリ配置: 要素が隣り合ったメモリ位置に配置
    2. 固定サイズ: 作成時にサイズが決まり、後から変更不可
    3. インデックスアクセス: 番号（インデックス）で直接要素にアクセス
    4. 高速アクセス: インデックスがあれば瞬時に要素にアクセス可能

    なぜ配列は高速なのか？

    int[] numbers = {10, 20, 30, 40, 50};
int value = numbers[3];  // なぜこれが高速？

    計算式: 配列の開始アドレス + (インデックス × データサイズ) - numbers[3] = 1000番地 + (3 × 4バイト) = 1012番地 - 計算一回で直接アクセス！

    制限1: サイズが固定される問題

    新しいクラス ArrayLimitationDemo を作成してください：

    public class ArrayLimitationDemo {
    public static void main(String[] args) {
        System.out.println(&quot;=== 配列の制限を体験してみよう ===\n&quot;);

        // 制限1: サイズ固定の問題
        System.out.println(&quot;【制限1: サイズが固定される問題】&quot;);

        // 学生名を保存する配列（3人分で作成）
        String[] students = new String[3];
        students[0] = &quot;田中&quot;;
        students[1] = &quot;佐藤&quot;; 
        students[2] = &quot;山田&quot;;

        System.out.println(&quot;現在の学生一覧:&quot;);
        for (int i = 0; i &lt; students.length; i++) {
            System.out.println((i+1) + &quot;. &quot; + students[i]);
        }

        System.out.println(&quot;\n🤔 問題発生！&quot;);
        System.out.println(&quot;新しい学生「鈴木」さんが転入してきました。&quot;);
        System.out.println(&quot;でも配列のサイズは3で固定...&quot;);
        System.out.println(&quot;どうすればいいでしょうか？\n&quot;);

        // 解決方法1: 新しい配列を作って要素をコピー
        System.out.println(&quot;【解決方法1: 新しい配列作成 + コピー】&quot;);
        String[] newStudents = new String[4];  // サイズ4の新配列

        // 古い配列から新しい配列にコピー
        for (int i = 0; i &lt; students.length; i++) {
            newStudents[i] = students[i];
        }
        newStudents[3] = &quot;鈴木&quot;;  // 新しい学生を追加

        System.out.println(&quot;新しい配列での学生一覧:&quot;);
        for (int i = 0; i &lt; newStudents.length; i++) {
            System.out.println((i+1) + &quot;. &quot; + newStudents[i]);
        }

        System.out.println(&quot;\n💭 この方法の問題点:&quot;);
        System.out.println(&quot;・ 毎回新しい配列を作成する必要がある&quot;);
        System.out.println(&quot;・ 全要素をコピーする手間がかかる&quot;);
        System.out.println(&quot;・ メモリを2倍使ってしまう期間がある&quot;);
        System.out.println(&quot;・ コードが複雑になる\n&quot;);
    }
}

        for (int i = insertIndex; i &lt; numbers.length; i++) {
            newNumbers[i + 1] = numbers[i];

        System.out.println(&quot;挿入後の配列: &quot; + java.util.Arrays.toString(newNumbers));

        System.out.println(&quot;\n💭 この処理の問題点:&quot;);
        System.out.println(&quot;・ 新しい配列を作成する必要がある&quot;);
        System.out.println(&quot;・ 要素を一つ一つずらす処理が必要&quot;);

    演習1: 動的なデータ管理の困難さ

    新しいクラス DynamicDataChallenge を作成してください：

    import java.util.Scanner;

public class DynamicDataChallenge {
    public static void main(String[] args) {
        Scanner scanner = new Scanner(System.in);

        System.out.println(&quot;=== ToDoリスト管理（配列版）===&quot;);
        System.out.println(&quot;配列の制限を体験してみましょう\n&quot;);

        // 最大10個のToDoを管理する配列
        String[] todos = new String[10];
        int todoCount = 0;  // 現在のToDo数を管理

        while (true) {
            System.out.println(&quot;\n--- ToDoリスト ---&quot;);
            if (todoCount == 0) {
                System.out.println(&quot;（まだToDoがありません）&quot;);
            } else {
                for (int i = 0; i &lt; todoCount; i++) {
                    System.out.println((i + 1) + &quot;. &quot; + todos[i]);
                }
            }

            System.out.println(&quot;\n1. ToDo追加&quot;);
            System.out.println(&quot;2. ToDo削除&quot;);
            System.out.println(&quot;3. 終了&quot;);
            System.out.print(&quot;選択してください: &quot;);

            int choice = scanner.nextInt();
            scanner.nextLine();  // 改行を消費

            switch (choice) {
                case 1:
                    // ToDo追加
                    if (todoCount &gt;= todos.length) {
                        System.out.println(&quot;⚠️ エラー: これ以上ToDoを追加できません！&quot;);
                        System.out.println(&quot;配列のサイズが &quot; + todos.length + &quot; で固定されているためです。&quot;);
                    } else {
                        System.out.print(&quot;新しいToDoを入力してください: &quot;);
                        String newTodo = scanner.nextLine();
                        todos[todoCount] = newTodo;
                        todoCount++;
                        System.out.println(&quot;✅ ToDoを追加しました！&quot;);
                    }
                    break;

                case 2:
                    // ToDo削除
                    if (todoCount == 0) {
                        System.out.println(&quot;削除するToDoがありません。&quot;);
                    } else {
                        System.out.print(&quot;削除するToDoの番号を入力してください: &quot;);
                        int deleteIndex = scanner.nextInt() - 1;

                        if (deleteIndex &lt; 0 || deleteIndex &gt;= todoCount) {
                            System.out.println(&quot;⚠️ 無効な番号です。&quot;);
                        } else {
                            // 削除処理（配列では複雑！）
                            System.out.println(&quot;削除中...&quot;);

                            // 削除する要素以降を前にずらす
                            for (int i = deleteIndex; i &lt; todoCount - 1; i++) {
                                todos[i] = todos[i + 1];
                            }

                            todos[todoCount - 1] = null;  // 最後の要素をクリア
                            todoCount--;

                            System.out.println(&quot;✅ ToDoを削除しました！&quot;);
                            System.out.println(&quot;💭 配列での削除は要素をずらす処理が必要で複雑ですね...&quot;);
                        }
                    }
                    break;

                case 3:
                    System.out.println(&quot;プログラムを終了します。&quot;);
                    scanner.close();
                    return;

                default:
                    System.out.println(&quot;1-3の番号を選択してください。&quot;);
            }
        }
    }
}

    配列の制限を学びましたが、配列にも素晴らしい特徴があります：

    配列の優れた特徴

    1. 超高速アクセス: インデックス指定で瞬時にアクセス
    2. メモリ効率: 連続領域で効率的なメモリ使用
    3. シンプルな構造: 理解しやすく、学習コストが低い
    4. 基本データ型対応: intやdoubleなどを直接格納可能

    配列が最適な場面

    // 例1: 固定サイズのデータ（曜日、月など）
String[] days = {&quot;日&quot;, &quot;月&quot;, &quot;火&quot;, &quot;水&quot;, &quot;木&quot;, &quot;金&quot;, &quot;土&quot;};

// 例2: 数値計算やゲームのスコア管理
int[] scores = new int[10];  // 10ゲーム分のスコア

// 例3: 画像の画素データなど大量データの高速処理
int[][] pixels = new int[1920][1080];  // フルHD画像の画素

    まとめ: 適材適所の考え方

    配列が適している場面：
✓ サイズが決まっている データ

✓ 大量データの数値計算
✓ メモリ使用量を最小化したい場合

配列が適していない場面：
✗ サイズが変動するデータ
✗ 頻繁な挿入・削除が必要
✗ 動的なリスト管理
✗ ユーザー操作による要素の増減
