    今回は、この問題をgetter/setterメソッドで解決します！

    1. getter/setterメソッドの基本概念

    getter/setterとは何か

    getter（ゲッター）: フィールドの値を取得（Get）するメソッド setter（セッター）: フィールドの値を設定（Set）するメソッド

    public class Student {
    private String name;  // privateで保護

    // getter: nameの値を取得するメソッド
    public String getName() {
        return this.name;
    }

    // setter: nameの値を設定するメソッド
    public void setName(String name) {
        this.name = name;
    }
}

    なぜgetter/setterが必要なのか

    1. 制御されたアクセス

    // ❌ 直接アクセス：制御不可
public String name;  // 誰でも自由に変更可能

// ✅ getter/setter：制御可能
private String name;
public void setName(String name) {
    if (name != null && !name.trim().isEmpty()) {  // 検証
        this.name = name;
    } else {
        throw new IllegalArgumentException("名前は空にできません");
    }
}

    2. データの整合性保証

    private int age;

public void setAge(int age) {
    if (age >= 0 && age <= 150) {  // 妥当な年齢範囲
        this.age = age;
    } else {
        throw new IllegalArgumentException("年齢は0-150の範囲で設定してください");
    }
}

    3. 将来の変更に対応

    // 内部実装を変更してもインターフェースは維持
private String firstName;
private String lastName;

public String getName() {  // 内部では2つのフィールドから組み立て
    return this.firstName + " " + this.lastName;
}

    📘 GetterとSetterの概念

    2. 基本的なgetter/setterの実装

    手動でのgetter/setter作成

    public class Student {
    private String name;
    private int age;
    private double gpa;

    // nameのgetter
    public String getName() {
        return this.name;
    }

    // nameのsetter
    public void setName(String name) {
        this.name = name;
    }

    // ageのgetter
    public int getAge() {
        return this.age;
    }

    // ageのsetter
    public void setAge(int age) {
        this.age = age;
    }

    // gpaのgetter
    public double getGpa() {
        return this.gpa;
    }

    // gpaのsetter
    public void setGpa(double gpa) {
        this.gpa = gpa;
    }
}

    2. コレクションの防御的コピー

    import java.util.*;

public class Student {
    private List<String> subjects;

    // ❌ 危険：内部リストの直接公開
    public List<String> getSubjects() {
        return this.subjects;  // 外部から変更される可能性
    }

    // ✅ 安全：防御的コピー
    public List<String> getSubjects() {
        if (this.subjects == null) {
            return new ArrayList<>();
        }
        return new ArrayList<>(this.subjects);  // コピーを返す
    }

    // ✅ 安全：防御的コピー
    public void setSubjects(List<String> subjects) {
        if (subjects == null) {
            this.subjects = new ArrayList<>();
        } else {
            this.subjects = new ArrayList<>(subjects);  // コピーを保存
        }
    }
}

    📝 練習課題

    余力がある方は、以下の課題にチャレンジしてみてください：

    基礎練習

    1. 図書クラス（Book）の作成
    2. フィールド：タイトル、著者、価格、出版日、ISBN
    3. 検証：タイトル・著者必須、価格正数、ISBN形式チェック

    4. 読み取り専用：出版からの日数、価格帯（安い・普通・高い）

    5. 銀行口座クラス（BankAccount）の作成

    6. フィールド：口座番号、残高、口座名義、作成日
    7. 検証：口座番号形式、残高非負数、名義必須
    8. 読み取り専用：口座年数、残高表示（***円の形式）

    応用練習

    1. 商品クラス（Product）の作成
    2. フィールド：商品ID、名前、価格、在庫数、カテゴリリスト
    3. 検証：全フィールドの妥当性検証
    4. 防御的コピー：カテゴリリストの安全な管理

    5. 読み取り専用：在庫状況（豊富・少量・品切れ）、税込価格

    6. 従業員クラス（Employee）の作成

    7. フィールド：社員ID、名前、部署、給与、入社日、スキルリスト
    8. 検証：各フィールドのビジネスルール検証
    9. 読み取り専用：勤続年数、年収、スキル数

    発展練習

    1. オリジナルクラスの設計
    2. 自分の興味のある分野のクラスを設計
    3. 今回学んだ全ての技術を活用
    4. 実用的な検証とプロパティを実装
