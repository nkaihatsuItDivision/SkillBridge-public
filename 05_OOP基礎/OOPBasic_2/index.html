    public class Student {
    String name;
    int score;

    String getGrade() {
        if (score >= 80) return "優";
        if (score >= 70) return "良";
        if (score >= 60) return "可";

    前回は1人の学生（田中さん）でクラスの基本を理解しました。 今回は複数の学生を管理する場面で、クラスとインスタンスの真の価値を体験します！

    🏗️ 設計図と実体の関係

    身近な例：家の設計図

    📘 クラスとインスタンスの設計図イメージ

    現実世界では、以下の関係があります：

    設計図（1つ）
├── 田中家の家（実体1）
├── 佐藤家の家（実体2）
└── 山田家の家（実体3）
    • 設計図：家の構造や機能を定義（1つ）
    • 実際の家：設計図をもとに建てられた家（複数可能）
    • 各家の独立性：田中家の家を赤く塗っても、佐藤家の家は変わらない

    プログラムでの対応関係

    // 設計図 = クラス（1つ）
public class Student {
    String name;     // 名前のフィールド
    int score;       // 得点のフィールド

    // 機能も定義
    String getGrade() { ... }
    void introduce() { ... }
}

// 実体 = インスタンス（複数作成可能）
Student student1 = new Student();  // 田中さん
Student student2 = new Student();  // 佐藤さん
Student student3 = new Student();  // 山田さん

    重要な理解： - 1つのクラスから複数のインスタンスを作成できる - 各インスタンスは独立したデータを持つ - 同じ機能（メソッド）を使える

    ステップ2: Studentクラスの作成

    src フォルダを右クリック → 新規 → クラス - パッケージ：空欄のまま - 名前：Student - 完了をクリック

    public class Student {
    // フィールド（各学生が持つデータ）
    String name;
    int score;

    // メソッド（各学生ができること）
    String getGrade() {
        if (score >= 80) return "優";
        if (score >= 70) return "良";
        if (score >= 60) return "可";
        return "不可";
    }

    void introduce() {
        System.out.println("私は" + name + "です。得点は" + score + "点、評価は" + getGrade() + "です。");
    }
}

    ステップ3: 複数インスタンスを作成するMainクラス

    新しいクラスを作成：StudentMain

    public class StudentMain {
    public static void main(String[] args) {
        // 設計図（Studentクラス）から実際の学生オブジェクトを作成

        // 1人目の学生：田中さん
        Student student1 = new Student();
        student1.name = "田中太郎";
        student1.score = 85;

        // 2人目の学生：佐藤さん
        Student student2 = new Student();
        student2.name = "佐藤花子";
        student2.score = 92;

        // 3人目の学生：山田さん
        Student student3 = new Student();
        student3.name = "山田一郎";
        student3.score = 76;

        // 各学生に自己紹介してもらう
        System.out.println("=== 学生紹介 ===");
        student1.introduce();
        student2.introduce();
        student3.introduce();

        // 個別に成績確認
        System.out.println("\n=== 個別成績確認 ===");
        System.out.println(student1.name + "さんの評価: " + student1.getGrade());
        System.out.println(student2.name + "さんの評価: " + student2.getGrade());
        System.out.println(student3.name + "さんの評価: " + student3.getGrade());
    }
}

    🆚 手続き型との比較：複数学生管理の困難さ

    手続き型で複数学生を管理する場合

    public class ProceduralStudentManager {
    public static void main(String[] args) {
        // 学生データをバラバラの変数で管理
        String student1Name = "田中太郎";
        int student1Score = 85;

        String student2Name = "佐藤花子";
        int student2Score = 92;

        String student3Name = "山田一郎";
        int student3Score = 76;

        // 成績評価も個別に処理
        String student1Grade = getGrade(student1Score);
        String student2Grade = getGrade(student2Score);
        String student3Grade = getGrade(student3Score);

        // 表示も個別に処理
        System.out.println("私は" + student1Name + "です。得点は" + student1Score + "点、評価は" + student1Grade + "です。");
        System.out.println("私は" + student2Name + "です。得点は" + student2Score + "点、評価は" + student2Grade + "です。");
        System.out.println("私は" + student3Name + "です。得点は" + student3Score + "点、評価は" + student3Grade + "です。");
    }

    static String getGrade(int score) {
        if (score >= 80) return "優";
        if (score >= 70) return "良";
        if (score >= 60) return "可";
        return "不可";
    }
}

    手続き型の問題点： - 変数が散らばる：student1Name, student2Name...と管理が複雑 - 関連データの分離：名前と得点の関係がコード上不明確 - 処理の重複：同じような処理を何度も書く必要 - 拡張の困難：学生が増えるたびに大量のコード追加

    OOP（Studentクラス）による解決

    public class OOPStudentManager {
    public static void main(String[] args) {
        // データと機能がまとまったオブジェクトとして管理
        Student[] students = {
            createStudent("田中太郎", 85),
            createStudent("佐藤花子", 92),
            createStudent("山田一郎", 76)
        };

        // シンプルなループで一括処理
        for (Student student : students) {
            student.introduce();  // 各オブジェクトが自分で紹介
        }
    }

    static Student createStudent(String name, int score) {
        Student student = new Student();
        student.name = name;
        student.score = score;
        return student;
    }
}

    OOPの利点： - データの集約：関連するデータが1つのオブジェクトにまとまる - 機能の内蔵：オブジェクト自身が必要な機能を持つ - 統一した処理：同じ方法で全てのオブジェクトを扱える - 簡単な拡張：新しい学生は新しいインスタンスを作るだけ

    結論：複数の同じような「モノ」を扱う場面では、OOPの威力が絶大です！

    💡 完成版のコード（クリックで表示） 
    public class ClassroomManager {
    public static void main(String[] args) {
        // 5人の学生インスタンスを作成
        Student[] classroom = new Student[5];

        // 学生データの設定
        classroom[0] = createStudent("田中太郎", 85);
        classroom[1] = createStudent("佐藤花子", 92);
        classroom[2] = createStudent("山田一郎", 76);
        classroom[3] = createStudent("鈴木美咲", 88);
        classroom[4] = createStudent("高橋健太", 63);

        // クラス全体の管理
        System.out.println("=== " + classroom.length + "人のクラス ===");

        // 全員の自己紹介
        for (int i = 0; i < classroom.length; i++) {
            System.out.print((i + 1) + ". ");
            classroom[i].introduce();
        }

        // 成績統計
        showClassStatistics(classroom);
    }

    // 学生インスタンス作成ヘルパーメソッド
    static Student createStudent(String name, int score) {
        Student student = new Student();
        student.name = name;
        student.score = score;
        return student;
    }

    // クラス統計表示メソッド
    static void showClassStatistics(Student[] students) {
        System.out.println("\n=== クラス統計 ===");

        int total = 0;
        int excellentCount = 0;  // 優の人数

        for (Student student : students) {
            total += student.score;
            if (student.getGrade().equals("優")) {
                excellentCount++;
            }
        }

        double average = (double) total / students.length;

        System.out.println("平均得点: " + String.format("%.1f", average) + "点");
        System.out.println("優の人数: " + excellentCount + "人");
        System.out.println("合格者数: " + getPassedCount(students) + "人");
    }

    // 合格者数計算メソッド
    static int getPassedCount(Student[] students) {
        int count = 0;
        for (Student student : students) {
            if (!student.getGrade().equals("不可")) {
                count++;
            }
        }
        return count;
    }
}

## 💡 よくある質問

### **Q1: なぜ毎回`new`を書く必要があるのですか？**

**A1:** `new`キーワードは「新しいインスタンスを作ってください」という明示的な指示です。

```java
Student student1;           // 変数の宣言（まだ実体はない）
student1 = new Student();   // 実際のインスタンスを作成して代入

    newがないと実体（オブジェクト）が作られないため、データを保存する場所がありません。毎回newすることで、独立した新しいオブジェクトが作成されます。

    Q2: クラスとインスタンスの違いがまだ分からない...

    A2: 身近な例で考えてみましょう：

    // 「車」という概念（クラス）
public class Car {
    String color;    // 色
    String brand;    // メーカー

    void start() {   // エンジンをかける機能
        System.out.println("エンジン始動");
    }
}

// 実際の車（インスタンス）
Car myCar = new Car();      // 私の車
myCar.color = "赤";
myCar.brand = "トヨタ";

Car yourCar = new Car();    // あなたの車
yourCar.color = "青";
yourCar.brand = "ホンダ";
    • クラス（Car）：「車とはこういうもの」という設計図
    • インスタンス（myCar, yourCar）：実際に存在する個別の車

    Q3: 同じクラスから作ったインスタンスは本当に独立していますか？

    A3: はい、完全に独立しています。実験で確認してみましょう：

    Student a = new Student();
Student b = new Student();

a.name = "Aさん";
a.score = 100;

// bのデータは変更していないので、初期値のまま
System.out.println("a.name: " + a.name);  // "Aさん"
System.out.println("b.name: " + b.name);  // null（初期値）

    各インスタンスは別々のメモリ領域を使用するため、片方の変更が他方に影響することは絶対にありません。

    Q4: 何個までインスタンスを作れますか？

    A4: メモリが許す限り何個でも作成できます。

    // 1000人の学生も可能
Student[] students = new Student[1000];
for (int i = 0; i < 1000; i++) {
    students[i] = new Student();
    students[i].name = "学生" + (i + 1);
    students[i].score = (int)(Math.random() * 100);
}

    実際のシステムでは数千、数万のオブジェクトを扱うことも珍しくありません。
