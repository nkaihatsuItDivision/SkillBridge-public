    🚨 まず体験：publicフィールドの問題点

    危険なpublicフィールドの実装

    まずは、なぜカプセル化が必要なのかを実際に体験してみましょう：

    public class DangerousStudent {
    // ❌ 危険：全て public で外部から直接アクセス可能
    public String name;
    public int age;
    public double gpa;
    public List<String> subjects;

    public DangerousStudent(String name, int age, double gpa) {
        this.name = name;
        this.age = age;
        this.gpa = gpa;
        this.subjects = new ArrayList<>();
    }

    public void showInfo() {
        System.out.println("名前：" + this.name + "、年齢：" + this.age + "歳、GPA：" + this.gpa);
    }
}

    問題の発生：不正な操作

    public class DangerTest {
    public static void main(String[] args) {
        DangerousStudent student = new DangerousStudent("田中太郎", 20, 3.5);

        // ✅ 正常な使用
        student.showInfo();

        // ❌ 危険：直接フィールドを変更可能
        student.name = "";           // 空文字に設定（異常値）
        student.age = -5;            // 負の年齢（不正値）
        student.gpa = 99.9;          // 不正なGPA（4.0が最大なのに）
        student.subjects = null;     // nullに設定（危険）

        System.out.println("\n=== 不正操作後 ===");
        student.showInfo();  // 異常な状態での表示

        // さらに危険な操作
        if (student.subjects != null) {
            student.subjects.add("不正科目");
        }
    }
}

    実行結果： 

    名前：田中太郎、年齢：20歳、GPA：3.5

=== 不正操作後 ===
名前：、年齢：-5歳、GPA：99.9

    📘 publicの危険性

    なぜこれが問題なのか？

    1. データの整合性が保てない: 年齢-5歳、GPA 99.9など異常値
    2. 予期しないエラーの原因: null参照による実行時エラー
    3. デバッグの困難: どこでデータが変更されたか追跡不可
    4. 保守の困難: 仕様変更時に全ての直接アクセス個所を修正必要

    このような問題を解決するのが「カプセル化」です！

    🛡️ 解決策：privateによるカプセル化

    public class SafeStudent {

    public class Book {
    // カプセル化：すべてprivateフィールド
    private String title;
    private String author;
    private double price;
    private int pages;
    private boolean available;

    // 検証付きコンストラクタ
    public Book(String title, String author, double price, int pages) {
        this.title = (title != null && !title.trim().isEmpty()) ? title : "タイトル未設定";
        this.author = (author != null && !author.trim().isEmpty()) ? author : "著者未設定";
        this.price = (price >= 0) ? price : 0.0;
        this.pages = (pages > 0) ? pages : 1;
        this.available = true; // 初期状態は貸出可能
    }

    // 安全な更新メソッド
    public void updatePrice(double newPrice) {
        if (newPrice >= 0) {
            this.price = newPrice;
            System.out.println("価格を" + newPrice + "円に更新しました");
        } else {
            System.out.println("無効な価格です（0円以上）");
        }
    }

    public void updatePages(int newPages) {
        if (newPages > 0) {
            this.pages = newPages;
            System.out.println("ページ数を" + newPages + "ページに更新しました");
        } else {
            System.out.println("無効なページ数です（1ページ以上）");
        }
    }

    // 貸出管理機能
    public boolean borrowBook() {
        if (this.available) {
            this.available = false;
            System.out.println("「" + this.title + "」を貸し出しました");
            return true;
        } else {
            System.out.println("「" + this.title + "」は貸出中です");
            return false;
        }
    }

    public void returnBook() {
        if (!this.available) {
            this.available = true;
            System.out.println("「" + this.title + "」が返却されました");
        } else {
            System.out.println("「" + this.title + "」は既に貸出可能です");
        }
    }

    // 読み取り専用メソッド（getter）
    public String getTitle() {
        return this.title;
    }

    public String getAuthor() {
        return this.author;
    }

    public double getPrice() {
        return this.price;
    }

    public int getPages() {
        return this.pages;
    }

    public boolean isAvailable() {
        return this.available;
    }

    public String getBookInfo() {
        return "「" + this.title + "」" + this.author + "著 " + 
               this.price + "円 " + this.pages + "ページ " +
               (this.available ? "貸出可能" : "貸出中");
    }

    // 比較・判定機能（thisキーワード活用）
    public boolean isSameAuthor(Book other) {
        return this.author.equals(other.author);
    }

    public boolean isExpensive(double threshold) {
        return this.price >= threshold;
    }

    public boolean isThicker(Book other) {
        return this.pages > other.pages;
    }

    💡 よくある質問

    Q1: すべてのフィールドをprivateにする必要がありますか？

    A: 基本的にははい、ただし例外もあります：

    public class Student {
    // ✅ 通常：全てprivate
    private String name;
    private int age;

    // ✅ 例外1：定数はpublic static final
    public static final int MAX_AGE = 150;
    public static final double MAX_GPA = 4.0;

    // ✅ 例外2：設定値など意図的に外部変更を許可する場合
    public boolean debugMode = false; // デバッグフラグなど
}

    原則: データを保護したいならprivate、外部から直接アクセスしてもよいならpublic

    Q2: privateフィールドにアクセスするgetterメソッドは必ず必要ですか？

    A: 必要な場合のみ作成してください：

    public class BankAccount {
    private String accountNumber;
    private double balance;
    private String password; // パスワードは絶対に外部に出してはいけない

    // ✅ 必要：口座番号は表示に使用
    public String getAccountNumber() {
        return this.accountNumber;
    }

    // ✅ 必要：残高は確認に使用
    public double getBalance() {
        return this.balance;
    }

    // ❌ 不要：パスワードは外部に出してはいけない
    // public String getPassword() { return this.password; } // 作らない
}

    Q3: thisキーワードは必ず書く必要がありますか？

    A: 必須ではありませんが、強く推奨します：

    public class Student {
    private String name;

    // 引数とフィールドが同名：thisが必須
    public void setName(String name) {
        this.name = name; // thisがないとエラー
    }

    // 引数が異なる名前：thisは省略可能だが推奨
    public void setName(String newName) {
        this.name = newName; // 推奨：thisありで明確に
        name = newName;      // 動作するが非推奨
    }
}

    推奨理由: コードの可読性と保守性の向上

    Q4: privateメソッドはいつ使うのですか？

    A: クラス内部でのみ使用するヘルパーメソッドに使用します：

    public class Student {
    private String name;
    private double gpa;

    // publicメソッド：外部から使用
    public void updateGpa(double newGpa) {
        if (isValidGpa(newGpa)) { // privateメソッドを呼び出し
            this.gpa = newGpa;
            System.out.println("GPA更新：" + formatGpa(newGpa)); // privateメソッド使用
        }
    }

    // privateメソッド：内部処理のみで使用
    private boolean isValidGpa(double gpa) {
        return gpa >= 0.0 && gpa <= 4.0;
    }

    private String formatGpa(double gpa) {
        return String.format("%.1f", gpa);
    }
}
