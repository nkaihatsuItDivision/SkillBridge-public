    🏗️ フィールドの詳細技術

    1. アクセス修飾子による保護

    フィールドにはアクセス修飾子を使って、適切な保護を行います。

    📘 アクセス修飾子による保護

    public class Student {
    // 推奨：privateで保護
    private String name;        // 外部から直接アクセス不可
    private int score;         // データの整合性を保護

    // 非推奨：publicは危険
    // public String name;     // 誰でも変更可能で危険
}

    なぜprivateが重要なのか？

    // ❌ publicフィールドの問題例
public class BadStudent {
    public String name;
    public int score;
}

// 使用時の問題
BadStudent student = new BadStudent();
student.score = -50;      // 負の点数？不正なデータ！
student.name = "";        // 空の名前？不正なデータ！

    // ✅ privateフィールドの安全性
public class GoodStudent {
    private String name;
    private int score;

    // メソッドでデータの整合性を保護
    public void setScore(int score) {
        if (score >= 0 && score <= 100) {
            this.score = score;
        } else {
            System.out.println("不正な点数です: " + score);
        }
    }
}

    private List<String> subjects = new ArrayList<>();  // 空リストで初期化

    // 方法3: コンストラクタでの初期化（後のレッスンで詳細学習）
    public Student() {
        // コンストラクタ内での初期化
    }
}

    初期化のベストプラクティス

    public class Student {
    // ✅ 適切な初期化
    private String name = "";                    // 空文字で初期化
    private int score = 0;                      // ゼロで初期化
    private boolean isActive = true;            // デフォルト値で初期化
    private List<String> subjects = new ArrayList<>();  // 空リストで初期化

    // ❌ 危険な初期化なし
    // private String name;        // null になり、NullPointerException の原因
    // private List<String> subjects;  // null のまま、使用時にエラー
}

    4. staticフィールド：クラス変数の活用

    staticフィールドとインスタンスフィールドの違い

    📘 staticとインスタンスフィールドの違い

    public class Student {
    // インスタンスフィールド（各学生固有のデータ）
    private String name;
    private int score;

    // staticフィールド（全学生共通のデータ）
    private static int totalStudents = 0;           // 学生の総数
    private static String schoolName = "○○高校";    // 学校名
    public static final int MAX_SCORE = 100;        // 最高点（定数）

    public Student(String name) {
        this.name = name;
        totalStudents++;  // 学生が作られるたびに総数を増加
    }

    // staticメソッドで静的フィールドにアクセス
    public static int getTotalStudents() {
        return totalStudents;
    }
}

    使用例での違いを確認

    public class Main {
    public static void main(String[] args) {

    import java.time.LocalDate;
import java.time.Period;
import java.util.ArrayList;
import java.util.List;

/**
 * 学生を表すクラス（完全版）
 * フィールドの詳細技術を活用した実用的な実装
 */
public class Student {
    // インスタンスフィールド（各学生固有のデータ）
    private String name = "";
    private LocalDate birthDate = LocalDate.now().minusYears(18);
    private double gpa = 0.0;
    private char grade = &apos;F&apos;;
    private boolean isActive = true;
    private List<String> subjects = new ArrayList<>();
    private List<Integer> testScores = new ArrayList<>();

    // staticフィールド（全学生共通のデータ）
    private static int totalStudents = 0;
    private static String schoolName = "プログラミング高校";

    // 定数定義
    public static final double MAX_GPA = 4.0;
    public static final double MIN_GPA = 0.0;
    public static final int MAX_AGE = 100;
    public static final int MIN_AGE = 6;

    // コンストラクタ（基本版）
    public Student(String name) {
        if (name != null && !name.trim().isEmpty()) {
            this.name = name.trim();
        }
        totalStudents++;
        System.out.println("学生 " + this.name + " が登録されました。");
    }

    // 年齢を計算するメソッド
    public int getAge() {
        return Period.between(birthDate, LocalDate.now()).getYears();
    }

    // GPAを安全に設定するメソッド
    public void setGpa(double gpa) {
        if (gpa >= MIN_GPA && gpa <= MAX_GPA) {
            this.gpa = gpa;
            updateGrade();
        } else {
            System.out.println("GPAは" + MIN_GPA + "から" + MAX_GPA + "の間で設定してください");
        }
    }

    // GPAに基づいてグレードを更新
    private void updateGrade() {
        if (gpa >= 3.5) {
            grade = &apos;A&apos;;
        } else if (gpa >= 3.0) {
            grade = &apos;B&apos;;
        } else if (gpa >= 2.0) {
            grade = &apos;C&apos;;
        } else if (gpa >= 1.0) {
            grade = &apos;D&apos;;
        } else {
            grade = &apos;F&apos;;
        }
    }

    // 科目を追加
    public void addSubject(String subject) {
        if (subject != null && !subject.trim().isEmpty()) {
            subjects.add(subject.trim());
            System.out.println(name + " が科目「" + subject + "」を履修しました。");
        }
    }

    // テスト点数を追加
    public void addTestScore(int score) {
        if (score >= 0 && score <= 100) {
            testScores.add(score);
            System.out.println(name + " のテスト点数 " + score + " 点が記録されました。");
        } else {
            System.out.println("テスト点数は0から100の間で入力してください");
        }
    }

    // 学生情報の表示
    public void showInfo() {
        System.out.println("=== 学生情報 ===");
        System.out.println("名前: " + name);
        System.out.println("年齢: " + getAge() + "歳");
        System.out.println("GPA: " + gpa);
        System.out.println("成績: " + grade);
        System.out.println("在学中: " + (isActive ? "はい" : "いいえ"));
        System.out.println("履修科目数: " + subjects.size());
        System.out.println("テスト受験回数: " + testScores.size());
    }

    // staticメソッド（クラスの情報を取得）
    public static int getTotalStudents() {
        return totalStudents;
    }

    public static String getSchoolName() {
        return schoolName;
    }

    public static void showSchoolInfo() {
        System.out.println("=== 学校情報 ===");
        System.out.println("学校名: " + schoolName);
        System.out.println("在籍学生数: " + totalStudents + "名");
    }
}

    💡 よくある質問

    Q1: フィールドを全てprivateにする必要がありますか？

    A: 基本的にはyesです。データの整合性とセキュリティを保つため、フィールドは原則privateにします：

    // ✅ 推奨：privateで保護
private String name;
private int score;

// ❌ 避けるべき：publicは危険
public String name;    // 外部から自由に変更可能

    例外的にpublicにする場合: - 定数（public static final） - 設定値として外部から参照する必要がある場合

    Q2: staticフィールドとインスタンスフィールドの使い分けは？

    A: データの性質で判断します：

    public class Student {
    // インスタンスフィールド：各学生固有のデータ
    private String name;        // 学生ごとに異なる
    private int score;         // 学生ごとに異なる

    // staticフィールド：全学生共通のデータ
    private static int totalStudents;     // 全体で共有
    private static String schoolName;     // 全体で共有
    public static final int MAX_SCORE = 100;  // 定数
}

    判断基準: - 各インスタンス固有 → インスタンスフィールド - クラス全体で共有 → staticフィールド - 変更されない値 → public static final（定数）

    Q3: フィールドの初期化はいつ行うのがベストですか？

    A: 宣言時の初期化が最も安全で推奨されます：

    public class Student {
    // ✅ 推奨：宣言時に安全な値で初期化
    private String name = "";
    private int score = 0;
    private boolean isActive = true;
    private List<String> subjects = new ArrayList<>();  // 空リスト

    // ❌ 危険：初期化なし
    // private String name;        // null → NullPointerException
    // private List<String> subjects;  // null → 使用時エラー
}

    初期化の優先順位: 1. 宣言時初期化 （最優先・最安全） 2. コンストラクタ初期化 （次回レッスンで詳細学習） 3. メソッド内初期化 （特殊な場合のみ）

    Q4: データ型の選択で迷った時はどうすればいいですか？

    A: データの性質と用途を考えて選択します：

    // 整数データの選択
private int score;          // 点数（0-100程度）
private long studentId;     // 学生番号（大きな数値）

// 文字列データの選択
private String name;        // 名前（可変長文字列）
private char grade;         // 成績（A, B, C, D, F の1文字）

// 真偽値データの選択
private boolean isActive;   // 在学中かどうか

// 日付データの選択
private LocalDate birthDate;    // 生年月日（日付のみ）
private LocalDateTime lastLogin; // 最終ログイン（日時）

// コレクションデータの選択
private List<String> subjects;      // 順序ありの科目リスト
private Set<String> uniqueSkills;   // 重複なしのスキルセット

    選択の指針: - 数値の範囲を考慮（int vs long） - 精度の必要性を考慮（int vs double） - データの性質を考慮（String vs char） - 重複の許可を考慮（List vs Set）
